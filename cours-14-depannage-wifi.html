Signal strength

Signal faible — RSSI < -70 dBm

RSSI (Received Signal Strength Indicator) = Mesure puissance signal Wi-Fi reçu (dBm). Signal faible → slow speeds, disconnections.

// RSSI scale (dBm)

RSSI (dBm)QualityPerformance
-30 to -50 ✅ Excellent Maximum throughput (400-1000+ Mbps)
-50 to -60 ✅ Very Good High throughput (200-400 Mbps)
-60 to -70 ⚠️ Good Moderate (50-200 Mbps), occasional drops
-70 to -80 ❌ Weak Poor (10-50 Mbps), frequent disconnects
-80 to -90 ❌ Very Weak Barely usable (<10 Mbps), unstable
< -90 ❌ No signal Connection impossible

// Symptômes signal faible

  • Slow speeds : Downloads slow, streaming buffering
  • Intermittent connectivity : Disconnects randomly
  • High latency : Ping times high (>50 ms), jittery
  • Cannot connect : Association fails (signal too weak)
  • Low Wi-Fi bars : 1-2 bars displayed (instead 4-5)

// Causes signal faible

1. Distance trop grande (AP → client)

Indoor range typical :
2.4 GHz : ~45 meters (150 feet)
5 GHz : ~15 meters (50 feet) — shorter range, higher speed

→ Client beyond range → Signal weak/none

2. Obstacles physiques (walls, metal)

  • Concrete walls : -10 to -20 dB signal loss
  • Metal (elevator shafts, filing cabinets) : -20+ dB loss
  • Wood/drywall : -3 to -5 dB loss
  • Water (aquarium, water pipes) : Significant absorption

3. Low AP transmit power

AP configured : Transmit power LOW (25%)
→ Reduced coverage area
→ Weak signal edges coverage

4. Antenna positioning incorrect

  • AP antennas horizontal (should be vertical dipole)
  • Directional antenna pointing wrong direction
  • Client device antenna obstructed (laptop closed partially)

// Diagnostic signal faible

Windows :
> netsh wlan show interfaces

SSID : CorpWiFi
Signal : 45% (weak - ~-75 dBm)

Linux :
$ iwconfig wlan0
Link Quality=25/70  Signal level=-78 dBm

Wi-Fi Analyzer app (smartphone) :
→ Shows RSSI graphically (-30 to -90 dBm scale)
→ Walk around measure signal strength different locations

// Résolution signal faible

  • Move closer to AP : Reduce distance improves signal
  • Add APs : Increase coverage (more access points, mesh network)
  • Increase transmit power : AP config (25% → 100%)
  • Reposition AP : Central location, elevated, minimize obstacles
  • Use 2.4 GHz : Better wall penetration (vs 5 GHz)
  • External antenna : Upgrade AP antenna (high-gain directional)
  • Wi-Fi extender/repeater : Boost signal weak areas
🎯
Question fréquente examen "User reports slow Wi-Fi. RSSI -75 dBm. BEST solution ?"
A) Change channel
B) Move closer to AP OR add AP
C) Update drivers
D) Reboot router

Réponse : B) Move closer to AP OR add AP
RSSI -75 dBm = weak signal (distance/obstacles problem).
Channel congestion

Interférences canal — Channel interference

Interférences canal = Multiple APs same/overlapping channel → collision frames → retransmissions → slow speeds.

// 2.4 GHz channels (rappel)

Channels 2.4 GHz : 1-11 (US), 1-13 (EU)
Channel width : 20-22 MHz
Non-overlapping channels : 1, 6, 11 (US)

Problem overlapping channels :
AP-A channel 1, AP-B channel 2 → Overlap (interfere)
→ Collision frames → Retransmissions → Slow

// Sources interférences

1. Co-channel interference (same channel)

Office building :
Your AP : Channel 6
Neighbor APs : 5 others channel 6

→ All APs compete airtime (CSMA/CA)
→ High contention → Slow speeds

2. Adjacent channel interference (overlapping)

Your AP : Channel 6
Neighbor : Channel 5 (overlaps 6)

→ Partial frequency overlap → Interference

3. Non-Wi-Fi interference (2.4 GHz band)

  • Microwave ovens : 2.45 GHz (same as Wi-Fi channel 11)
  • Bluetooth : 2.4 GHz (frequency hopping, minimal impact)
  • Cordless phones : 2.4 GHz (older DECT phones)
  • Baby monitors : 2.4 GHz
  • Wireless cameras : 2.4 GHz analog

// Symptômes interférences

  • Slow speeds despite good signal : RSSI -50 dBm BUT 5 Mbps (high interference)
  • High retransmissions : Wi-Fi analyzer shows retry rate >10%
  • Intermittent drops : Connection stable, then sudden drops
  • Timing patterns : Slow lunchtime (microwave usage)

// Diagnostic interférences

Wi-Fi Analyzer (smartphone app) :

Channel graph shows :
Channel 1 : 3 APs (-60, -65, -70 dBm)
Channel 6 : 8 APs (-50 to -80 dBm) — CONGESTED
Channel 11 : 2 APs (-75, -80 dBm) — Less congested

→ Your AP channel 6 (heavily congested)
→ Recommendation : Switch channel 11 (fewer APs)

// Résolution interférences

Change channel (non-overlapping)

2.4 GHz :
→ Use 1, 6, OR 11 (non-overlapping)
→ Choose least congested (Wi-Fi Analyzer scan)

5 GHz :
→ Many non-overlapping channels (20+)
→ Less congestion (fewer devices)
→ Recommended modern networks

Enable auto-channel selection

AP configuration :
→ Channel : Auto (scans least congested, adapts)
→ Modern APs : Dynamic Frequency Selection (DFS)

Reduce channel width (20 MHz)

2.4 GHz : 20 MHz (default) vs 40 MHz (wider, more interference)
→ 40 MHz = faster speeds BUT higher interference
→ 20 MHz = slower BUT more reliable congested environments

Migrate to 5 GHz

  • 5 GHz : More channels, less congestion, faster speeds
  • 2.4 GHz : Legacy devices, better range/wall penetration
  • Dual-band APs : Support both (best both worlds)
Security threat

Evil Twin — Rogue AP attaque

Evil Twin = Attacker AP same SSID legitimate → users connect attacker → man-in-the-middle (Cours S04 rappel).

// Scénario Evil Twin

Legitimate AP :
SSID : "CoffeeShop-WiFi"
Security : WPA2-PSK (password protected)
Channel : 6

Attacker Evil Twin :
SSID : "CoffeeShop-WiFi" (identical!)
Security : Open (no password) — easier connect
Channel : 11
Transmit power : HIGH (stronger signal attracts users)

User behavior :
Laptop sees 2 "CoffeeShop-WiFi" networks
Connects stronger signal (Evil Twin)
→ All traffic routed through attacker
→ Credentials, browsing captured

// Symptômes Evil Twin

  • Multiple same SSID : Wi-Fi list shows duplicate SSIDs
  • Unexpected security : Usually WPA2, suddenly Open
  • Captive portal phishing : Fake login page (harvest credentials)
  • SSL certificate warnings : HTTPS sites show cert errors (MitM)
  • Different MAC address : Same SSID, different BSSID (AP MAC)

// Diagnostic Evil Twin

Wi-Fi Analyzer :

SSID : "CoffeeShop-WiFi"
BSSID (AP MAC) : aa:bb:cc:dd:ee:ff
Channel : 6, RSSI -55 dBm
Security : WPA2-PSK

SSID : "CoffeeShop-WiFi" (duplicate)
BSSID (AP MAC) : 11:22:33:44:55:66 (different!)
Channel : 11, RSSI -50 dBm (stronger)
Security : Open

→ Different BSSID same SSID = suspect Evil Twin
→ Verify with admin : "Do we have 2 APs same SSID ?"

// Résolution / Prévention Evil Twin

User side (protect yourself)

  • Verify network : Ask staff correct SSID, security type
  • Use VPN : Encrypt all traffic (even on Evil Twin, attacker sees encrypted)
  • Check HTTPS : Verify SSL certificates valid (not self-signed warnings)
  • Avoid auto-connect : Disable auto-join public networks

Network admin (detect/prevent)

  • WIDS (Wireless IDS) : Monitor rogue APs (Cisco, Aruba)
  • WPA2-Enterprise : Mutual authentication (certificate validation prevents Evil Twin)
  • Regular scans : Wi-Fi survey detect unauthorized APs
  • MAC filtering : Whitelist authorized AP MACs (clients reject others)
⚠️
Evil Twin = serious security threat Attacker intercepts: Passwords (email, banking), cookies (session hijacking), browsing (tracking).
WPA2-Enterprise + VPN = defense layers.
Public Wi-Fi : Always assume untrusted (use VPN).
Network identification

Mauvais SSID — Wrong network

Mauvais SSID = Client connected wrong network (similar name, auto-connect, misconfiguration) → no access resources.

// Scénarios mauvais SSID

1. Similar SSID names (typo/confusion)

Corporate : "CompanyWiFi" (correct)
Guest : "Company-Guest" (different network)

User connects "Company-Guest" (mistake)
→ Gets IP, Internet works
→ Cannot access internal resources (different VLAN, firewall)

2. Hidden SSID (manual entry error)

Correct SSID : "CorpSecure" (hidden, case-sensitive)
User types : "corpsecure" (lowercase)

→ Connection fails (SSID doesn't exist)

3. Auto-connect wrong network

Laptop saved : "Home-WiFi", "Starbucks", "Office"
At office : Auto-connects "Starbucks" (neighbor AP same SSID)

→ Wrong network, no office access

// Symptômes mauvais SSID

  • Internet works, internal doesn't : Can browse web, can't access file server
  • Different IP range : Expected 192.168.10.x, got 192.168.99.x (guest network)
  • Connection works, resources fail : Connected, but applications timeout

// Diagnostic mauvais SSID

Windows :
> netsh wlan show interfaces

SSID : Company-Guest (wrong!)
BSSID : 11:22:33:44:55:66

Should be : "CompanyWiFi"

Verify IP address :
C:\> ipconfig
IP : 192.168.99.50 (guest network range)
Should be : 192.168.10.x (corporate range)

// Résolution mauvais SSID

  • Disconnect, reconnect correct SSID : Forget network, join correct
  • Disable auto-connect : Prevent wrong network auto-join
  • Verify SSID spelling : Case-sensitive, exact match
  • Remove saved networks : Delete old/untrusted SSIDs
WPA3 Enterprise

Problèmes certificat WPA3 — Certificate issues

WPA3-Enterprise = Uses certificates (802.1X authentication). Certificate problems → authentication fails → cannot connect.

// WPA3-Enterprise authentication (rappel)

EAP-TLS workflow :

1. Client connects AP (SSID)
2. AP requests authentication (802.1X)
3. Client presents client certificate
4. RADIUS server validates certificate
5. RADIUS server presents server certificate
6. Client validates server certificate
7. If both valid → Authentication success → Access granted

// Problèmes certificat

1. Server certificate expired

RADIUS server certificate :
Valid from : 2023-01-01
Valid to : 2024-12-31 (expired!)
Today : 2026-03-01

→ Client rejects server certificate (expired)
→ Authentication fails
→ Cannot connect

2. Client certificate missing/invalid

Client : No certificate installed (OR expired)
RADIUS : Requires client certificate

→ RADIUS rejects authentication
→ Connection denied

3. CA certificate not trusted

Server certificate : Signed by "Company Internal CA"
Client : "Company Internal CA" not in trusted CA list

→ Client rejects server certificate (untrusted CA)
→ Authentication fails

4. Certificate name mismatch

Server certificate : CN=radius.company.com
Client configured : Validate server = radius-server.company.com

→ Name mismatch
→ Client rejects certificate

// Symptômes problèmes certificat

  • Authentication failed : "Cannot connect to network"
  • Certificate warnings : "Server certificate invalid/expired"
  • Prompt accept certificate : User asked accept untrusted cert (security risk)
  • RADIUS logs : "Certificate validation failed" errors

// Résolution problèmes certificat

Renew expired certificates

RADIUS server :
→ Request new certificate from CA
→ Install renewed certificate (valid 1-3 years)
→ Monitor expiration (auto-renewal recommended)

Install client certificates

Client :
→ Deploy client certificates (Group Policy, MDM)
→ Install in Personal certificate store
→ Configure Wi-Fi profile select correct certificate

Trust CA certificate

Client :
→ Import CA certificate (Trusted Root CA store)
→ Windows : certmgr.msc → Trusted Root Certification Authorities
→ Deploy via GPO (centralized)

Verify certificate names

Wi-Fi profile :
→ Server validation : Match CN=radius.company.com
→ OR disable validation (NOT recommended production)
🔐
WPA3-Enterprise = highest security Mutual authentication (both client + server validated).
Prevents Evil Twin (attacker can't fake certificate).
Certificate management : Critical (PKI infrastructure, monitoring expiration).
⚠️ Pièges d'examen

Ce qui trompe les candidats

❌ Piège #1 : Good Wi-Fi bars = good performance
FAUX. Wi-Fi bars = signal strength, NOT performance.

Can have : Good signal (-50 dBm) BUT slow speeds (interference, congestion).
Performance factors : Signal (RSSI) + Interference + Channel congestion.

Examen : Good signal + slow = suspect interference/congestion (not signal).
❌ Piège #2 : 5 GHz always better than 2.4 GHz
NUANCE. 5 GHz = faster, 2.4 GHz = better range/penetration.

5 GHz : Higher speeds, less congestion, shorter range (doesn't penetrate walls well).
2.4 GHz : Slower speeds, more congestion, better range (penetrates walls).

Best : Dual-band network (both available), client chooses based location/needs.
Examen : Context matters (large building with walls → 2.4 GHz may be better).
❌ Piège #3 : WPA3 = no security issues possible
FAUX. WPA3 = stronger security, NOT invulnerable.

WPA3-Enterprise : Certificate issues (expiration, trust, misconfiguration) → authentication fails.
WPA3-Personal : Transition mode (allows WPA2 fallback) → downgrade attacks possible.

Still vulnerable : Evil Twin (if users bypass warnings), deauth attacks (DoS).
Examen : WPA3 improves security, doesn't eliminate all Wi-Fi problems.
