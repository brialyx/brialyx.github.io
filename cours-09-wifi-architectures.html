Architecture de base

BSS (Basic Service Set)

📡
Analogie — cellule téléphonique BSS = 1 antenne de téléphone portable avec sa zone de couverture. Tous les téléphones dans la zone se connectent à cette antenne. En Wi-Fi, BSS = 1 Access Point + tous ses clients dans sa zone de couverture.

BSS (Basic Service Set) = unité de base d'un réseau Wi-Fi. 1 AP + tous les clients dans sa portée radio.

// Composants BSS

  • 1 Access Point (AP) : Point d'accès sans fil
  • Clients (stations) : Laptops, smartphones, tablettes, IoT
  • Zone de couverture : Portée radio de l'AP (~30-100m selon obstacles)

// BSSID (Basic Service Set Identifier)

BSSID = adresse MAC de l'AP. Identifie UNIQUEMENT cet AP parmi tous les autres.

Format : 00:1A:2B:3C:4D:5E (adresse MAC 48 bits)

🎯
BSSID = MAC de l'AP 1 AP = 1 BSSID. Si AP dual-band (2.4 + 5 GHz) → 2 BSSIDs (1 MAC par bande).
Identification réseau

SSID (Service Set Identifier)

SSID = nom du réseau Wi-Fi visible par utilisateurs. C'est ce que tu vois quand tu cherches réseaux disponibles.

// Caractéristiques SSID

  • Longueur : 0-32 caractères (ASCII)
  • Case-sensitive : "WiFi" ≠ "wifi"
  • Broadcast : Annoncé dans beacons par défaut
  • Hidden SSID : Peut être masqué (non broadcast)

// SSID vs BSSID

CritèreSSIDBSSID
C'est quoiNom réseau (lisible)Adresse MAC de l'AP
FormatTexte (0-32 char)MAC 48 bits
VisibleOui (sauf hidden)Oui (identifiant technique)
UniqueNon (plusieurs APs même SSID)Oui (1 BSSID = 1 AP)
Exemple"Corp-WiFi"00:1A:2B:3C:4D:5E
⚠️
Hidden SSID ≠ sécurité Masquer SSID n'empêche PAS détection. Outils (Wireshark, airodump-ng) voient toujours réseau. Hidden SSID = "security through obscurity" (inefficace). Vrais sécurité = WPA3 + long mot de passe.
Architecture étendue

ESS (Extended Service Set)

ESS = plusieurs APs avec même SSID connectés au même réseau. Permet roaming (itinérance) entre APs.

// ESS = Plusieurs BSS interconnectés

ESS "Corp-WiFi" :

AP1 (BSSID: 00:11:22:33:44:55) → SSID "Corp-WiFi"
AP2 (BSSID: AA:BB:CC:DD:EE:FF) → SSID "Corp-WiFi"
AP3 (BSSID: 11:22:33:44:55:66) → SSID "Corp-WiFi"

→ Utilisateur voit 1 seul réseau "Corp-WiFi"
→ En réalité 3 APs différents (3 BSSIDs)
→ Client bascule automatiquement entre APs (roaming)

// Roaming (itinérance)

Client se déplace entre zones de couverture d'APs différents → bascule automatiquement vers AP avec meilleur signal SANS perdre connexion.

Conditions roaming :

  • Tous APs avec même SSID
  • Tous APs sur même VLAN/subnet (Layer 2 roaming)
  • Zones de couverture avec overlap 10-15%
  • Même sécurité (WPA2/WPA3) et même clé
✅
ESS = campus/entreprise Bureau avec 50 APs "Corp-WiFi" → 1 ESS, 50 BSS. Employé se déplace librement, toujours connecté au même SSID. Smartphone bascule automatiquement entre APs selon signal.
Architecture Ad-Hoc

IBSS (Independent Basic Service Set)

IBSS = réseau Wi-Fi peer-to-peer SANS Access Point. Clients communiquent directement entre eux.

Aussi appelé : Ad-Hoc mode

// Cas d'usage IBSS

  • Partage fichiers entre 2 laptops (pas d'infrastructure)
  • Gaming LAN party temporaire
  • Connexion directe entre appareils (rare aujourd'hui)

// IBSS vs BSS

CritèreBSS (Infrastructure)IBSS (Ad-Hoc)
AP requis✅ Oui❌ Non
TopologieÉtoile (clients→AP)Mesh (peer-to-peer)
PortéeZone AP (~100m)Portée entre clients
UsageStandard (99% des cas)Temporaire, rare
💡
IBSS = quasi obsolète Aujourd'hui remplacé par Wi-Fi Direct, Bluetooth, AirDrop. IBSS rarement utilisé en production. CompTIA teste concept pour culture générale.
Gestion centralisée

WLC (Wireless LAN Controller)

WLC (Wireless LAN Controller) = équipement centralisant gestion de tous les APs. Contrôle configuration, sécurité, roaming.

// Fonctions WLC

  • Configuration centralisée : SSIDs, canaux, puissance TX, sécurité
  • Gestion APs légers : APs "dumb" sans config locale
  • Roaming fluide : Orchestration handoffs entre APs
  • Sécurité : Authentification RADIUS, policies centralisées
  • Monitoring : Surveillance trafic, clients, performances
  • Load balancing : Distribution clients entre APs

// Architecture WLC

[WLC] ←→ [Switch Core]
↓ CAPWAP tunnels
[AP1] [AP2] [AP3] ... [AP50]
↓ Radio
[Clients Wi-Fi]

→ WLC contrôle 50 APs depuis 1 point central
→ Admin configure SSIDs/sécurité sur WLC uniquement
→ Tous APs appliquent config automatiquement

// Avantages WLC

  • ✅ Config centralisée (pas AP par AP)
  • ✅ Roaming Layer 3 (entre subnets)
  • ✅ Scalabilité (centaines d'APs)
  • ✅ Monitoring unifié
  • ✅ Zero-touch provisioning APs
Protocole

CAPWAP — protocole WLC ↔ AP

CAPWAP (Control And Provisioning of Wireless Access Points) = protocole standard communication WLC ↔ APs légers.

// 2 tunnels CAPWAP

1. Control tunnel (UDP 5246)

  • Configuration AP (SSID, canal, puissance)
  • Monitoring, statistiques
  • Management, découverte APs

2. Data tunnel (UDP 5247)

  • Trafic client encapsulé (données utilisateurs)
  • Client → AP → CAPWAP tunnel → WLC → réseau
🎯
Ports CAPWAP à retenir Control : UDP 5246
Data : UDP 5247

// LWAPP vs CAPWAP

LWAPP (Lightweight Access Point Protocol) = prédécesseur propriétaire Cisco de CAPWAP. CAPWAP = standard IETF RFC 5415/5416 (successeur).

Types d'AP

AP Autonome vs AP Léger

// AP Autonome (Standalone / Fat AP)

AP indépendant avec config locale complète. Fonctionne seul sans WLC.

Caractéristiques :

  • Configuration via interface web locale
  • Intelligence complète (DHCP, NAT, firewall, VPN...)
  • Pas besoin WLC
  • Usage : SOHO, petits bureaux (1-5 APs)

// AP Léger (Lightweight / Thin AP)

AP "dumb" qui dépend totalement du WLC. Pas de config locale.

Caractéristiques :

  • Zéro config locale (WLC configure tout via CAPWAP)
  • Juste radio + tunneling vers WLC
  • Requiert WLC obligatoirement
  • Usage : Entreprises, campus, datacenters (10-1000+ APs)

// Comparaison

⚠️ Pièges d'examen

Ce qui trompe les candidats

❌ Piège #1 : SSID = BSSID
FAUX.
SSID : Nom réseau (texte, plusieurs APs peuvent avoir même)
BSSID : Adresse MAC de l'AP (unique par AP)
❌ Piège #2 : ESS = 1 AP
FAUX. ESS = plusieurs APs avec même SSID.
BSS = 1 AP
❌ Piège #3 : CAPWAP port TCP 5246
FAUX. CAPWAP = UDP (pas TCP).
Control : UDP 5246, Data : UDP 5247
Résumé

🔑 Ce que tu dois retenir absolument

→BSS : 1 AP + clients. BSSID = MAC de l'AP (unique)
→SSID : Nom réseau Wi-Fi (0-32 char, plusieurs APs peuvent avoir même)
→ESS : Plusieurs APs même SSID → roaming entre APs
→IBSS : Ad-Hoc peer-to-peer sans AP (obsolète)
→WLC : Contrôleur centralisé gérant tous APs légers
→CAPWAP : Protocole WLC↔AP. UDP 5246 (control) + UDP 5247 (data)
→AP Autonome : Config locale, indépendant. AP Léger : Dépend WLC via CAPWAP
