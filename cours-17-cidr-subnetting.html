Avant de commencer

C'est quoi le subnetting — découper un réseau en sous-réseaux

🏢
Analogie — diviser un immeuble en appartements Tu as un grand immeuble (réseau 192.168.1.0/24 = 256 adresses). Le subnetting, c'est comme diviser l'immeuble en plusieurs petits appartements (sous-réseaux). Au lieu d'un grand espace pour 254 personnes, tu crées 4 appartements de 62 personnes chacun. Plus facile à gérer, plus de sécurité, moins de bordel.

Subnetting = diviser un grand réseau en plusieurs sous-réseaux plus petits. CIDR (Classless Inter-Domain Routing) = notation moderne pour spécifier la taille d'un réseau.

Pourquoi faire du subnetting ?

  • Organisation : Séparer les départements (RH, IT, Finance) sur des sous-réseaux différents
  • Sécurité : Isoler les réseaux sensibles (serveurs, VLAN invités)
  • Performance : Réduire les domaines de broadcast → moins de trafic inutile
  • Économie d'adresses : Utiliser exactement la taille nécessaire (pas de gaspillage)
🎯
Pour l'examen Network+ Le subnetting est L'UN DES SUJETS LES PLUS TESTÉS. CompTIA pose des questions de calcul direct : "Quel est le réseau ? Le broadcast ? Combien d'hôtes ?" Tu DOIS maîtriser les calculs rapides et connaître les masques courants par cœur.
Vocabulaire essentiel

Les mots clés de ce cours

CIDR (Classless Inter-Domain Routing)
Notation moderne pour spécifier un masque. Format : IP/préfixe (ex: 192.168.1.0/24). Le /24 indique combien de bits sont à 1 dans le masque.
Masque de sous-réseau (Subnet Mask)
Nombre 32 bits qui définit quelle partie de l'IP = réseau et quelle partie = hôte. Format décimal (255.255.255.0) ou CIDR (/24).
Adresse réseau (Network Address)
Premier IP du sous-réseau. Tous les bits hôte = 0. Non assignable à un appareil. Identifie le réseau lui-même.
Adresse broadcast
Dernier IP du sous-réseau. Tous les bits hôte = 1. Envoie à TOUS les hôtes du réseau. Non assignable.
Hôtes utilisables
IPs entre l'adresse réseau et le broadcast. Assignables aux appareils. Nombre = 2^(bits hôte) - 2.
Préfixe (Prefix Length)
Nombre de bits à 1 dans le masque. /24 = 24 bits réseau, 8 bits hôte. /25 = 25 bits réseau, 7 bits hôte.
Notation moderne

Notation CIDR — /8, /16, /24...

La notation CIDR remplace le masque décimal par un simple nombre : /préfixe

Le préfixe indique combien de bits sont à 1 dans le masque (= partie réseau)

// Formule magique

Nombre d'hôtes utilisables = 2^(bits hôte) - 2

Pourquoi -2 ?

  • -1 pour l'adresse réseau (tous bits hôte = 0)
  • -1 pour l'adresse broadcast (tous bits hôte = 1)

Exemples :

  • /24 → 8 bits hôte → 2^8 - 2 = 256 - 2 = 254 hôtes
  • /25 → 7 bits hôte → 2^7 - 2 = 128 - 2 = 126 hôtes
  • /26 → 6 bits hôte → 2^6 - 2 = 64 - 2 = 62 hôtes
  • /30 → 2 bits hôte → 2^2 - 2 = 4 - 2 = 2 hôtes (liaisons point-à-point)
💡
Mémoriser les puissances de 2 2^1=2, 2^2=4, 2^3=8, 2^4=16, 2^5=32, 2^6=64, 2^7=128, 2^8=256
Indispensable pour les calculs rapides à l'examen.
Méthode de calcul

Calculer réseau, broadcast et plage d'hôtes

Méthode simple (sans binaire) pour /24, /25, /26, /27, /28, /29, /30

// Étape 1 : Identifier le masque et la taille de bloc

Taille de bloc = 256 - (valeur du dernier octet du masque)

Exemple : /26 → masque = 255.255.255.192 → taille = 256 - 192 = 64

// Étape 2 : Trouver l'adresse réseau

Diviser le dernier octet de l'IP par la taille de bloc, puis multiplier par la taille de bloc.

Exemple : 192.168.1.150/26

  1. Masque /26 → taille de bloc = 64
  2. Dernier octet IP = 150
  3. 150 ÷ 64 = 2.34... → partie entière = 2
  4. 2 × 64 = 128
  5. Adresse réseau = 192.168.1.128

// Étape 3 : Trouver le broadcast

Adresse réseau + taille de bloc - 1

Suite de l'exemple :

  • Réseau = 192.168.1.128
  • Taille = 64
  • 128 + 64 - 1 = 191
  • Broadcast = 192.168.1.191

// Étape 4 : Plage d'hôtes utilisables

Entre réseau et broadcast (exclus)

Suite de l'exemple :

  • Premier hôte = réseau + 1 = 192.168.1.129
  • Dernier hôte = broadcast - 1 = 192.168.1.190
  • Plage utilisable : 192.168.1.129 - 192.168.1.190
  • Nombre : 62 hôtes (64 - 2)
🧮 Résumé 192.168.1.150/26
Réseau : 192.168.1.128
Masque : 255.255.255.192
Premier IP: 192.168.1.129
Dernier IP: 192.168.1.190
Broadcast : 192.168.1.191
Nb hôtes : 62
Exercices guidés

Exemples de calculs pas à pas

// Exemple 1 : 10.5.10.75/27

📝 Calcul détaillé
1. Taille de bloc : /27 → masque = 255.255.255.224 → taille = 256 - 224 = 32

2. Adresse réseau :
  75 ÷ 32 = 2.34 → partie entière = 2
  2 × 32 = 64
  Réseau = 10.5.10.64

3. Broadcast :
  64 + 32 - 1 = 95
  Broadcast = 10.5.10.95

4. Plage hôtes :
  Premier = 10.5.10.65
  Dernier = 10.5.10.94
  Nb hôtes = 30

// Exemple 2 : 172.16.50.200/28

📝 Calcul détaillé
1. Taille de bloc : /28 → masque = 255.255.255.240 → taille = 256 - 240 = 16

2. Adresse réseau :
  200 ÷ 16 = 12.5 → partie entière = 12
  12 × 16 = 192
  Réseau = 172.16.50.192

3. Broadcast :
  192 + 16 - 1 = 207
  Broadcast = 172.16.50.207

4. Plage hôtes :
  Premier = 172.16.50.193
  Dernier = 172.16.50.206
  Nb hôtes = 14

// Exemple 3 : 192.168.20.17/30 (liaison point-à-point)

📝 Calcul détaillé
1. Taille de bloc : /30 → masque = 255.255.255.252 → taille = 256 - 252 = 4

2. Adresse réseau :
  17 ÷ 4 = 4.25 → partie entière = 4
  4 × 4 = 16
  Réseau = 192.168.20.16

3. Broadcast :
  16 + 4 - 1 = 19
  Broadcast = 192.168.20.19

4. Plage hôtes :
  Premier = 192.168.20.17 (routeur 1)
  Dernier = 192.168.20.18 (routeur 2)
  Nb hôtes = 2 (parfait pour 2 routeurs)
🔗
/30 pour liaisons point-à-point Le /30 est le masque standard pour relier deux routeurs. 2 IPs utilisables exactement → pas de gaspillage. Très courant dans les WAN.

⚠️ Pièges d'examen

Ce qui trompe les candidats

❌ Piège #1 : Confondre nombre total vs hôtes utilisables
/24 = 256 adresses au total, mais seulement 254 utilisables (-2 pour réseau et broadcast).

Question piège : "Combien d'appareils dans un /26 ?" → Réponse : 62 (pas 64 !)

Toujours faire 2^bits - 2, jamais oublier le -2.
❌ Piège #2 : Oublier que réseau et broadcast ne sont pas utilisables
Dans 192.168.1.0/24 :
- 192.168.1.0 = réseau → ne PAS assigner ❌
- 192.168.1.255 = broadcast → ne PAS assigner ❌
- 192.168.1.1 à 192.168.1.254 = utilisables ✅
❌ Piège #3 : /30 a 4 adresses mais seulement 2 utilisables
Réseau 192.168.1.16/30 :
- 192.168.1.16 = réseau ❌
- 192.168.1.17 = routeur 1 ✅
- 192.168.1.18 = routeur 2 ✅
- 192.168.1.19 = broadcast ❌

Beaucoup pensent qu'on peut utiliser les 4 → FAUX.
❌ Piège #4 : Mauvais calcul de taille de bloc
Erreur fréquente : penser que /25 = taille 255.
FAUX. Formule : taille = 256 - (dernier octet du masque)

/25 → masque = 255.255.255.128 → taille = 256 - 128 = 128
❌ Piège #5 : Ne pas vérifier si l'IP appartient au réseau calculé
Question : "192.168.1.200/26 — quelle est l'adresse réseau ?"

Calcul : 200 ÷ 64 = 3.125 → 3 × 64 = 192
Réseau = 192.168.1.192

Piège : certains répondent .200 → FAUX, l'adresse réseau est TOUJOURS au début du bloc.

Ce que CompTIA peut te poser

Question type #1 : Calcul réseau
"Un hôte a l'IP 172.16.35.120/27. Quelle est son adresse réseau ?"

Question type #2 : Calcul broadcast
"Réseau 10.10.10.128/26. Quelle est l'adresse broadcast ?"

Calcul :
/26 → taille = 64
Broadcast = 128 + 64 - 1 = 191
Réponse : 10.10.10.191
Question type #3 : Nombre d'hôtes
"Combien d'hôtes utilisables dans un réseau /28 ?"

Calcul :
/28 → 4 bits hôte
2^4 - 2 = 16 - 2 = 14 hôtes
Question type #4 : Vérification IP dans réseau
"192.168.50.75/26 — cette IP est-elle dans le même réseau que 192.168.50.100 ?"

Calcul :
/26 → taille = 64
75 ÷ 64 = 1 → 1 × 64 = 64 → réseau = 192.168.50.64
100 ÷ 64 = 1 → 1 × 64 = 64 → réseau = 192.168.50.64
Réponse : Oui (même réseau 192.168.50.64/26)

Résumé

🔑 Ce que tu dois retenir absolument

→CIDR = notation /préfixe. /24 = 24 bits réseau, 8 bits hôte
→Hôtes utilisables = 2^(bits hôte) - 2 (toujours -2 pour réseau et broadcast)
→Taille de bloc = 256 - (dernier octet du masque)
→Réseau = (IP ÷ taille) × taille. Broadcast = réseau + taille - 1
→Masques à connaître : /24=254h, /25=126h, /26=62h, /27=30h, /28=14h, /29=6h, /30=2h
→/30 = standard pour liaisons point-à-point (2 routeurs)
→PIÈGE : toujours faire -2, jamais oublier que réseau et broadcast ne sont pas utilisables
→Mémoriser les puissances de 2 : 2, 4, 8, 16, 32, 64, 128, 256
