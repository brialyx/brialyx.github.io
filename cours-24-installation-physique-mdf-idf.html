Point central réseau

MDF — Main Distribution Frame

🏢
Analogie — cœur immeuble MDF = salle des machines immeuble (électricité, eau, ascenseurs). Tout converge ici. IDFs = armoires électriques étages. MDF = hub central connecte tout, IDFs = distribution locale.

MDF (Main Distribution Frame) = salle réseau centrale bâtiment. Point convergence TOUS câbles backbone, connexion WAN/Internet, équipements core.

// Localisation

  • Généralement rez-de-chaussée ou sous-sol (accès facile techniciens, ISP)
  • Centre géométrique bâtiment (minimise longueur câbles)
  • Salle dédiée sécurisée (accès contrôlé, badge)
  • Proximité entrée ISP (fibre, câble, ligne télécom)

// Équipements MDF

  • Routeur bordure : Connexion WAN/Internet
  • Firewall principal : Sécurité périmètre
  • Core switches : Backbone haute capacité (Layer 3)
  • Patch panels : Terminaisons câbles backbone vers IDFs
  • Serveurs critiques (optionnel) : DNS, DHCP, AD
  • UPS : Onduleur backup électrique
  • Demarc point : Démarcation ISP (où responsabilité ISP s'arrête)

// Rôle MDF

  • ✅ Connexion Internet/WAN : Entrée unique vers extérieur
  • ✅ Interconnexion IDFs : Tous IDFs étages connectent au MDF
  • ✅ Routage inter-VLANs : Core switches Layer 3
  • ✅ Services centralisés : DHCP, DNS, authentification
⚠️
MDF = single point of failure potentiel MDF tombe → TOUT réseau bâtiment tombe. Best practice : Redondance équipements critiques (dual core switches, dual UPS, dual ISP).
Distribution étage

IDF — Intermediate Distribution Frame

IDF (Intermediate Distribution Frame) = salle réseau secondaire par étage/zone. Distribue connectivité localement depuis MDF.

// Localisation

  • 1 IDF par étage (petit bâtiment) ou plusieurs IDFs par étage (grand bâtiment)
  • Centre zone desservie (limite distance câbles horizontaux ≤ 90m)
  • Placard/salle dédiée sécurisée
  • Proximité verticale (shaft) vers MDF

// Équipements IDF

  • Access switches : Connectivité endpoints (PCs, téléphones, imprimantes)
  • Patch panels : Terminaisons câbles horizontaux (bureaux)
  • PoE switches (optionnel) : Alimentation téléphones IP, WAPs
  • Petit UPS : Backup switches critiques

// Rôle IDF

  • ✅ Distribution locale : Connecte endpoints étage au réseau
  • ✅ Uplink vers MDF : Fibre/cuivre backbone
  • ✅ Agrégation trafic : Concentre trafic étage avant envoi MDF

// MDF vs IDF

Topologie

Architecture MDF/IDF — Backbone hiérarchique

Bâtiment 3 étages :

[MDF — Rez-de-chaussée]
- Routeur → Internet/WAN
- Firewall
- Core switch Layer 3
- Patch panel backbone

↕ Backbone fibre (vertical riser)

[IDF Étage 1]
- Access switches (24 ports × 2)
- Patch panel horizontal (48 ports)
→ Connecte 48 bureaux étage 1

↕ Backbone fibre

[IDF Étage 2]
- Access switches (24 ports × 2)
- Patch panel horizontal (48 ports)
→ Connecte 48 bureaux étage 2

↕ Backbone fibre

[IDF Étage 3]
- Access switches (24 ports × 2)
- Patch panel horizontal (48 ports)
→ Connecte 48 bureaux étage 3

// Backbone vertical (Riser)

Riser = conduit vertical transportant câbles entre étages.

  • Fibre optique : Standard backbone MDF↔IDFs (bande passante élevée, longue distance)
  • Cat6a/Cat7 (optionnel) : Si distances courtes (<100m)

// Câblage horizontal

Horizontal = câbles IDF → endpoints (bureaux, salles).

  • Cat6/Cat6a : Standard actuel (1-10 Gbps)
  • Limite 90m : IDF → prise murale (TIA-568 standard)
  • +10m patch cords : Total 100m max (90m permanent + 10m patchs)
Interconnexion

Patch Panels — Organisation câblage

Patch Panel = panneau passif ports RJ45 (ou fibre). Termine câbles permanents (murs) et permet connexion flexible vers switches via patch cords.

// Rôle

  • ✅ Organisation : Tous câbles terminés proprement
  • ✅ Flexibilité : Patch cords courts changent connexions facilement
  • ✅ Protection switches : Pas brancher/débrancher directement switches (usure ports)
  • ✅ Documentation : Étiquetage clair (port 12 = Bureau 305)

// Workflow typique

[Bureau 305 — Prise murale RJ45]
↑ Câble permanent dans mur (Cat6, 60m)
[IDF Étage 3 — Patch Panel Port 12]
↓ Patch cord court (1m, Cat6)
[Access Switch Port 12]

→ Câble permanent JAMAIS touché
→ Changements = juste changer patch cord

// Types patch panels

Cuivre (RJ45) : 24 ou 48 ports, Cat5e/Cat6/Cat6a

Fibre : LC, SC, MTP/MPO connectors (backbone)

✅
Best practice : Patch panels TOUJOURS JAMAIS câbles permanents directs dans switches. Toujours terminer sur patch panels → flexibilité, organisation, maintenance facile.
Racks

Armoires réseau — Types et organisation

// Rack standard 19" (unités U)

Unité rack (U) : 1U = 1.75 pouces (44.45mm) hauteur.

Hauteurs courantes : 42U (datacenter), 24U, 12U (IDFs)

// Types armoires

Open Frame Rack

  • 4 montants verticaux (pas parois)
  • ✅ Refroidissement excellent (air circule)
  • ✅ Accès facile équipements
  • ⚠️ Pas sécurité physique
  • Usage : Salles serveurs sécurisées (MDF)

Enclosed Rack (Cabinet)

  • Parois + porte verrouillable
  • ✅ Sécurité physique (lock)
  • ✅ Protection poussière
  • ⚠️ Refroidissement (besoin ventilation/climatisation)
  • Usage : IDFs, zones non-sécurisées

Wall-Mount Rack

  • Petit rack fixé mur (6U-12U)
  • ✅ Gain espace sol
  • Usage : Petits IDFs, placards

// Organisation rack (bas → haut)

Rack 42U MDF (exemple) :

[42U] ─ Vide (expansion future)
[40U] ─ Patch panel fibre backbone
[38U] ─ Core switch 1 (Layer 3)
[36U] ─ Core switch 2 (Layer 3 redundant)
[34U] ─ Vide (cable management)
[32U] ─ Firewall
[30U] ─ Routeur bordure
[28U] ─ Patch panel cuivre
...
[5U] ─ Cable management horizontal
[3U] ─ UPS (onduleur)
[1U] ─ PDU (power distribution unit)

→ Équipements lourds BAS (stabilité)
→ Cable management régulier (évite chaos)
Organisation

Gestion câbles et documentation

// Cable Management

Objectifs : Organisation, airflow, maintenance facile, apparence professionnelle.

Équipements :

  • Cable managers horizontaux : Panneaux avec anneaux/doigts (1U entre équipements)
  • Cable managers verticaux : Côtés rack (guident câbles haut↔bas)
  • Velcro straps : Attacher faisceaux câbles (PAS zip ties → coupent câbles)
  • Cable trays : Chemins câbles sous faux-plafond

// Étiquetage

CRITIQUE : Identifier chaque câble, port.

Convention étiquetage :

Patch panel port : IDF3-PP1-P12
→ IDF étage 3, Patch Panel 1, Port 12

Câble horizontal : IDF3-305
→ IDF 3 vers Bureau 305

Switch port : SW-IDF3-1-P12
→ Switch IDF 3 numéro 1, Port 12

// Documentation

Indispensable : Schémas réseau, inventaire câbles, plans étages.

Documents maintenir :

  • Network diagram : Topologie logique (VLANs, IPs)
  • Rack elevation diagrams : Vue frontale chaque rack (quoi dans chaque U)
  • Cable schedule : Spreadsheet (câble ID, source, destination, longueur, type)
  • Port mapping : Patch panel port 12 = Bureau 305 = Switch port 12
  • As-built drawings : Plans bâtiment annotés (chemins câbles, IDFs)
📝
Documentation = investissement temps payant Pas documentation = cauchemar troubleshooting. "Quel câble va où ?" → heures perdues. Documentation claire = minutes résoudre problème.
⚠️ Pièges d'examen

Ce qui trompe les candidats

❌ Piège #1 : MDF = serveurs uniquement
FAUX. MDF = hub réseau central (routeur, firewall, core switches).

Serveurs peuvent être MDF MAIS rôle principal = connectivité réseau backbone.
❌ Piège #2 : IDF connecte directement Internet
FAUX. IDF uplink vers MDF uniquement.

Connexion Internet/WAN = MDF.
IDF = distribution locale étage → MDF → Internet
❌ Piège #3 : Câbles permanents directs dans switches OK
MAUVAISE PRATIQUE. TOUJOURS terminer sur patch panels.

Patch panels → patch cords courts → switches.
Flexibilité, organisation, protection ports switches
Résumé

🔑 Ce que tu dois retenir absolument

→MDF : Hub central (1 par bâtiment), routeur/firewall/core switches, connexion WAN/Internet, tous IDFs connectent ici
→IDF : Distribution locale (par étage/zone), access switches, uplink MDF, connecte endpoints (PCs, téléphones)
→Patch panels : Terminent câbles permanents, patch cords courts → switches, organisation + flexibilité
→Backbone : Vertical riser (MDF↔IDFs, fibre), Horizontal (IDF→bureaux, Cat6, ≤90m)
→Documentation : Étiquetage (IDF3-PP1-P12), cable schedule, rack diagrams, as-built drawings CRITIQUES
