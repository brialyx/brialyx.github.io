Avant de commencer

C'est quoi une topologie réseau ?

Une topologie réseau décrit la manière dont les appareils (ordinateurs, switches, routeurs) sont connectés physiquement ou logiquement entre eux.

🗺️
Analogie — plan du métro vs itinéraire réel Topologie physique = le plan réel des câbles (comment ils sont tirés dans les murs, les conduits)
Topologie logique = comment les données circulent (le "plan du métro" — qui peut parler à qui)

Topologie physique : L'arrangement réel des câbles et équipements. Exemple : tous les câbles vont au switch central (étoile physique).

Topologie logique : Comment les données circulent. Exemple : tous les appareils reçoivent tous les messages (bus logique), même si physiquement c'est une étoile.

🎯
Pour l'examen Network+ CompTIA teste : reconnaître chaque topologie, savoir leurs avantages/inconvénients, identifier la topologie physique vs logique, et choisir la bonne topologie selon un scénario. C'est conceptuel — pas de calculs, juste de la compréhension.
Vocabulaire essentiel

Les mots clés de ce cours

Topologie physique
L'arrangement réel et physique des câbles, équipements et connexions.
Topologie logique
La manière dont les données circulent sur le réseau, indépendamment de la disposition physique.
Nœud (Node)
Tout appareil connecté au réseau (PC, switch, routeur, imprimante…)
Point de défaillance unique (Single Point of Failure - SPOF)
Un élément dont la panne entraîne l'arrêt de tout le réseau. À éviter pour la haute disponibilité.
Redondance
Avoir plusieurs chemins possibles entre deux points. Si un chemin tombe, un autre prend le relais.
Topologie 1

Topologie en Bus

📏 Bus (linéaire)
Tous les appareils sont connectés à un câble central unique (le "bus" ou "backbone"). Les données circulent sur le câble et tous les appareils les reçoivent, mais seul le destinataire traite le message.

// Caractéristiques

  • Un seul câble central (coaxial, généralement)
  • Les appareils se connectent via des "T-connectors"
  • Les extrémités du câble ont des terminateurs (50 ohms) pour éviter les réflexions de signal
  • Tous les appareils partagent le même support de transmission

// Avantages

  • Simple et peu coûteux
  • Peu de câblage nécessaire
  • Facile à étendre (ajouter un appareil = connecter au câble)

// Inconvénients

  • Point de défaillance unique : si le câble principal se casse → tout le réseau tombe
  • Difficile à dépanner (où est la coupure ?)
  • Performance dégradée avec beaucoup d'appareils (collisions)
  • Pas de sécurité : tous les appareils voient tout le trafic
⚠️
Statut aujourd'hui La topologie en bus est obsolète. Utilisée dans les années 1980-90 avec Ethernet 10Base2 et 10Base5 (coaxial). Remplacée par l'étoile avec switches. Tu ne la verras que dans l'examen, pas dans la vraie vie.
Topologie 2

Topologie en Étoile

⭐ Étoile (Star)
Tous les appareils sont connectés à un point central (switch ou hub). Toutes les communications passent par ce point central.

// Caractéristiques

  • Un switch (ou hub) au centre
  • Chaque appareil a son propre câble vers le centre
  • Topologie physique ET logique la plus courante aujourd'hui
  • Utilisée dans presque tous les réseaux LAN modernes

// Avantages

  • Facile à dépanner : si un câble tombe, seul cet appareil est affecté
  • Facile à étendre : ajouter un appareil = brancher un câble au switch
  • Performance : avec un switch, chaque appareil a sa bande passante dédiée
  • Centralisé : gestion et surveillance faciles

// Inconvénients

  • Point de défaillance unique : si le switch central tombe → tout le réseau tombe
  • Plus de câblage qu'en bus (un câble par appareil)
  • Coût du switch central
✅
Standard actuel Topologie en étoile = standard moderne. Presque tous les réseaux locaux (maison, bureau, entreprise) utilisent une étoile physique avec switch central.
Topologie 3

Topologie en Étoile étendue

🌟 Étoile étendue (Extended Star / Hierarchical Star)
Plusieurs étoiles connectées entre elles via leurs switches centraux. C'est une étoile d'étoiles — hiérarchique.

// Caractéristiques

  • Un switch central principal (core switch)
  • Plusieurs switches secondaires connectés au switch principal
  • Les appareils finaux se connectent aux switches secondaires
  • Structure à 3 niveaux typique : Core → Distribution → Access

// Avantages

  • Permet de couvrir de grandes distances (immeubles, campus)
  • Scalable (facile d'ajouter des switches et appareils)
  • Bonne performance
  • Isolation : un problème sur un switch secondaire n'affecte que cette branche

// Inconvénients

  • Si le switch central tombe → tout le réseau tombe
  • Plus complexe à configurer
  • Plus coûteux (plusieurs switches)
🎯
Usage typique Étoile étendue = standard pour les entreprises et campus. Exemple : switch central dans la salle serveur, switches par étage, PC/téléphones sur chaque étage.
Topologie 4

Topologie en Anneau

⭕ Anneau (Ring)
Les appareils sont connectés en boucle fermée. Chaque appareil est connecté à exactement deux voisins. Les données circulent dans une direction (ou les deux avec double anneau).

// Caractéristiques

  • Chaque appareil a deux connexions (entrée et sortie)
  • Les données circulent de nœud en nœud jusqu'au destinataire
  • Chaque nœud régénère le signal (pas d'atténuation)
  • Accès au réseau contrôlé par jeton (Token Ring) ou autre mécanisme

// Avantages

  • Pas de collisions (accès contrôlé par jeton)
  • Performance prévisible même avec beaucoup d'appareils
  • Pas d'atténuation (signal régénéré à chaque nœud)

// Inconvénients

  • Point de défaillance : si un nœud ou un câble tombe → tout l'anneau tombe (sauf double anneau)
  • Difficile à dépanner
  • Ajouter/retirer un appareil perturbe tout le réseau
  • Plus coûteux que l'étoile

// Variante : Double anneau (Dual Ring)

Deux anneaux en sens inverse. Si un anneau se coupe, le trafic passe par l'autre → redondance. Utilisé dans FDDI et SONET.

⚠️
Statut aujourd'hui Topologie en anneau rare aujourd'hui. Token Ring (IBM, années 80-90) est mort. Seule utilisation moderne : réseaux fibre optique SONET/SDH en télécommunications (opérateurs).
Topologie 5

Topologie Maillée

🕸️ Maillé (Mesh)
Chaque appareil est connecté à plusieurs (voire tous) les autres appareils. Plusieurs chemins possibles entre deux points.

// Deux types

Full Mesh (Maillage complet) : Chaque nœud est connecté à TOUS les autres nœuds.

  • Nombre de connexions = n × (n - 1) / 2
  • Exemple : 5 nœuds → 5 × 4 / 2 = 10 connexions
  • 10 nœuds → 45 connexions

Partial Mesh (Maillage partiel) : Certains nœuds ont plusieurs connexions, mais pas tous connectés à tous.

// Avantages

  • Redondance maximale : plusieurs chemins entre deux points
  • Haute disponibilité : si un lien tombe, le trafic passe par un autre chemin
  • Fiabilité : pas de point de défaillance unique
  • Performance : plusieurs chemins → répartition de charge

// Inconvénients

  • Très coûteux : beaucoup de câbles et de ports nécessaires
  • Complexe : configuration et maintenance difficiles
  • Overhead de routage (trouver le meilleur chemin)
🎯
Usage typique Full mesh : rare, seulement pour infrastructures critiques (centres de données, backbones Internet)
Partial mesh : plus courant, WAN d'entreprise, connexions entre datacenters, Wi-Fi mesh
Topologie 6

Topologie Point-à-Point

↔️ Point-à-Point (P2P)
Connexion directe entre exactement deux appareils. Pas d'intermédiaire.

// Caractéristiques

  • Deux appareils uniquement
  • Lien dédié
  • Simple, performant, sécurisé

// Exemples d'usage

  • Liaison entre deux routeurs (connexion WAN)
  • Connexion entre deux datacenters
  • Câble USB entre ordinateur et imprimante
  • Liaison série entre deux équipements réseau

// Avantages

  • Bande passante dédiée (pas de partage)
  • Faible latence
  • Sécurisé (seulement 2 appareils)
  • Simple à configurer

// Inconvénients

  • Pas scalable (1 lien = 2 appareils seulement)
  • Pour connecter 10 appareils → besoin de 45 liens (devient mesh)
Topologie 7

Topologie Hub-and-Spoke

🎯 Hub-and-Spoke (étoile WAN)
Une topologie en étoile utilisée pour les WAN. Un site central (hub) connecté à plusieurs sites distants (spokes). Tout le trafic entre spokes passe par le hub.

// Caractéristiques

  • Un site central (siège, datacenter principal)
  • Plusieurs sites distants (agences, bureaux régionaux)
  • Chaque site distant a une liaison WAN vers le hub
  • Pas de connexion directe entre spokes

// Avantages

  • Économique : moins de liaisons WAN coûteuses (n liaisons au lieu de n×(n-1)/2)
  • Gestion centralisée (firewall, serveurs, backup au hub)
  • Simple à déployer et maintenir

// Inconvénients

  • Point de défaillance unique : si le hub tombe → tous les spokes sont isolés
  • Latence : communication spoke-to-spoke passe par le hub (2× la distance)
  • Bande passante hub saturée si beaucoup de spokes

// Variante : Hub-and-Spoke avec backup

Certains déploiements ajoutent une connexion de secours (backup) entre spokes critiques → devient partial mesh.

🎯
Usage typique WAN d'entreprise avec siège + agences, VPN site-to-site (toutes les agences se connectent au siège via VPN), SD-WAN.

Résumé

🔑 Ce que tu dois retenir absolument

→Bus : un câble central, obsolète, point de défaillance unique
→Étoile : switch central, standard LAN actuel, facile à dépanner, SPOF au centre
→Étoile étendue : étoile hiérarchique (core → distribution → access), campus/entreprise
→Anneau : boucle fermée, Token Ring obsolète, dual ring pour redondance, SONET/SDH opérateurs
→Full Mesh : tous connectés à tous, redondance max, très coûteux (n×(n-1)/2 connexions)
→Partial Mesh : certains nœuds multiples connexions, bon compromis redondance/coût
→Point-à-Point : 2 appareils, lien dédié, WAN entre 2 sites
→Hub-and-Spoke : étoile WAN, siège + agences, économique, SPOF au hub
