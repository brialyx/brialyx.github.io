AAA Protocols

RADIUS vs TACACS+ — Différences clés examen

RAPPEL : Cours Module 3.0 - O13 couvrait détails. Ici = focus différences critiques examen.

// Tableau comparatif RADIUS vs TACACS+

CritèreRADIUSTACACS+
Vendor ✅ Standard (multi-vendor, RFC 2865) ⚠️ Cisco propriétaire
Protocol UDP 1812 (auth), 1813 (acct) TCP 49
AAA séparation ⚠️ Auth + Author combinés ✅ AAA complètement séparés
Chiffrement ⚠️ Password seul (MD5) ✅ Payload COMPLET chiffré
Command authorization ❌ Non supporté ✅ Per-command authorization
Usage principal Network access (802.1X, VPN, WiFi) Device admin (SSH switches/routers)
Adoption ✅ Universal (ISPs, enterprise WiFi) ⚠️ Cisco environments
🎯
Questions TRÈS fréquentes examen "Protocole AAA UDP ports 1812/1813 ?"
Réponse : RADIUS

"Protocole AAA TCP port 49, command authorization granulaire ?"
Réponse : TACACS+

"Protocole AAA chiffre payload complet ?"
Réponse : TACACS+

// Quand utiliser RADIUS vs TACACS+ ?

RADIUS :

  • Network access control (802.1X ports, VPN, WiFi WPA2-Enterprise)
  • Multi-vendor environment (Cisco + Juniper + Aruba)
  • Standard compliance requis

TACACS+ :

  • Device administration (SSH admin switches/routers)
  • Command authorization granulaire (contrôler QUELLES commandes user exécute)
  • Cisco-heavy environment (IOS, NX-OS)
  • Sécurité maximale (full payload encryption)
🔄
Production = souvent LES DEUX RADIUS : 802.1X port auth, VPN users, WiFi
TACACS+ : SSH admin switches/routers, privilege levels
Pas exclusif → complémentaires différents use cases.
Port-based NAC

802.1X — Port-based Network Access Control

RAPPEL : Cours Module 3.0 - O13 détails complets. Ici = focus workflow examen.

// 802.1X Architecture (3 composants)

  • Supplicant : Client demandant accès (PC, laptop, phone)
  • Authenticator : Switch/AP contrôlant port (proxy RADIUS)
  • Authentication Server : RADIUS server (backend AAA)

// Workflow 802.1X simplifié

1. Laptop branché port switch
   → Port état UNAUTHORIZED (bloqué, VLAN 1 guest only)

2. Switch envoie EAP-Request-Identity
   → Laptop répond username

3. Switch forward credentials → RADIUS server
   → RADIUS vérifie AD/LDAP

4. RADIUS répond :
   Access-Accept → Port AUTHORIZED (VLAN 10 production)
   Access-Reject → Port reste UNAUTHORIZED

5. Laptop accède réseau (production VLAN assignée dynamiquement)

// Dynamic VLAN Assignment

Avantage 802.1X : RADIUS attributes assignent VLAN basé user/device.

RADIUS attributes :
Tunnel-Type = VLAN
Tunnel-Private-Group-ID = 10

→ Switch place user VLAN 10 (data)

Exemples policies :
Employee → VLAN 10 (data, full access)
Contractor → VLAN 40 (guest, Internet only)
IoT device → VLAN 50 (isolated, restricted)
🔌
802.1X = empêche rogue devices Laptop non autorisé branché switch = bloqué (port unauthorized).
Defense contre : rogue APs, malware propagation, contractor devices.
Enterprise standard (PCI-DSS compliance souvent exige 802.1X).
Multi-factor authentication

MFA / 2FA — Authentification multi-facteurs

// 3 facteurs authentification

Something you KNOW

  • Password, PIN, passphrase
  • Security questions (mère maiden name)

Something you HAVE

  • Hardware token (RSA SecurID, YubiKey)
  • Smartphone (SMS code, authenticator app)
  • Smart card, badge

Something you ARE

  • Biométrie : Empreinte digitale, reconnaissance faciale, iris scan
  • Voiceprint, behavioral patterns (typing speed)

// 2FA vs MFA

2FA (Two-Factor Authentication)

  • Exactement 2 facteurs DIFFÉRENTS types
  • Exemple : Password (know) + SMS code (have)
  • ⚠️ Pas 2FA : Password + security question (BOTH = know)

MFA (Multi-Factor Authentication)

  • 2+ facteurs différents types
  • 2FA = sous-ensemble MFA (minimum 2)
  • Exemples : Password + fingerprint + smart card (3 facteurs)
✅ 2FA exemples VALIDES :
- Password + SMS code (know + have)
- Password + fingerprint (know + are)
- Smart card + PIN (have + know)

❌ PAS 2FA :
- Password + security question (BOTH know)
- Password + backup password (BOTH know)
- Fingerprint + face scan (BOTH are)

// Méthodes 2FA/MFA courantes

SMS codes

  • ✅ Simple, widespread adoption
  • ⚠️ Vulnérable SIM swapping, SMS interception

Authenticator apps (TOTP)

  • Google Authenticator, Microsoft Authenticator, Authy
  • Time-based One-Time Password (30-sec codes)
  • ✅ Plus sécurisé SMS (offline, pas network)

Hardware tokens (FIDO2, U2F)

  • YubiKey, Titan Security Key
  • ✅ Phishing-resistant (crypto challenge-response)
  • ✅ Standard moderne (Google, GitHub, banks)

Push notifications

  • Duo Mobile, Microsoft Authenticator push
  • ✅ UX excellent (tap Approve)
  • ⚠️ MFA fatigue attacks (users approve sans vérifier)
🔐
MFA = défense #1 contre credential theft 99%+ account compromises bloqués MFA (Microsoft stat).
Passwords seuls = insuffisants (phishing, leaks, brute-force).
MFA obligatoire : Admin accounts, VPN, cloud services, banking.
Network Access Control

NAC — Posture Assessment

RAPPEL : Cours Module 3.0 - O13 détails complets. Ici = focus concept examen.

NAC (Network Access Control) = Vérifier conformité device AVANT accès réseau (posture assessment + remediation).

// Posture Checks NAC

  • Antivirus : Installé ? Up-to-date ? Enabled ?
  • OS patches : Security updates current (Windows Update, macOS)
  • Firewall : Host firewall enabled
  • Encryption : Disk encryption (BitLocker, FileVault)
  • Device type : Corporate managed vs BYOD vs guest
  • Compliance : Company policies (password complexity, screen lock timeout)

// NAC Workflow (3 étapes)

1. Authentication (802.1X)
Device connects → 802.1X auth → Identity verified

2. Posture Assessment
NAC agent scans device :
- Antivirus : ✅ McAfee updated (today)
- OS patches : ❌ Windows 3 months outdated
- Firewall : ✅ Enabled
→ Verdict : NON-COMPLIANT

3. Policy Enforcement
Compliant → VLAN 10 (Production, full access)
Non-compliant → VLAN 99 (Quarantine, update servers only)
Guest/Unknown → VLAN 40 (Guest, Internet only)

Device redirected quarantine VLAN → Install patches → Re-scan
→ Compliant → Moved VLAN 10 production

// NAC modes

Agent-based NAC

  • Software agent installé device (deep inspection)
  • ✅ Granular checks (registry, processes, config files)
  • ⚠️ Requires deployment (corporate devices)

Agentless NAC

  • Network-based scanning (passive, active probes)
  • ✅ BYOD, guests, IoT devices (no agent install)
  • ⚠️ Limited visibility (external checks only)

// Solutions NAC

Cisco ISE (Identity Services Engine) : Leader marché, integration Cisco ecosystem.

Aruba ClearPass : Multi-vendor, policy-based.

ForeScout : Agentless, IoT/OT visibility.

🎯
Question fréquente examen "Solution vérifier antivirus/patches device AVANT accès réseau ?"
Réponse : NAC (Network Access Control)

"VLAN isolant devices non-compliant jusqu'à remediation ?"
Réponse : Quarantine VLAN

INCOMPLET. 802.1X = framework (pas protocol complet).

Facteurs DOIVENT être types différents (know/have/are)
Résumé

🔑 Ce que tu dois retenir absolument

→RADIUS vs TACACS+ : RADIUS = UDP 1812/1813, standard, network access (802.1X/VPN). TACACS+ = TCP 49, Cisco, device admin SSH, command authorization, full payload encryption
→802.1X : Port-based NAC. 3 composants (Supplicant, Authenticator, Auth Server RADIUS). Port UNAUTHORIZED → Auth → AUTHORIZED. Dynamic VLAN assignment
→2FA vs MFA : 2FA = exactement 2 facteurs types différents. MFA = 2+ facteurs. 3 types : Know (password), Have (token/phone), Are (biométrie). Password + security question ≠ 2FA (both know)
→NAC : Posture assessment (antivirus, patches, firewall ?). Compliant → Production VLAN, Non-compliant → Quarantine VLAN remediation, Guest → Internet only. Agent-based vs Agentless
→Solutions : RADIUS (FreeRADIUS, Microsoft NPS), TACACS+ (Cisco ISE, ACS), NAC (Cisco ISE, Aruba ClearPass, ForeScout). MFA (YubiKey, Duo, Google Authenticator)
