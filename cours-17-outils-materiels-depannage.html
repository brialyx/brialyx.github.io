Basic testing

Cable Tester — Test continuité câble

Cable tester = Device test câbles Ethernet continuité, pinout, faults (opens, shorts, split pairs).

// Fonctions cable tester

  • Continuité : Chaque fil connecté correctement ?
  • Pinout verification : T568A vs T568B, correct wiring ?
  • Open circuit : Fil cassé (broken wire)
  • Short circuit : Deux fils touchent (shorted together)
  • Split pairs : Twisted pairs incorrectement arrangés (performance loss)
  • Cable length : Approximation longueur (basic testers)

// Usage typique

Test procedure :

1. Connect cable ends → Tester main unit + remote unit
2. Power ON → Press TEST button
3. Tester sends signal each wire (1-8)
4. Remote unit receives signals
5. LED indicators show results :
   ✅ Green LED = Wire OK
   ❌ Red LED / No LED = Wire fault

Example output :
Pin 1 → Green (OK)
Pin 2 → Green (OK)
Pin 3 → OFF (Open circuit - broken wire)
Pin 4 → Green (OK)
...

→ Pin 3 broken = Cable defective (replace OR re-terminate)

// Types cable testers

TypeFeaturesCostUsage
Basic Continuity, pinout $20-$50 Home, small office
Advanced + Length, wire map $100-$300 Professional install
Certifier + Performance specs $2,000-$8,000+ Certified install, datacenter

// Limitations

  • Tests connectivity ONLY (not performance/speed)
  • Cannot certify cable Cat6/Cat6a standards (needs certifier)
  • Length measurement approximation (not precise)
💡
Cable tester = first tool troubleshooting Quick test : Cable good or bad (5 seconds).
Identifies : Opens, shorts, miswires (80% cable problems).
Inexpensive : Every network tech should have basic tester.
Cable tracing

Toner & Probe — Localiser câble

Toner (tone generator) + Probe (inductive amplifier) = Locate specific cable dans bundle/patch panel.

// Principe fonctionnement

Toner (Generator) :
→ Connect to cable one end
→ Sends audio tone signal (audible frequency ~1 kHz)
→ Signal travels cable length

Probe (Detector) :
→ Inductive pickup (no contact needed)
→ Touch/hover cables → Detects tone
→ Speaker/LED indicates signal strength
→ Loudest tone = correct cable ✅

// Usage scenarios

1. Locate cable in patch panel

Problem : 100 cables patch panel, which one goes Office 205 ?

Solution :
1. Connect toner → Wall jack Office 205
2. Go patch panel → Use probe each cable
3. Probe detects tone → Correct cable identified ✅
4. Label cable (Office 205)

2. Trace cable in ceiling/walls

Problem : Cable runs ceiling, where exactly ?

Solution :
1. Connect toner → Cable end
2. Walk ceiling/wall with probe
3. Follow tone signal → Trace cable path
4. Identify exact location (install AP, troubleshoot break)

// Best practices

  • Disconnect cables : Unplug from switch (avoid tone spreading network)
  • Adjust sensitivity : Probe volume control (reduce false positives)
  • Test multiple points : Confirm correct cable (not adjacent interference)
  • Label immediately : Once identified, label both ends
🔍
Toner & Probe = cable identification essential Unlabeled patch panels = nightmare troubleshooting.
Toner finds cable quickly (vs trial-and-error hours).
Cost : $30-$100 (pays for itself first use).
Performance testing

Cable Certifier (Fluke) — Certification professionnelle

Cable certifier = Advanced tester certifies cable meets standards (Cat5e, Cat6, Cat6a). Mesure performance parameters.

// Tests performed

  • Wire map : Pinout correct (like basic tester)
  • Length : Precise measurement (TDR - Time Domain Reflectometry)
  • Attenuation : Signal loss over distance (dB)
  • NEXT (Near-End Crosstalk) : Interference between pairs
  • Return loss : Impedance mismatch reflections
  • Propagation delay : Signal travel time
  • Delay skew : Timing difference between pairs

// Certification standards

CategoryBandwidthMax LengthUsage
Cat5e100 MHz100m1 Gbps Ethernet
Cat6250 MHz100m (55m 10G)1/10 Gbps
Cat6a500 MHz100m10 Gbps full distance
Cat7600 MHz100m10 Gbps shielded

// Usage certification

Certification workflow :

1. Select standard (Cat6, Cat6a, etc.)
2. Connect certifier → Cable both ends
3. Run automated test suite (~30-60 seconds)
4. Certifier displays : PASS or FAIL
5. Print certification report (compliance documentation)

PASS :
✅ All parameters meet/exceed standard
→ Cable certified Cat6 (warranty compliant)

FAIL :
❌ One or more parameters outside spec
→ Shows which test failed (NEXT fail, attenuation high)
→ Re-terminate or replace cable

// Popular certifiers

  • Fluke DSX-5000 : Industry standard ($5,000-$8,000)
  • Fluke Versiv : Modular platform (copper + fiber)
  • Ideal LanTek : Budget option ($2,000-$4,000)

// When certification required ?

  • New installations : Structured cabling projects (building, datacenter)
  • Warranty compliance : Cable manufacturer warranty requires certification
  • High-speed networks : 10 Gbps (Cat6a required, must certify)
  • Troubleshooting : Intermittent errors (certify cable meets spec)
💰
Certifier = expensive professional tool Cost : $2,000-$8,000+ (vs basic tester $50).
Required : Professional installers, datacenters (warranty).
Not needed : Home, small office (basic tester sufficient).
Fiber optic testing

OTDR — Optical Time Domain Reflectometer

OTDR = Advanced fiber testing. Locates breaks, splices, connectors. Measures loss per segment (voir Cours TR06 rappel).

// Principe OTDR (recap)

1. OTDR sends light pulse into fiber
2. Light reflects back from imperfections
3. OTDR measures time reflection → Calculate distance
4. Display trace graph :
   X-axis : Distance (meters/km)
   Y-axis : Signal loss (dB)

Trace shows :
→ Gradual slope = fiber attenuation (normal loss distance)
→ Small drop = connector (0.3-0.5 dB)
→ Larger drop = splice (0.1-0.3 dB)
→ Sudden drop end = fiber break (repair needed)

// Usage OTDR

  • Locate fiber breaks : Distance exact (repair crews)
  • Measure total loss : Verify optical budget compliance
  • Identify bad splice/connector : Excessive loss specific point
  • Certification new fiber : Document baseline performance
  • Troubleshoot intermittent : Detect microbends, stress points

// OTDR vs Light Meter

ToolMeasuresInformationCost
Light Meter Total loss end-to-end Total dB loss (no distance) $200-$500
OTDR Loss per segment Distance + loss each event $5,000-$20,000+

// When use OTDR ?

  • Long haul fiber : Telco, ISP (km fiber runs)
  • Complex installations : Multiple splices, connectors
  • Troubleshooting breaks : Locate exact distance fault

When light meter sufficient ?

  • Short distance datacenter (<500m)
  • Simple point-to-point links (no splices)
  • Total loss measurement only (pass/fail optical budget)
Interface testing

Loopback Adapter — Test interface NIC/port

Loopback adapter = Plug connects TX → RX (transmit loops back receive). Tests interface hardware isolated.

// Types loopback

1. Ethernet loopback (RJ-45)

Loopback wiring (RJ-45 plug) :
Pin 1 (TX+) → Pin 3 (RX+)
Pin 2 (TX-) → Pin 6 (RX-)

→ Data transmitted immediately received same port

2. Fiber loopback (LC/SC)

Fiber loopback plug :
TX fiber → Looped back → RX fiber

→ Light transmitted loops back receiver

// Usage loopback

Test procedure :

1. Insert loopback adapter → NIC port
2. Run diagnostic software :
   Windows : Device Manager → NIC properties → Diagnostics
   Linux : ethtool -t eth0
3. Software sends test packets → Loopback
4. Verify packets received correctly

PASS : NIC hardware OK ✅
FAIL : NIC defective (replace card) ❌

// Troubleshooting scenarios

Problem : No link on port

Question : Is problem NIC or cable ?

Test :
1. Loopback adapter → NIC
2. Run diagnostics

If PASS → NIC OK, problem = cable/switch port
If FAIL → NIC defective, replace

Problem : Intermittent errors

Test : Loopback isolates NIC hardware
→ Errors in loopback = NIC fault
→ No errors loopback = External problem (cable, switch, EMI)
🔧
Loopback = isolation testing Removes variables : Cable, switch, other devices.
Tests ONLY : NIC hardware transmit/receive.
DIY : Can make loopback plug (wire RJ-45 connector pins 1→3, 2→6).
Electrical testing

Multimètre — Voltage, continuity, resistance

Multimètre = Electrical measurement tool. Tests voltage, current, resistance, continuity.

// Network usage multimeter

1. PoE voltage test

Problem : IP phone/AP not powering on

Test PoE :
1. Set multimeter → DC voltage
2. Probe switch PoE port pins :
   Pins 1/2 (+) and 3/6 (-) — Mode A
   Pins 4/5 (+) and 7/8 (-) — Mode B
3. Measure voltage :
   PoE (802.3af) : 44-57V DC
   PoE+ (802.3at) : 50-57V DC
   PoE++ (802.3bt) : 50-57V DC

If voltage present → PoE works, device problem
If no voltage → PoE not enabled or switch fault

2. Cable continuity test

Test broken wire :
1. Set multimeter → Continuity mode (beep icon)
2. Probe both ends same wire
3. Beep sound → Wire continuous ✅
4. No beep → Wire broken ❌

3. Ground/earth test

Verify equipment properly grounded :
1. Measure voltage chassis → ground
2. Should be 0V (or <1V)
3. High voltage → Grounding fault (electrical hazard)

// Safety warnings

  • AC mains voltage : 120/240V dangerous (electrocution risk)
  • PoE safe : 48-57V DC (low current, safe touch)
  • Never probe live AC : Unless trained electrician
  • Use correct setting : AC vs DC, voltage range
Wireless troubleshooting

Wi-Fi Analyzer — Signal strength, channels, interference

Wi-Fi Analyzer = Software/app visualize Wi-Fi networks. Shows signal strength (RSSI), channel usage, interference (voir Cours TR14 rappel).

// Features Wi-Fi Analyzer

  • Network discovery : List all visible SSIDs
  • Signal strength : RSSI (-30 to -90 dBm)
  • Channel graph : Visualize channel congestion
  • Best channel recommendation : Least crowded channel
  • Speed test : Measure throughput
  • Connection info : Frequency (2.4/5 GHz), security type, MAC address

// Usage troubleshooting

1. Measure signal strength

User reports slow Wi-Fi

Wi-Fi Analyzer shows :
SSID : "CorpWiFi"
RSSI : -75 dBm (weak signal)

→ Signal weak (distance/obstacles)
→ Solution : Move closer AP, add AP, increase power

2. Identify channel congestion

Channel graph shows :
Channel 1 : 3 APs (-60, -65, -70 dBm)
Channel 6 : 8 APs (-50 to -80 dBm) — CONGESTED
Channel 11 : 2 APs (-75, -80 dBm) — Less congested

→ Recommendation : Change AP to channel 11
→ Reduce interference, improve performance

3. Detect rogue APs

Network scan shows :
SSID : "CorpWiFi"
BSSID : aa:bb:cc:dd:ee:ff (authorized AP)

SSID : "CorpWiFi" (duplicate)
BSSID : 11:22:33:44:55:66 (unauthorized - rogue!)

→ Evil Twin OR accidental duplicate
→ Investigate, locate, remove rogue AP

Wi-Fi Analyzer = smartphone essential Free apps available (Android: WiFi Analyzer, iOS: WiFiman).
Instant visibility : Signal, channels, interference.
Every network tech : Install on phone (troubleshooting tool).
