Packet filtering

ACL — Access Control List

🚪
Analogie — videur boîte de nuit ACL = videur avec liste invités. Règles : "Ami du patron → enter", "Moins 18 ans → deny", "Pas dress code → deny". Chaque personne (packet) vérifiée contre liste séquentiellement. Premier match = décision finale.

ACL (Access Control List) = Liste séquentielle règles filtrant trafic réseau basé critères (IP source/destination, protocole, port). Appliquée interfaces routers/firewalls.

// Principe ACL

  • Chaque paquet examine contre ACL règles séquentiellement (top → bottom)
  • Premier match = action appliquée (permit/deny)
  • Si aucun match = implicit deny (deny all à fin liste)
  • Pas processing après match (stop immediately)

// Objectifs ACL

  • Security : Bloquer trafic malveillant (source IP untrusted, ports dangereux)
  • Traffic control : Filtrer protocols (block P2P, FTP, Telnet)
  • QoS : Classifier trafic (priority voice, limit video)
  • NAT : Définir trafic NATé
Exemple ACL simple :

access-list 10 permit 192.168.1.0 0.0.0.255
access-list 10 deny 192.168.2.50 0.0.0.0
access-list 10 permit any

→ Règle 1 : Permit subnet 192.168.1.0/24
→ Règle 2 : Deny host 192.168.2.50
→ Règle 3 : Permit tout le reste
→ Implicit deny (jamais atteint car règle 3 = any)
⚠️
ACL = ordre CRITIQUE Règles évaluées séquentiellement top → bottom.
Ordre incorrect = bypass security (règle "permit any" avant deny = inutile).
Best practice : Spécifique → Générale (deny hosts → deny subnets → permit any).
IP source filtering

ACL Standard — Filtrage source IP uniquement

Standard ACL = Filtre trafic basé source IP UNIQUEMENT. Simple, limité. Numéros 1-99, 1300-1999.

// Caractéristiques Standard ACL

  • Critère : Source IP address uniquement
  • Pas : Destination IP, protocol, port
  • Numéros : 1-99 (legacy), 1300-1999 (expanded)
  • Placement : Proche destination (éviter bloquer trafic légitime early)

// Syntaxe Standard ACL

access-list NUMBER ACTION SOURCE WILDCARD

NUMBER : 1-99 ou 1300-1999
ACTION : permit | deny
SOURCE : Source IP address
WILDCARD : Wildcard mask (inverse subnet mask)

// Exemples Standard ACL

Bloquer host spécifique :
access-list 10 deny 192.168.1.100 0.0.0.0
access-list 10 permit any

→ Deny host 192.168.1.100
→ Permit tous autres

Permettre subnet uniquement :
access-list 20 permit 10.1.1.0 0.0.0.255
(implicit deny all à fin)

→ Permit 10.1.1.0/24
→ Deny tout le reste (implicit)

Application interface :
interface GigabitEthernet0/0
  ip access-group 10 in

→ Apply ACL 10 trafic ENTRANT (inbound) interface

// Limitation Standard ACL

Problème : Bloque TOUT trafic source IP (pas granulaire protocol/port).

Scénario problème :

Besoin : Bloquer Telnet (port 23) depuis 192.168.1.100
MAIS : Permettre HTTP (port 80) depuis 192.168.1.100

Standard ACL :
access-list 10 deny 192.168.1.100 0.0.0.0
→ Bloque TOUT trafic 192.168.1.100 (Telnet + HTTP + ICMP + ...)

→ Pas granulaire → Besoin Extended ACL
Advanced filtering

ACL Extended — Filtrage granulaire

Extended ACL = Filtre trafic basé source + destination IP, protocol, port. Granularité maximale. Numéros 100-199, 2000-2699.

// Caractéristiques Extended ACL

  • Critères : Source IP, Destination IP, Protocol (TCP/UDP/ICMP), Port source/destination
  • Numéros : 100-199 (legacy), 2000-2699 (expanded)
  • Placement : Proche source (bloquer trafic early, économise bandwidth)

// Syntaxe Extended ACL

access-list NUMBER ACTION PROTOCOL SOURCE DEST [PORT]

NUMBER : 100-199 ou 2000-2699
ACTION : permit | deny
PROTOCOL : ip | tcp | udp | icmp | ...
SOURCE : Source IP + wildcard
DEST : Destination IP + wildcard
PORT : eq (equal), gt (greater than), lt (less than), range

// Exemples Extended ACL

Bloquer Telnet (port 23) subnet spécifique :
access-list 100 deny tcp 192.168.1.0 0.0.0.255 any eq 23
access-list 100 permit ip any any

→ Deny TCP port 23 (Telnet) source 192.168.1.0/24 → any dest
→ Permit tout le reste

Permettre HTTP/HTTPS uniquement vers web server :
access-list 110 permit tcp any host 203.0.113.10 eq 80
access-list 110 permit tcp any host 203.0.113.10 eq 443
access-list 110 deny ip any any

→ Permit HTTP (80) → 203.0.113.10
→ Permit HTTPS (443) → 203.0.113.10
→ Deny tout le reste (explicit deny)

Bloquer ICMP ping :
access-list 120 deny icmp any any echo
access-list 120 permit ip any any

→ Deny ICMP echo (ping request)
→ Permit tout le reste
🎯
Question TRÈS fréquente examen "ACL filtrant source IP uniquement ?"
Réponse : Standard ACL (1-99)

"ACL filtrant source, destination, protocol, port ?"
Réponse : Extended ACL (100-199)

"Placement Standard ACL ?"
Réponse : Proche DESTINATION

"Placement Extended ACL ?"
Réponse : Proche SOURCE
Masking

Wildcard Masks — Inverse subnet mask

Wildcard mask = Inverse subnet mask. Indique quels bits IP adresse vérifier (0 = check, 1 = ignore).

// Principe wildcard mask

Subnet mask : 1 = network, 0 = host
Wildcard mask : 0 = must match, 1 = don't care (any)

// Calcul wildcard mask

Formule : Wildcard = 255.255.255.255 - Subnet Mask

Exemples calcul :

Subnet mask /24 (255.255.255.0) :
255.255.255.255 - 255.255.255.0 = 0.0.0.255
→ Match 3 premiers octets, ignore dernier

Subnet mask /16 (255.255.0.0) :
255.255.255.255 - 255.255.0.0 = 0.0.255.255
→ Match 2 premiers octets, ignore 2 derniers

Subnet mask /32 (255.255.255.255) :
255.255.255.255 - 255.255.255.255 = 0.0.0.0
→ Match TOUS bits (host exact)

// Exemples wildcard ACL

Host exact (192.168.1.100) :
access-list 10 deny 192.168.1.100 0.0.0.0
→ Wildcard 0.0.0.0 = match ALL bits (exact host)
→ Alias : host 192.168.1.100

Subnet /24 (192.168.1.0/24) :
access-list 10 permit 192.168.1.0 0.0.0.255
→ Match 192.168.1.x (x = 0-255 any)

Subnet /16 (10.1.0.0/16) :
access-list 10 permit 10.1.0.0 0.0.255.255
→ Match 10.1.x.x

Tout trafic (any) :
access-list 10 permit 0.0.0.0 255.255.255.255
→ Ignore ALL bits (match everything)
→ Alias : any

// Wildcard avancé (non-contiguous)

Exemple : Matcher hosts pairs uniquement (192.168.1.0, .2, .4, .6, ...)

192.168.1.0    0.0.0.254

Binary last octet wildcard : 11111110
→ Bit 0 (LSB) = 0 (must match → even)
→ Bits 1-7 = 1 (don't care)

→ Matches : .0, .2, .4, .6, .8, ..., .254 (pairs uniquement)
💡
Raccourcis wildcard host X.X.X.X = X.X.X.X 0.0.0.0 (exact host)
any = 0.0.0.0 255.255.255.255 (tout trafic)

Exemples :
access-list 10 permit host 192.168.1.1
access-list 10 deny any
Rule processing

Ordre des règles — First match wins

Principe fondamental ACL : Règles évaluées séquentiellement top → bottom. Premier match = action appliquée (stop processing).

// Exemple ordre CORRECT

✅ Ordre spécifique → générale :

access-list 100 deny tcp host 192.168.1.100 any eq 23
access-list 100 deny tcp 192.168.1.0 0.0.0.255 any eq 23
access-list 100 permit ip any any

Paquet : 192.168.1.100 → 10.1.1.1 Telnet (23)
→ Règle 1 match : DENY (host 192.168.1.100 Telnet)
→ Stop processing

Paquet : 192.168.1.50 → 10.1.1.1 Telnet (23)
→ Règle 1 no match
→ Règle 2 match : DENY (subnet 192.168.1.0/24 Telnet)
→ Stop processing

Paquet : 192.168.2.10 → 10.1.1.1 HTTP (80)
→ Règle 1 no match
→ Règle 2 no match
→ Règle 3 match : PERMIT (any any)
→ Allow traffic

// Exemple ordre INCORRECT

❌ Ordre générale → spécifique (ERREUR) :

access-list 100 permit ip any any
access-list 100 deny tcp host 192.168.1.100 any eq 23

Paquet : 192.168.1.100 → 10.1.1.1 Telnet (23)
→ Règle 1 match : PERMIT (any any → matches everything)
→ Stop processing
→ Règle 2 JAMAIS évaluée (unreachable code)

→ Security bypassed ! Telnet autorisé malgré deny rule.

// Best practices ordre

  1. Spécifique avant générale : Hosts → Subnets → Any
  2. Deny avant permit : Block exceptions → Allow general
  3. Fréquent en premier : Rules matchant souvent top (performance)
  4. Explicit permit any : Éviter implicit deny si besoin audit logs
Default action

Implicit Deny — Deny all final

Implicit Deny = Règle invisible fin TOUTE ACL : deny ip any any. Bloque tout trafic pas explicitly permitted.

// Principe implicit deny

ACL visible :
access-list 10 permit 192.168.1.0 0.0.0.255

ACL réelle (implicit deny ajouté automatiquement) :
access-list 10 permit 192.168.1.0 0.0.0.255
access-list 10 deny ip any any (IMPLICIT)

→ Tout trafic PAS 192.168.1.0/24 = DENIED

// Conséquences implicit deny

ACL vide = deny everything (implicit deny seule règle).

ERREUR commune :

access-list 100 deny tcp any any eq 23
(pas permit rule)

→ Règle 1 : Deny Telnet
→ Implicit deny : Deny TOUT le reste

→ Résultat : TOUT trafic bloqué (Telnet + HTTP + ICMP + ...)
→ Besoin : access-list 100 permit ip any any à fin

// Explicit vs Implicit deny

Explicit deny : Règle deny visible (logging possible).

Implicit deny : Invisible (pas logs par défaut).

Best practice : Ajouter explicit deny final si besoin logs.

access-list 100 permit tcp any any eq 80
access-list 100 permit tcp any any eq 443
access-list 100 deny ip any any log

→ Permit HTTP/HTTPS
→ Explicit deny all + logging
→ Logs montrent quel trafic bloqué (audit/troubleshooting)
🔴
Implicit deny = cause #1 ACL bugs Oubli permit statement → Tout trafic bloqué.
Symptôme : "ACL applied, réseau down completely".
Fix : Toujours terminer ACL permit ip any any (sauf si deny all voulu).
⚠️ Pièges d'examen

Ce qui trompe les candidats

❌ Piège #1 : Wildcard mask = subnet mask
FAUX. Wildcard mask = INVERSE subnet mask.

Subnet /24 (255.255.255.0) → Wildcard 0.0.0.255
Calcul : 255.255.255.255 - subnet mask

0 = must match, 1 = don't care (opposite subnet mask logic)
❌ Piège #2 : Extended ACL placement proche destination
FAUX. Extended ACL = proche SOURCE.

Standard ACL : Proche destination (évite bloquer trafic légitime early)
Extended ACL : Proche source (block early, économise bandwidth)

Mémo : Extended = granulaire → peut bloquer specifiquement → safe block early
❌ Piège #3 : Ordre ACL pas important si rules différentes
FAUX. Ordre = TOUJOURS critique.

First match wins → Processing stops.
Permit any AVANT deny specific → deny JAMAIS évaluée.

TOUJOURS : Spécifique → Générale (hosts → subnets → any)
Résumé

🔑 Ce que tu dois retenir absolument

→ACL : Liste séquentielle filtrant trafic. Premier match = action (permit/deny). Implicit deny final. Standard (1-99) source IP only. Extended (100-199) src/dst/proto/port
→Standard ACL : Source IP uniquement. Numéros 1-99. Placement proche DESTINATION. Limité (block all traffic from IP, no granularity)
→Extended ACL : Source, destination, protocol, port. Numéros 100-199. Placement proche SOURCE. Granulaire (block specific protocol/port)
→Wildcard masks : Inverse subnet mask. 0 = must match, 1 = don't care. Calcul : 255.255.255.255 - subnet. Host exact = 0.0.0.0. Any = 255.255.255.255
→Ordre règles : Spécifique → Générale. First match wins. Permit any AVANT deny = deny jamais hit. Deny AVANT permit exceptions
→Implicit deny : Deny ip any any final invisible. ACL vide = deny all. Best practice : explicit permit any final (sauf deny all voulu) + log deny rule
