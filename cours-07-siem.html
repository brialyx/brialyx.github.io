SIEM — Security Information & Event Management

🕵️
Analogie — détective analysant indices SIEM = détective enquêtant crime. Syslog = témoins individuels (chacun voit 1 indice). SIEM interroge TOUS témoins (firewall, switch, serveurs), corrèle indices → reconstruit attaque complète.

SIEM (Security Information & Event Management) = Plateforme centralisée collectant logs multiples sources, corrélant événements, détectant menaces, générant alertes temps réel.

// SIEM = SIM + SEM

SIM (Security Information Management)

  • Collecte/stockage long-terme logs
  • Compliance reporting
  • Forensics historique

SEM (Security Event Management)

  • Analyse temps réel
  • Corrélation événements
  • Alerting immédiat

SIEM = SIM + SEM = Stockage long-terme + Analyse temps réel

Event correlation

Corrélation d'événements — Détecter patterns

Corrélation = Relier événements multiples sources → identifier patterns suspects invisibles isolément.

// Exemple corrélation : Détection brute-force

Logs individuels (syslog) :

14:05:10 Firewall: Login failed user 'admin' from 185.x.x.x
14:05:12 Firewall: Login failed user 'admin' from 185.x.x.x
14:05:15 Firewall: Login failed user 'admin' from 185.x.x.x
14:05:18 Firewall: Login failed user 'root' from 185.x.x.x
14:05:20 Firewall: Login failed user 'root' from 185.x.x.x
...
(100 tentatives 5 minutes)

SIEM corrélation :

→ Pattern détecté : 100+ login failures même IP 5 min
→ Règle : Brute-force attack detected
→ Action : Alerte SOC + Bloquer IP 185.x.x.x automatiquement

→ Syslog seul = juste logs brut (humain manque pattern)
→ SIEM = détecte pattern automatiquement

// Exemple corrélation : Attaque multi-étapes

Scénario attaque APT (Advanced Persistent Threat) :

10:00 AM — Firewall logs :
Connexion VPN user 'jdoe' depuis Russie (anormal, user USA normalement)

10:05 AM — AD logs :
User 'jdoe' accès serveur fichiers sensibles (Finance folder, jamais accédé avant)

10:10 AM — Proxy logs :
Upload 500 MB vers Dropbox IP suspect

SIEM corrélation :

→ Pattern : VPN anormal + Accès fichiers sensibles + Upload massif externe
→ Règle : Compromised account + Data exfiltration
→ Alerte : CRITIQUE — Incident response team notifié immédiatement

Corrélation impossible sans SIEM : Logs isolés (firewall, AD, proxy) différents admins → personne voit pattern global. SIEM corrèle automatiquement.

// Types corrélation

Temporal (temporelle) : Événements liés temps (ex: 100 login failures 5 minutes).

Spatial (sources multiples) : Même IP attaque firewall + IDS + serveur web simultanément.

Behavioral (comportemental) : Déviation baseline (user accède serveur jamais utilisé avant).

Threat intelligence : IP connue malveillante (blacklist) + connexion détectée → alerte.

Architecture

Workflow SIEM

  1. Collecte : SIEM ingère logs multiples sources (syslog, NetFlow, SNMP traps, APIs)
  2. Normalisation : Parse/transforme logs formats différents → schéma unifié
  3. Enrichissement : Ajoute contexte (GeoIP, threat intelligence, asset info)
  4. Corrélation : Moteur règles analyse patterns multi-événements
  5. Alerting : Génère alertes incidents critiques (email, SMS, ticketing)
  6. Dashboards : Visualisation temps réel (SOC analysts)
  7. Forensics : Stockage long-terme investigation incidents passés
Architecture SIEM typique :

[Sources logs]
- Firewalls (100+ devices)
- Switches/Routers (500+ devices)
- Serveurs Windows/Linux (1000+ servers)
- IDS/IPS
- Cloud APIs (AWS CloudTrail, Azure AD)
↓ Syslog/APIs
[SIEM Platform — Splunk/QRadar]
- Collecteurs (agents/syslog receivers)
- Normalisation/Parsing
- Corrélation engine (règles + ML)
- Stockage (TB-PB logs)
↓
[SOC Analysts]
- Dashboards temps réel
- Alertes critiques
- Investigation forensics
Applications

Cas d'usage SIEM

// 1. Threat Detection (Détection menaces)

  • Brute-force attacks : Login failures répétés
  • DDoS : Spike connexions simultanées
  • Malware C2 : Beacon régulier vers IP suspect
  • Data exfiltration : Upload massif anormal
  • Privilege escalation : User standard devient admin soudainement
  • Lateral movement : Compromission PC → accès multiples serveurs rapide

// 2. Compliance Reporting

Réglementations exigent preuves contrôles sécurité.

  • PCI-DSS : Logs accès cardholder data, changements firewall
  • HIPAA : Accès dossiers médicaux, qui vu quoi quand
  • SOX : Changements systèmes financiers
  • GDPR : Accès données personnelles EU

SIEM génère reports automatisés compliance → audits simplifiés.

// 3. Incident Response & Forensics

Incident détecté : SIEM timeline reconstruction.

  • Qui : Attaquant IP, user compromis
  • Quand : Timeline précise attaque (première compromission → exfiltration)
  • Quoi : Fichiers accédés, commandes exécutées, data volée
  • Comment : Vecteur initial (phishing email, vuln exploitée)

// 4. User Behavior Analytics (UBA)

Détection anomalies comportement utilisateurs.

  • User login 03:00 AM (jamais fait avant)
  • Download 10 GB données (baseline 10 MB/jour)
  • Accès serveurs jamais utilisés auparavant

SIEM + ML apprend baseline comportement → alerte déviations.

⚠️ Pièges d'examen

Ce qui trompe les candidats

❌ Piège #1 : SIEM = firewall avancé
NON. SIEM = analyse logs, PAS firewall.

Firewall : Bloque trafic malveillant (prévention)
SIEM : Analyse logs, détecte menaces, alerte (détection)

SIEM ≠ protection active, c'est monitoring intelligent
❌ Piège #2 : SIEM remplace syslog server
FAUX. SIEM complète syslog (pas remplace).

Syslog : Collecte basique logs
SIEM : Syslog + Corrélation + Threat detection + Alerting

Architecture typique : Syslog collecte → SIEM analyse
❌ Piège #3 : SIEM détecte 100% attaques
FAUX. SIEM = outil puissant MAIS pas magique.

Limitations :
- Détecte patterns connus (règles/signatures)
- Zero-days sophistiqués peuvent bypasser
- False positives possibles
- Qualité règles = critique

SIEM améliore détection drastiquement MAIS pas infaillible
