Human hacking

Social Engineering — Manipulation psychologique

🎭
Analogie — pickpocket vs cambrioleur Hacker technique = cambrioleur (force serrures, casse fenêtres). Social engineer = pickpocket (manipulation, distraction, persuasion). Résultat identique (vol) MAIS méthode différente (humains vs systèmes).

Social Engineering = Manipulation psychologique humains → obtenir informations confidentielles ou accès non autorisé. Exploite confiance, peur, urgence, curiosité.

// Principe fondamental

"Humans = maillon faible sécurité"

  • Firewall parfait, chiffrement fort → inutile si user donne password
  • 95%+ breaches impliquent erreur humaine (Verizon DBIR)
  • Social engineering = souvent plus facile que exploit technique

// Techniques psychologiques

Authority (Autorité) : Attaquant prétend être patron, IT, police → victim obéit.

Urgency (Urgence) : "Compte bloqué dans 1h !" → victim panique, pas réfléchit.

Scarcity (Rareté) : "Offre limitée 10 min !" → FOMO (fear of missing out).

Trust (Confiance) : Attaquant construit relation → victim baisse garde.

Fear (Peur) : "Virus détecté !" "IRS lawsuit !" → victim réagit émotionnellement.

🧠
Social engineering = attaque humains, pas machines Défense technique (firewall, antivirus) = inefficace.
Défense = awareness training + policy + culture sécurité.
User éduqué = meilleure défense social engineering.
Email attacks

Phishing / Spear Phishing / Whaling

// Phishing (Hameçonnage)

Définition : Email frauduleux MASSE non ciblée → voler credentials ou installer malware.

Caractéristiques :

  • Volume : Millions emails envoyés (spam campaign)
  • Generic : "Dear Customer" (pas personnalisé)
  • Urgence : "Account suspended ! Click now !"
  • Fake sender : Spoof email (from: security@paypa1.com vs paypal.com)
  • Malicious link : URL fake login page (http://paypal-verify.tk)
Exemple phishing email :

From: security@paypa1.com (fake 'l' = 1)
Subject: URGENT: Account Suspended

Dear Customer,

Your PayPal account has been suspended due to suspicious activity.
Click below to verify identity within 24h or account deleted.

[Verify Account Now] → http://paypal-secure-login.tk

PayPal Security Team

Red flags :
- Generic greeting
- Urgency (24h deadline)
- Suspicious URL domain (.tk, not .com)
- Typo sender (paypa1 vs paypal)

// Spear Phishing (Hameçonnage ciblé)

Définition : Phishing CIBLÉ individu/organisation spécifique (personnalisé, recherche préalable).

Différences vs phishing :

  • Ciblé : 10-100 victims (vs millions phishing)
  • Personnalisé : "Hi John," (nom réel, poste, projets)
  • Contexte : Référence infos vraies (collègues, deals en cours)
  • Crédibilité : Très convaincant (OSINT research approfondie)
Exemple spear phishing :

From: sarah.jones@company.com (spoofed CEO)
To: john.smith@company.com (CFO)
Subject: Re: Q4 Merger Confidential

Hi John,

Per our discussion yesterday re: Acme acquisition,
legal needs updated financials ASAP for Friday board meeting.

Please review attached NDA and return signed by EOD.
Attachment: Acme_NDA_Final.pdf.exe (malware)

Thanks,
Sarah

Attacker research :
- LinkedIn: John = CFO, Sarah = CEO
- Press release: Acme merger public knowledge
- Calendar leak: Board meeting Friday
→ Highly credible (John likely clicks)

// Whaling (Chasse baleine)

Définition : Spear phishing ciblant EXECUTIVES (C-level : CEO, CFO, CTO).

Caractéristiques :

  • High-value targets : Accès maximum (financier, strategic)
  • Sophistication maximale : Recherche extensive, contexte parfait
  • Business Email Compromise (BEC) : Fraud wire transfer ($M)
🎯
Différences clés examen Phishing : Mass email, generic, millions victims
Spear phishing : Targeted, personalized, 10-100 victims
Whaling : Spear phishing ciblant executives (CEO/CFO/CTO)
Voice & SMS attacks

Vishing et Smishing

// Vishing (Voice Phishing)

Définition : Phishing via appel téléphonique (voice call social engineering).

Techniques vishing :

  • Caller ID spoofing : Affiche numéro légitime (banque, IRS, tech support)
  • VoIP : Appels gratuits, anonymes (difficile tracer)
  • Automated (robocalls) : IVR automatique ("Press 1 verify account")
  • Live attacker : Script persuasif, manipulation temps réel
Scénario vishing typique :

[Phone rings, Caller ID shows "IRS Official"]

Attacker: "IRS Tax Department. You owe $5,000 back taxes.
          Warrant issued arrest unless pay TODAY.
          Provide credit card settle immediately."

Victim: [panics, fear authority] → Gives CC number

Red flags :
- Urgency extreme (arrest TODAY)
- Fear tactic (warrant)
- Payment unusual (IRS never demands phone payment)
- No written notice (IRS always mails first)

// Smishing (SMS Phishing)

Définition : Phishing via SMS text message.

Caractéristiques smishing :

  • Mobile-specific : Exploite petits écrans (URLs tronquées)
  • High open rate : SMS taux lecture >90% (vs email ~20%)
  • Urgency : Messages courts, action immédiate
  • Malicious links : Shortened URLs (bit.ly) cachent destination
Exemple smishing SMS :

From: "USPS"
Message:
"Package delivery failed. Reconfirm address:
http://bit.ly/usps-track-3x7k"

→ User clicks → Fake USPS site → Enter personal info

Variantes :
- "Bank fraud alert! Verify: [link]"
- "Prize won! Claim: [link]"
- "Missed call voicemail: [link]" (installs malware)
Physical security

Tailgating et Piggybacking

Tailgating = Suivre employee autorisé passage porte sécurisée SANS autorisation (physical social engineering).

// Tailgating (non consensuel)

Scénario :

  1. Employee badge porte accès datacenter
  2. Attaquant suit juste derrière (pretend forgot badge)
  3. Porte ferme trop lentement → attaquant entre
  4. Employee pas challenge (politesse, assume légitime)

// Piggybacking (consensuel)

Différence : Victim AIDE attaquant intentionnellement.

Scénario :

  1. Attaquant mains pleines (café, laptop, boxes)
  2. "Excuse me, can you hold door ? Forgot badge car."
  3. Employee OUVRE porte intentionnellement (helpful)
  4. Attaquant entre (victim complice involontaire)

// Techniques tailgating

  • Timing : Suivre rush hours (nombreux employees, moins vigilance)
  • Props : Uniforme delivery, laptop bag, téléphone (air occupied/légitime)
  • Confidence : Marcher confiant (assume belongs)
  • Distraction : Fausse urgence ("Bathroom emergency !")

// Mitigation tailgating

  • Mantraps : Sas 2 portes (1 seule personne à la fois)
  • Turnstiles : Tourniquets physiques
  • Security guards : Vérification visuelle chaque personne
  • Policy + training : Challenge unknown persons, pas hold doors
  • Badges visible : Tous employees badge apparent
🚪
Culture sécurité vs politesse Employees hésite challenge (peur impoli).
Training doit normaliser : "Excuse me, badge please ?" = acceptable, pas rude.
Mantra : "Friendly but firm."
Identity fraud

Impersonation — Usurpation d'identité

Impersonation = Attaquant prétend être quelqu'un d'autre (employee, vendor, authority) → gain accès/infos.

// Types impersonation

1. Authority Impersonation

  • Prétend être : Police, FBI, IRS, auditor, manager
  • Exploite : Authority bias (humans obey authority figures)
  • Demande : Passwords, access, documents confidentiels
Attacker calls reception:
"FBI Agent Johnson. Cybercrime investigation your company.
Need CEO email logs past 6 months. Send secure-fbi-portal.tk"

Receptionist: [intimidated] → Forwards request IT
IT: [assume legitimate FBI] → Provides data

2. Third-Party Vendor

  • Prétend être : IT support, HVAC repair, delivery, cleaner
  • Uniforme : Fake badge, branded clothing
  • Access : Wander building, plant devices, shoulder surfing

3. Trusted Insider

  • Prétend être : Colleague, IT helpdesk, HR
  • Contexte : "Remember me ? We met last conference."
  • Request : "Can you reset my password ? Locked out."

// Mitigation impersonation

  • Verification procedures : Callback known numbers (not provided attacker)
  • Badge systems : Visual verification (photo ID, expiration)
  • Escort policy : Vendors ALWAYS escorted employee
  • Challenge culture : "May I see credentials ?" = normal
  • Out-of-band confirmation : CEO email request → call CEO verify
Fabricated scenario

Pretexting — Scénario fabriqué

Pretexting = Créer scénario fabriqué (pretext) gagner confiance → victim révèle infos volontairement.

// Principe pretexting

Différence vs autres attaques :

  • Phishing : Email simple, peu interaction
  • Pretexting : Relation construite, scénario élaboré, interaction prolongée

Étapes pretexting :

  1. Research : OSINT cible (job, hobbies, contacts)
  2. Develop pretext : Inventer identité/raison crédible
  3. Build rapport : Conversations multiples, gagner confiance
  4. Extract info : Demandes subtiles, progressives
Exemple pretexting IT support :

Call 1 (establish pretext) :
"Hi, John from IT. New security audit.
Need inventory all software versions.
What OS you running ?"
→ Victim: "Windows 10 Pro"

Call 2 (build trust, next day) :
"John again. Thanks yesterday.
Found vulnerability your department.
Need verify patch levels. What antivirus ?"
→ Victim: [trusts John now] "McAfee 2020"

Call 3 (extract credentials) :
"Critical patch tonight. Need test account verify.
Can you create temp admin account 'Patch_Test' ?
Password 'SecureTemp123' ?"
→ Victim: [believes ongoing audit] Creates account

→ Attacker gains admin access

// Contextes pretexting courants

  • IT Support : "Password reset", "Software audit", "Patch deployment"
  • HR : "Benefits verification", "Background check update"
  • Survey/Research : "Academic study", "Industry survey" (extract company info)
  • Vendor : "Contract renewal", "Invoice discrepancy" (wire transfer fraud)

// Mitigation pretexting

  • Verification policy : Callback official numbers AVANT share infos
  • Compartmentalization : Employees connaissent only need-to-know
  • Suspicious requests : Escalate manager (password reset calls, urgent wire transfers)
  • Awareness training : Recognize pretexting red flags (urgency, unusual requests)
⚠️ Pièges d'examen

Ce qui trompe les candidats

❌ Piège #1 : Whaling = phishing emails nombreux executives
FAUX. Whaling = spear phishing ciblant 1 executive (CEO/CFO/CTO).

Pas "nombreux executives" → 1 seul high-value target.
Whaling = sous-catégorie spear phishing (executives uniquement).
❌ Piège #2 : Vishing = email avec lien malveillant
FAUX. Vishing = Voice phishing (téléphone).

Email phishing = Phishing (email)
Voice call = Vishing
SMS = Smishing

Mémo : Vishing = Voice + Phishing
❌ Piège #3 : Tailgating = piggybacking (synonymes)
NUANCE. Tailgating = suivre SANS permission (furtif).
Piggybacking = victim AIDE attaquant (hold door).

Tailgating : Non consensuel
Piggybacking : Consensuel (victim complice involontaire)

Résultat identique (accès non autorisé) MAIS méthode différente
