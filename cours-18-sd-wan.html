Concept

SD-WAN — Software-Defined WAN

🎯
Analogie — GPS Waze intelligent SD-WAN = GPS qui choisit route DYNAMIQUEMENT selon conditions temps réel (trafic, accidents). Si route A congestionnée → bascule route B automatiquement. SD-WAN fait pareil avec liens WAN (Internet, MPLS, LTE).

SD-WAN = technologie créant réseau WAN virtuel (overlay) au-dessus de multiples liens physiques (underlay). Contrôle centralisé software, routage intelligent applicatif.

// Problème WAN traditionnel (MPLS)

  • ❌ Coût très élevé
  • ❌ Provisioning lent (30-90 jours nouveau site)
  • ❌ Bande passante limitée (prix exorbitant scaling up)
  • ❌ Rigidité (contrats longs, vendor lock-in)
  • ❌ Pas optimisé cloud (SaaS détour via datacenter)

// Solution SD-WAN

  • ✅ Utilise Internet broadband (50-70% moins cher)
  • ✅ Agrégation multiples liens (Internet + MPLS + 4G/5G)
  • ✅ Routage intelligent par application
  • ✅ Zero-touch provisioning (site online en heures)
  • ✅ Gestion centralisée cloud
🚀
SD-WAN = révolution WAN 2024 Majorité entreprises migrent MPLS → SD-WAN ou hybrid (SD-WAN + MPLS). Économies massives + agilité + cloud-ready.
Architecture

Architecture SD-WAN — Overlay / Underlay

// Underlay (Liens physiques)

Liens WAN réels transportant trafic :

  • Internet broadband (fibre, câble, DSL)
  • MPLS (optionnel, pour apps critiques)
  • 4G/5G LTE (backup ou mobilité)

// Overlay (Réseau virtuel)

Réseau logique SD-WAN au-dessus des underlays :

  • Tunnels chiffrés entre sites (IPsec, GRE)
  • Abstraction liens physiques
  • Routage intelligent software-defined

// Composants SD-WAN

Orchestrateur (Cloud Controller)
↓ Gestion centralisée, politiques, monitoring

Edge appliances (Sites)
[Bureau Paris] — SD-WAN appliance
↓ Internet
↓ MPLS (optionnel)
↓ 4G backup
↓
[Bureau NY] — SD-WAN appliance

→ Overlay : Tunnels virtuels Paris ↔ NY
→ Underlay : Internet + MPLS + 4G
Multi-path

Agrégation de liens WAN

SD-WAN utilise TOUS les liens simultanément (pas juste failover). Agrégation active.

// Modes utilisation liens

1. Active/Active

  • Tous liens utilisés simultanément
  • Load balancing intelligent (par application)
  • Bande passante = somme de tous liens

2. Active/Standby (Failover)

  • Lien primaire actif
  • Lien secondaire en backup (utilisé si primaire tombe)
  • Bascule automatique < 1 seconde

// Exemple agrégation

Site avec 3 liens :
- Internet fibre : 500 Mbps
- MPLS : 100 Mbps
- 4G LTE : 50 Mbps

Bande passante totale agrégée : ~650 Mbps

SD-WAN distribue :
→ Applications critiques (VoIP, SAP) : MPLS (QoS garanti)
→ SaaS (Office 365, Salesforce) : Internet direct (rapide)
→ Backup data : Internet fibre (capacité)
→ Si Internet tombe → 4G prend le relais
💡
Coût vs MPLS pur MPLS 500 Mbps = $5000/mois
SD-WAN (Internet 500 + MPLS 100 + 4G) = $1500/mois
→ Économie 70% + résilience accrue !
Routage intelligent

Application-Aware Routing

SD-WAN identifie applications (DPI - Deep Packet Inspection) et route intelligemment selon politiques.

// Fonctionnement

  1. SD-WAN identifie application (Zoom, Office 365, SAP, YouTube...)
  2. Applique politique définie par admin
  3. Route vers lien optimal (latence, bande passante, coût, QoS)
  4. Monitore performance temps réel → bascule si dégradation

// Exemples politiques

ApplicationPolitique SD-WANRaison
VoIP (Zoom, Teams) MPLS prioritaire Latence faible garantie
Office 365 Internet direct (local breakout) Rapide, pas détour via datacenter
SAP / ERP critique MPLS SLA garanti, sécurité
YouTube / Streaming Internet Bande passante élevée, pas critique
Backup data 4G (off-hours) Économie bande passante primaire

// Path Selection dynamique

SD-WAN monitore liens temps réel (latence, jitter, perte paquets) → bascule automatiquement si dégradation.

VoIP call en cours sur MPLS :
→ SD-WAN détecte latence MPLS augmente (50ms → 150ms)
→ Bascule automatiquement VoIP vers Internet (latence 20ms)
→ Appel pas interrompu, qualité maintenue ✅
🎯
Application-Aware = killer feature SD-WAN Routage traditionnel = décision basée IP/port uniquement.
SD-WAN = identifie application ET contexte business → routage optimal automatique.
Déploiement rapide

Zero-Touch Provisioning (ZTP)

ZTP = déploiement nouveau site en heures sans intervention technicien sur place. Plug & play.

// Workflow ZTP

  1. Admin cloud : Ajoute nouveau site dans orchestrateur (nom, politiques)
  2. Expédition : Appliance SD-WAN préconfiguré envoyé au site
  3. Installation locale : Employé bureau branche appliance (Internet + power)
  4. Appel maison automatique : Appliance contacte orchestrateur cloud
  5. Config download : Orchestrateur pousse config complète (tunnels, politiques, certificats)
  6. Online : Site actif en ~10-60 minutes ✅

// vs Provisioning traditionnel MPLS

Config manuelle✅ CLI équipement❌ Automatique cloud
Coût rollout💰💰💰 Élevé💰 Faible

ZTP = game-changer retail/franchises Chaîne retail 500 magasins → rollout SD-WAN en semaines (vs années MPLS). Employé magasin branche box, site online automatiquement.
Gestion unifié

Centralized Policy Management

Orchestrateur cloud = console unique gérant TOUS les sites. Politiques déployées en quelques clics.

// Fonctions orchestrateur

  • Politiques globales : "VoIP → MPLS" appliqué instantanément tous sites
  • Monitoring temps réel : Dashboard visibilité globale (latence, throughput, santé liens)
  • Troubleshooting : Logs centralisés, analytics
  • Firmware updates : Push software tous sites simultanément
  • Reporting : Compliance, utilisation bande passante

// Exemple politique centralisée

Admin créé politique "Office 365 Direct" :
→ Détecte trafic Office 365 (SharePoint, Teams, Outlook)
→ Route vers Internet local (pas via datacenter)
→ QoS : Priorité haute

Deploy to: All sites (500 bureaux)
[Apply button]

→ 500 sites adoptent politique instantanément
→ Pas besoin SSH chaque équipement
⚠️ Pièges d'examen

Ce qui trompe les candidats

❌ Piège #1 : SD-WAN = pas besoin Internet
FAUX. SD-WAN UTILISE Internet comme underlay principal (+ optionnel MPLS/4G).

SD-WAN sécurise Internet via chiffrement (IPsec) → équivalent sécurité MPLS
❌ Piège #2 : SD-WAN remplace firewall
NON. SD-WAN = WAN optimization + routage intelligent.

Beaucoup SD-WAN incluent firewall basique MAIS firewall complet (NGFW) toujours recommandé séparé/intégré.
❌ Piège #3 : ZTP = aucune intervention
NUANCE. ZTP = zero intervention technique réseau.

Quelqu'un doit quand même physiquement brancher appliance (employé bureau OK, pas besoin ingénieur réseau).
