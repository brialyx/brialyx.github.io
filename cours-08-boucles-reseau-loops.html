Broadcast storm symptoms

Symptômes boucle réseau — Network meltdown

Boucle réseau (Network Loop) = Chemin redondant Layer 2 sans STP → broadcast frames loop infiniment → network catastrophe.

// Symptôme #1 : CPU 100% switches

Switch# show processes cpu sorted

CPU utilization for five seconds: 99%/0%
one minute: 99%; five minutes: 99%

PID  Process               CPU
104  Ethernet Input      95%
23   ARP Input          2%

→ Switch CPU saturé processing broadcast frames

// Symptôme #2 : MAC address table flapping

MAC flapping = Same MAC address seen multiple ports → table entries change rapidly.

Switch# show mac address-table dynamic

VLAN  MAC Address         Port
10    aa:bb:cc:dd:ee:ff  Gi0/1
10    aa:bb:cc:dd:ee:ff  Gi0/2 (flapping!)

→ Same MAC multiple ports = loop indicator

// Symptôme #3 : LED ports rapid blinking

  • Switch port LEDs blink extremely fast (flood traffic)
  • All ports affected (broadcast propagates everywhere)
  • Link utilization 90-100% (bandwidth saturated)

// Symptôme #4 : Total connectivity loss

User experience : Everything stops working instantly.

  • Ping : Timeout (network overwhelmed)
  • Web : Doesn't load (applications timeout)
  • Email : Can't send/receive
  • File shares : Inaccessible

→ Excessive topology changes = loop or unstable STP

Root causes

Causes boucles réseau — Comment arrivent ?

// Cause #1 : STP disabled (JAMAIS faire production)

BAD configuration :
Switch(config)# no spanning-tree vlan 10

→ STP disabled VLAN 10
→ Redundant paths = instant loop
→ NEVER disable STP production

// Cause #2 : Unmanaged switch inserted (rogue device)

Scénario : User plugs personal unmanaged switch (no STP support).

Managed Switch A ←→ Managed Switch B (STP enabled)
      ↑
      |
   Unmanaged Switch (no STP) ← User plugs here
      |
      ↓
   Managed Switch B

→ Unmanaged switch creates path bypasses STP
→ Loop immediate (broadcast storm)

// Cause #3 : User cables both ends PC → switch (accidental)

User PC : 2 Ethernet ports (NIC + docking station)
User plugs BOTH → Switch ports Gi0/5 + Gi0/6

→ PC creates bridge between ports
→ Loop on VLAN

Prevention : Port Security (1 MAC max per port)

// Cause #4 : PortFast on trunk port (misconfiguration)

INCORRECT config :
interface GigabitEthernet0/1
  switchport mode trunk
  spanning-tree portfast trunk

→ PortFast bypasses STP states (Listening, Learning)
→ Port goes Forwarding immediately
→ If loop exists → No STP protection → Storm

// Cause #5 : Fiber TX/RX swapped (creates loop)

Scenario : Fiber optic link, TX/RX reversed both ends.

Normal : TX-A → RX-B, TX-B → RX-A (correct)

Swapped :
TX-A → TX-B (looped back)
RX-A ← RX-B (looped back)

→ Data sent immediately received same switch
→ Creates internal loop → Storm
Spanning Tree Protocol

STP — Solution boucles Layer 2

// Principe STP (IEEE 802.1D)

STP = Prevents loops blocking redundant paths logiquement. Maintains loop-free topology.

STP workflow :

1. Elect Root Bridge
   → Switch lowest Bridge ID (priority + MAC)
   → Default priority 32768 (all switches)
   → Lowest MAC wins (tie-breaker)

2. Calculate best path each switch → Root
   → Path cost : 100 Mbps = 19, 1 Gbps = 4, 10 Gbps = 2
   → Shortest cost = Root Port (forwarding)

3. Elect Designated Port each segment
   → Port closest Root Bridge
   → Forwarding state

4. Block redundant ports
   → Non-Root, Non-Designated = Blocking
   → Prevents loops

5. Monitor topology
   → Link fails ? → Blocked port activates (convergence ~30-50 sec)

// STP Port States

StateForwards Data ?Learns MACs ?Duration
Blocking ❌ No ❌ No Permanent (until topology change)
Listening ❌ No ❌ No 15 seconds (Forward Delay)
Learning ❌ No ✅ Yes 15 seconds (Forward Delay)
Forwarding ✅ Yes ✅ Yes Permanent (normal operation)
Disabled ❌ No ❌ No Manual shutdown

Switch# show spanning-tree

→ Port Gi0/3 blocked (redundant path)
Rapid Spanning Tree

RSTP — Convergence rapide

RSTP (IEEE 802.1w) = Rapid Spanning Tree Protocol. Améliore STP : convergence <5 seconds (vs STP 30-50 sec).

// Améliorations RSTP vs STP

FeatureSTP (802.1D)RSTP (802.1w)
Convergence 30-50 seconds <5 seconds
Port States 5 (Blocking, Listening, Learning, Forwarding, Disabled) 3 (Discarding, Learning, Forwarding)
Port Roles Root, Designated, Blocking Root, Designated, Alternate, Backup
Proposal/Agreement ❌ No ✅ Yes (fast convergence)
Backward compatible - ✅ Yes (works with STP)

// RSTP Port Roles

  • Root Port : Best path to Root Bridge (forwarding)
  • Designated Port : Forwards frames segment (forwarding)
  • Alternate Port : Backup path to Root (discarding, ready fast failover)
  • Backup Port : Backup Designated Port (discarding, rare)

// Enable RSTP (Cisco)

Switch(config)# spanning-tree mode rapid-pvst

→ Enable RSTP per-VLAN (Cisco proprietary Rapid PVST+)
→ Modern default most switches

Verify :
Switch# show spanning-tree summary
Switch is in rapid-pvst mode
⚡
RSTP = modern standard Most modern switches : RSTP enabled default.
Legacy STP (802.1D) : Slow convergence (30-50 sec downtime link failure).
RSTP : <5 sec convergence (acceptable production environments).
Always use RSTP (or MSTP 802.1s) modern networks.
Access port optimization

PortFast — Bypass STP delays access ports

PortFast = Feature bypass STP states (Listening, Learning) → port goes Forwarding immediately.

// Problème sans PortFast

User boots PC → NIC link UP
→ STP port : Listening (15 sec) → Learning (15 sec) → Forwarding
→ 30 seconds delay before network access

→ DHCP request sent immediately (PC boots) → Discarded (port not forwarding)
→ DHCP timeout → PC gets APIPA 169.254.x.x
→ User : "No network access" (frustration)

// Solution PortFast

User boots PC → NIC link UP
→ PortFast enabled → Port Forwarding immediately (0 sec)
→ DHCP request sent → Success ✅
→ User : Network access instant

// Configuration PortFast

Per-interface (access ports)

interface GigabitEthernet0/5
  switchport mode access
  switchport access vlan 10
  spanning-tree portfast

→ PortFast enabled port Gi0/5 (PC connected)

Global (all access ports)

Switch(config)# spanning-tree portfast default

→ Enable PortFast ALL access ports automatically
→ Does NOT apply trunk ports (safe)

// PortFast Best Practices

✅ USE PortFast :

  • Access ports connected end devices (PCs, printers, phones, servers)
  • Ports où loops impossible (single device, no switches downstream)

❌ NEVER PortFast :

  • Trunk ports (switch ↔ switch links)
  • Ports connected autre switch (creates loops)
  • Ports où user peut plug switch (rogue device risk)

// BPDU Guard (protection PortFast)

BPDU Guard = Security feature : disable port si reçoit BPDU (switch connected → loop risk).

interface GigabitEthernet0/5
  spanning-tree portfast
  spanning-tree bpduguard enable

→ If BPDU received (switch plugged) → Port err-disabled
→ Prevents loops rogue switches

Global enable ALL PortFast ports :
Switch(config)# spanning-tree portfast bpduguard default
⚠️
PortFast trunk = DANGEROUS JAMAIS enable PortFast trunk ports.
Command : spanning-tree portfast trunk = DISABLE STP protection → Loops.
Examen : PortFast = access ports ONLY (end devices), never trunks.
⚠️ Pièges d'examen

Ce qui trompe les candidats

❌ Piège #1 : Broadcast storm = Layer 3 routing problem
FAUX. Broadcast storm = Layer 2 switching loop.

Layer 2 : Switches, broadcast frames, MAC addresses, STP.
Layer 3 : Routers, IP packets, routing protocols.

Broadcast storm caused redundant switch paths no STP.
Routing loops prevented TTL (Time To Live) IP packets.
Examen : "Broadcast storm" = L2 STP problem, not L3 routing.
❌ Piège #2 : STP convergence = instant
FAUX. STP convergence = 30-50 seconds (slow).

STP (802.1D) : Listening 15s + Learning 15s = 30s minimum.
RSTP (802.1w) : <5 seconds (rapid convergence).

Link failure STP network → 30-50s downtime before blocked port activates.
RSTP → <5s downtime (acceptable production).
Examen : Know STP slow, RSTP fast convergence times.
❌ Piège #3 : PortFast = disable STP
FAUX. PortFast = bypass initial states, not disable STP.

PortFast : Port goes Forwarding immediately (skip Listening/Learning).
STP still active : If BPDU received → Port transitions normal STP states.

PortFast + BPDU Guard = Best combo (fast boot + loop protection).
Examen : PortFast optimizes access ports, doesn't disable STP completely.
