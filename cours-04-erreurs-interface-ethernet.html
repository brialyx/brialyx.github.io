Frame integrity

CRC Errors — Corruption données

CRC (Cyclic Redundancy Check) = Checksum Ethernet frame vérifier intégrité données. Frame reçue → recalcule CRC → compare. Mismatch = CRC error.

// Principe CRC

Ethernet frame structure :

[Preamble] [Dest MAC] [Src MAC] [Type] [Data] [FCS]

FCS (Frame Check Sequence) = 4 bytes CRC-32 checksum

Sender :
→ Calculate CRC-32 over frame data → Append FCS

Receiver :
→ Recalculate CRC-32 received data
→ Compare calculated CRC vs received FCS
→ Match ? ✅ Frame OK
→ Mismatch ? ❌ CRC error → Frame dropped

// Causes CRC errors

  • Câble défectueux : Bad termination, damaged cable (Cours TR03)
  • EMI : Electromagnetic interference corrupts bits transit (Cours TR03)
  • Duplex mismatch : Half-duplex ↔ Full-duplex mismatch (collisions → corruption)
  • NIC defective : Hardware failure NIC transmit/receive circuitry
  • Switch port failure : Port hardware defect

// Diagnostic CRC errors

Symptom : CRC errors incrementing

Switch# show interfaces Gi0/1
→ 250 input errors, 245 CRC, 5 frame

Step 1 : Check duplex mismatch
Switch# show interfaces Gi0/1 | include duplex
→ Full-duplex, 1000Mb/s

Connected device (PC) : ipconfig /all
→ Speed: 1 Gbps, Duplex: Half
→ Mismatch found ! Switch Full ↔ PC Half

Fix : Configure both Full-duplex (auto-negotiation ou manual)

Step 2 : If duplex OK, check cable
→ Swap cable known-good → Errors stop ? ✅ Cable defective

Step 3 : If cable OK, check NIC/port
→ Move device different switch port → Errors follow device ? NIC bad
→ Errors stay original port ? Switch port bad
🎯
Question fréquente examen "High CRC errors interface. MOST likely cause ?"
A) Duplex mismatch
B) Broadcast storm
C) Routing loop
D) DNS failure

Réponse : A) Duplex mismatch
CRC errors = Layer 1/2 physical/data link. Duplex mismatch = cause commune CRC/collisions.
Frame size errors

Runts & Giants — Frames taille incorrecte

// Runts (undersized frames)

Runt = Frame < 64 bytes (minimum Ethernet frame size). Considéré invalid, dropped.

Ethernet frame min size : 64 bytes
(Includes: 14 bytes header + 46 bytes min payload + 4 bytes FCS)

Payload < 46 bytes ? → Padded to 46 bytes (zeros added)

Runt example : 50 bytes frame
→ Too short (< 64 bytes) → Runt error

// Causes Runts

  • Collisions : Collision fragments frame (partial transmission)
  • Duplex mismatch : Half ↔ Full → Collisions → Runts
  • Faulty NIC : Hardware malfunction transmits incomplete frames
  • Software bug : Driver/firmware bug generates malformed frames

// Giants (oversized frames)

Giant = Frame > 1518 bytes (maximum standard Ethernet frame). Considéré invalid, dropped.

Ethernet frame max size : 1518 bytes
(14 bytes header + 1500 bytes max payload + 4 bytes FCS)

Exception : Jumbo frames
→ Jumbo frames : Up to 9000 bytes (non-standard, requires support)
→ If switch pas support jumbo → frames > 1518 = Giants

Giant example : 2000 bytes frame
→ Too large (> 1518 bytes) → Giant error

// Causes Giants

  • Jumbo frames mismatch : Device sends jumbo (9000B), switch expects standard (1518B)
  • Faulty NIC : Hardware malfunction transmits oversized frames
  • Software bug : Driver bug generates malformed large frames
  • VLAN tag issue : 802.1Q tag adds 4 bytes → 1518 + 4 = 1522 (some switches count as giant si strict)

// Diagnostic Runts/Giants

Runts troubleshooting :
→ Check collisions (runts often accompany collisions)
→ Check duplex mismatch (show interfaces | include duplex)
→ Swap NIC (runts follow device ? NIC faulty)

Giants troubleshooting :
→ Check jumbo frames config (MTU settings consistent ?)
→ Disable jumbo frames test (if enabled, disable → giants stop ?)
→ Swap NIC (giants follow device ? NIC faulty)
⚠️
Runts vs Giants = frame size violations Runts < 64 bytes (usually collisions/duplex mismatch).
Giants > 1518 bytes (usually jumbo frames mismatch/NIC fault).
Both = frames dropped (impact connectivity/performance).
CSMA/CD

Collisions — Half-Duplex legacy

Collision = 2+ devices transmettent simultanément shared medium Ethernet (half-duplex). Signaux se chevauchent → corruption.

// CSMA/CD (Collision Detection)

CSMA/CD = Carrier Sense Multiple Access / Collision Detection. Protocole Ethernet half-duplex gestion collisions.

CSMA/CD workflow :

1. Carrier Sense : Listen medium (wire busy ?)
   → Busy ? Wait. Idle ? Proceed.

2. Transmit : Send frame

3. Collision Detection : Monitor while transmitting
   → Collision detected ? (voltage spike, jamming signal)

4. Jam Signal : Send 32-bit jam pattern (alert all devices collision occurred)

5. Backoff : Wait random time (exponential backoff algorithm)
   → Retry transmission

6. Retry : Maximum 16 attempts → If still collisions, abort

// Half-Duplex vs Full-Duplex

CritèreHalf-DuplexFull-Duplex
Transmission One direction at a time (send OR receive) Both directions simultaneously (send AND receive)
Collisions ⚠️ Possible (shared medium) ✅ Impossible (dedicated paths TX/RX)
CSMA/CD ✅ Required ❌ Disabled (no collisions possible)
Performance ⚠️ ~50-60% efficiency (collision overhead) ✅ ~95%+ efficiency
Usage moderne ❌ Obsolete (hubs legacy) ✅ Standard (switches)

// Causes Collisions modernes

Modern networks = Full-duplex switches → Collisions = abnormal.

  • Duplex mismatch : Switch Full ↔ Device Half (most common cause)
  • Legacy hub : Hub still present network (rare 2024)
  • Faulty auto-negotiation : Auto-negotiate failed → defaults half-duplex
Duplex mismatch scenario :

Switch port Gi0/1 : Full-duplex (auto-negotiation)
PC NIC : Half-duplex (manual config ou failed auto-neg)

PC transmits → Thinks shared medium (half-duplex logic)
Switch transmits simultaneously → Full-duplex allows
→ Collision PC side (PC detects overlap)
→ Switch NO collision detection (full-duplex disabled CSMA/CD)

→ PC retransmits (backoff) → Performance degraded
→ Switch sees late collisions, CRC errors
Timing violations

Late Collisions — Duplex mismatch signature

Late Collision = Collision détectée APRÈS 512 bits (64 bytes) transmis. Violation timing Ethernet. Signature duplex mismatch.

// Normal Collision vs Late Collision

Normal Collision (Half-Duplex) :
→ Detected WITHIN first 64 bytes (512 bits) transmission
→ Device stops transmitting, sends jam signal, backoff, retry
→ Expected behavior half-duplex

Late Collision :
→ Detected AFTER 64 bytes transmitted
→ Device already committed significant data
→ Too late stop cleanly → Frame corrupted
→ Abnormal, indicates problem

// Slot Time (512 bits)

Slot Time = 512 bit-times = 64 bytes @ 10 Mbps. Minimum frame size garantit collision detection.

Principe : Worst-case propagation delay network < slot time → Collision detected sender AVANT finit transmission min frame.

Late collision = Violation slot time rule → Indicates network diameter trop large OU duplex mismatch.

// Causes Late Collisions

  • Duplex mismatch : #1 cause (90%+ late collisions)
  • Cable trop long : Ethernet max 100m → >100m late collisions possible (rare modern)
  • Too many repeaters : Excessive hubs/repeaters (legacy, obsolete)

// Diagnostic Late Collisions

Switch# show interfaces Gi0/1
→ 150 late collisions

Step 1 : Check duplex
Switch# show interfaces Gi0/1 | include duplex
→ Full-duplex, 1000Mb/s

Connected device (Server) NIC settings :
→ Speed: 1 Gbps, Duplex: Half

→ Mismatch confirmed ! Fix both Full-duplex

Fix :
Switch(config-if)# duplex full
Server NIC : Configure Full-duplex (driver settings)

Verify : Late collisions stop incrementing ✅
🎯
Late collisions = RED FLAG duplex mismatch Examen : "Interface shows late collisions. MOST likely cause ?"
Réponse : Duplex mismatch

Late collisions + CRC errors + runts = Trinity duplex mismatch symptoms.
Error counters

Input / Output Errors — Compteurs généraux

// Input Errors

Input Errors = Total erreurs frames REÇUES. Somme : CRC + Runts + Giants + Frame errors + Overruns + Ignored.

Input Errors breakdown :

CRC : Frame Check Sequence mismatch (corruption)
Runts : Frames < 64 bytes
Giants : Frames > 1518 bytes
Frame : Frame alignment errors (bits pas byte-aligned)
Overrun : Receiver buffer overflow (traffic faster than processing)
Ignored : Frames dropped (buffer full, no resources)

// Output Errors

Output Errors = Total erreurs frames TRANSMISES. Somme : Collisions + Late Collisions + Deferred + Underruns.

Output Errors breakdown :

Collisions : Normal collisions (half-duplex)
Late Collisions : Collisions after 64 bytes (duplex mismatch)
Deferred : Transmission delayed (medium busy, half-duplex waiting)
Underrun : Transmit buffer underflow (CPU too slow supply data)

// Overruns vs Underruns

Overrun (Input) : Receiver buffer overflow → Incoming data faster than CPU process → Frames dropped.

Underrun (Output) : Transmit buffer underflow → CPU pas supply data fast enough → Transmission gaps, errors.

Causes : Usually CPU overloaded, memory insufficient, hardware limitations.

// Interprétation Error Counters

  • Input Errors dominants : Problème incoming (cable, EMI, sender NIC)
  • Output Errors dominants : Problème outgoing (duplex, local NIC, collisions)
  • Both high : Bidirectional problem (duplex mismatch, faulty switch port)
  • Incrementing rapidly : Active problem (fix urgently)
  • Static count : Old errors (monitor, pas urgent si stable)
📈
Monitor error rates, not absolute counts Interface 10 years uptime → 1000 CRC errors total = OK (rare glitches).
Interface 1 hour uptime → 1000 CRC errors = PROBLEM (16 errors/min).
Command : show interfaces twice 5 min apart → Compare counters increment.
⚠️ Pièges d'examen

Ce qui trompe les candidats

❌ Piège #1 : Collisions = toujours problème
NUANCE. Dépend duplex mode.

Half-duplex : Collisions = NORMAL (expected CSMA/CD behavior).
Full-duplex : Collisions = ABNORMAL (indicates duplex mismatch).

Modern networks full-duplex switches → Collisions = problem.
Examen : Context matters. "Full-duplex interface collisions" = duplex mismatch.
❌ Piège #2 : Late collisions = cable trop long
RARE moderne. Late collisions = 90%+ duplex mismatch.

Cable length violations possible (>100m Ethernet), MAIS rare production.
Duplex mismatch = cause #1 late collisions (configuration error common).

Examen : "Late collisions interface. FIRST check ?"
Réponse : Duplex settings (pas cable length).
❌ Piège #3 : Giants = frames > 1500 bytes
FAUX. Giants = frames > 1518 bytes.

Standard Ethernet frame max : 1518 bytes (14 header + 1500 payload + 4 FCS).
MTU 1500 = payload max, pas frame total.

Examen : Giants threshold = 1518 bytes (pas 1500).
Résumé

🔑 Ce que tu dois retenir absolument

→CRC errors : Frame Check Sequence mismatch (data corruption). Causes = cable defect, EMI, duplex mismatch, NIC failure. Diagnostic = check duplex, swap cable, test NIC/port
→Runts : Frames < 64 bytes. Causes = collisions (fragments), duplex mismatch, faulty NIC. Giants : Frames > 1518 bytes. Causes = jumbo frames mismatch, faulty NIC
→Collisions : Half-duplex normal (CSMA/CD). Full-duplex abnormal (duplex mismatch). Modern networks full-duplex → collisions = problem
→Late collisions : Collision after 64 bytes. Signature #1 duplex mismatch (90%+ cause). Also possible cable >100m (rare). Late collisions + CRC + runts = duplex mismatch trinity
→Input Errors : Sum CRC + Runts + Giants + Frame + Overrun + Ignored. Incoming problems. Output Errors : Sum Collisions + Late Collisions + Deferred + Underrun. Outgoing problems
→Duplex mismatch : Most common cause interface errors. Symptoms = CRC errors, late collisions, runts, poor performance. Fix = configure both ends same duplex (Full recommended)
