Concept

VPN — Virtual Private Network

🔐
Analogie — tunnel blindé VPN = tunnel blindé dans Internet public. Données chiffrées circulent dans tunnel → personne ne voit contenu même si intercepte. Comme envoyer lettre dans coffre-fort verrouillé par La Poste.

VPN = tunnel chiffré sécurisant communications sur réseau non sécurisé (Internet).

// Fonctions VPN

  • Confidentialité : Chiffrement bout-en-bout (tiers ne peuvent lire)
  • Intégrité : Détecte modifications paquets
  • Authentification : Vérifie identité endpoints
  • Accès distant : Employés → ressources entreprise depuis maison
  • Interconnexion sites : Relier bureaux via Internet (alternative MPLS)
Types VPN

Site-to-Site vs Remote-Access

// Site-to-Site VPN

Connecte 2 réseaux (bureau ↔ bureau). Endpoints = routeurs/firewalls.

Architecture :

[Bureau Paris] — [VPN Gateway Paris]
↓ Tunnel VPN chiffré
[Internet]
↓
[VPN Gateway NY] — [Bureau New York]

→ Employés Paris accèdent serveurs NY transparemment
→ Comme si même LAN

Caractéristiques :

  • Always-on : Tunnel permanent 24/7
  • Transparent : Users ne savent pas VPN existe
  • Gateway-to-gateway : Équipements réseau, pas clients finaux
  • Usage : Relier bureaux multi-sites

// Remote-Access VPN

Connecte utilisateur individuel (laptop/mobile) à réseau entreprise.

Architecture :

[Laptop employé maison/café]
↓ VPN client software
[Internet]
↓
[VPN concentrator entreprise]
↓
[Réseau corporate + serveurs]

→ Employé travaille de n'importe où
→ Accède ressources comme au bureau

Caractéristiques :

  • On-demand : Utilisateur initie connexion manuellement
  • Client software : App VPN sur device (OpenVPN, Cisco AnyConnect, etc.)
  • User-to-network : Individuel → réseau corporate
  • Usage : Télétravail, accès distant

// Comparaison

CritèreSite-to-SiteRemote-Access
ConnexionRéseau ↔ RéseauUser ↔ Réseau
EndpointsGateways (routeurs/FW)Client device (laptop/phone)
ActivationAlways-on (permanent)On-demand (manuel)
UtilisateursTransparent (tous users site)Explicite (user lance VPN)

🎯
Question TRÈS fréquente examen "Type VPN pour relier bureaux Paris et New York ?"

Réponse : Site-to-Site VPN

"Employé télétravail doit accéder serveurs entreprise ?"
Réponse : Remote-Access VPN
Protocole Layer 3

IPsec — IP Security

IPsec = suite protocoles Layer 3 (Network) sécurisant trafic IP. Standard VPN site-to-site.

// Composants IPsec

AH (Authentication Header) : Authentification + intégrité (pas chiffrement)

ESP (Encapsulating Security Payload) : Chiffrement + authentification + intégrité (standard actuel)

IKE (Internet Key Exchange) : Négociation clés, authentification endpoints (Phase 1 + Phase 2)

// 2 modes IPsec

1. Transport Mode

  • Chiffre UNIQUEMENT payload (données), pas header IP
  • IPs source/dest originales visibles
  • Usage : Host-to-host (2 endpoints directs)
  • ⚠️ Rare en production (sauf cas spécifiques)

2. Tunnel Mode

  • Chiffre TOUT le paquet IP original (header + payload)
  • Encapsule dans nouveau header IP (IPs VPN gateways)
  • Usage : Site-to-Site VPN (standard)
  • ✅ Cache topologie réseau interne

// Transport vs Tunnel

Tunnel Mode (standard) :
[New IP header] [ESP] [Original IP header] [Payload] [ESP trailer]
↑ ↑ ↑
VPN gateways IPs Chiffré complet

Transport Mode (rare) :
[Original IP header] [ESP] [Payload] [ESP trailer]
↑ ↑
Visible (pas chiffré) Chiffré

// Avantages IPsec

  • ✅ Layer 3 (transparent applications)
  • ✅ Strong crypto (AES-256, SHA-256)
  • ✅ Standard ouvert (interopérabilité)
  • ✅ Performances (accélération HW possible)
  • ⚠️ Configuration complexe
  • ⚠️ NAT traversal problématique (résolu avec NAT-T)
Protocole Layer 7

SSL/TLS VPN

SSL/TLS VPN = VPN basé sur protocoles SSL/TLS (HTTPS). Accès via navigateur web ou client léger.

// 2 modes SSL/TLS VPN

1. Clientless (Portail web)

  • Accès via navigateur web (HTTPS)
  • Pas d'installation client
  • Accès limité : Webmail, intranet, applications web
  • ✅ Simplicité (n'importe quel browser)
  • ⚠️ Fonctionnalités limitées

2. Full Tunnel (Client software)

  • Client VPN léger (OpenVPN, Pulse Secure, Cisco AnyConnect)
  • Tunnel complet comme IPsec
  • Accès toutes ressources réseau
  • ✅ Fonctionnalités complètes

// SSL/TLS VPN vs IPsec

SSL/TLS VPN = standard Remote-Access moderne Port 443 (HTTPS) passe TOUS firewalls → idéal utilisateurs mobiles (hôtels, cafés, aéroports). Client léger, simple déploiement.
Routage trafic

Split Tunneling vs Full Tunneling

// Full Tunneling

TOUT le trafic passe par tunnel VPN (Internet inclus).

User veut YouTube :
[Laptop] → VPN tunnel → [Corporate FW] → [Internet] → YouTube

→ TOUT trafic (corporate + Internet) via VPN

Avantages :

  • ✅ Sécurité maximale : Tout trafic inspecté par corporate firewall
  • ✅ Contrôle total : Policies entreprise appliquées
  • ✅ DLP (Data Loss Prevention) : Monitoring complet

Inconvénients :

  • ⚠️ Bande passante corporate surchargée (Netflix/YouTube des employés → pipe entreprise)
  • ⚠️ Latence accrue : Détour via corporate gateway
  • ⚠️ Performances dégradées streaming/gaming

// Split Tunneling

Trafic corporate → VPN, trafic Internet → direct.

User veut serveur corporate :
[Laptop] → VPN tunnel → [Corporate network] ✅

User veut YouTube :
[Laptop] → Internet direct (pas VPN) ✅

→ Routes intelligentes selon destination

Avantages :

  • ✅ Performances : Internet direct = rapide
  • ✅ Économie bande passante corporate
  • ✅ Latence faible pour streaming/gaming

Inconvénients :

  • ⚠️ Risque sécurité : Trafic Internet pas inspecté
  • ⚠️ Malware : User peut télécharger malware depuis Internet direct
  • ⚠️ DLP bypass : Données sensibles exfiltrées via Internet direct

// Comparaison

Sécurité✅ Maximale⚠️ Moindre

⚠️
Split Tunneling = débat sécurité Entreprises haute sécurité (finance, défense, santé) interdisent split tunneling → full tunneling obligatoire.
Autres entreprises autorisent split pour UX (Netflix pas via VPN).
⚠️ Pièges d'examen

Ce qui trompe les candidats

❌ Piège #1 : VPN = anonymat complet
FAUX pour VPN corporate.

VPN corporate = employer VOIT tout trafic (logs, monitoring).
VPN commercial (NordVPN, etc.) ≠ VPN corporate (différent usage)
❌ Piège #2 : IPsec Transport = standard site-to-site
FAUX. IPsec Tunnel Mode = standard site-to-site.

Transport Mode = rare (host-to-host spécifique)
❌ Piège #3 : Split Tunneling = plus sécurisé
FAUX. Split Tunneling = MOINS sécurisé que Full.

Internet direct = bypass firewall corporate, risque malware/DLP.
Full Tunneling = tout inspecté (sécurité max)
Résumé

🔑 Ce que tu dois retenir absolument

→Site-to-Site : Réseau↔Réseau, always-on, gateways. Remote-Access : User↔Réseau, on-demand, client software
→IPsec : Layer 3, Tunnel Mode (chiffre tout, standard site-to-site) vs Transport Mode (payload seul, rare)
→SSL/TLS VPN : Layer 7, port 443, clientless (browser) ou full tunnel (client léger), standard remote-access moderne
→Full Tunneling : Tout via VPN (sécurité max, performances ⚠️). Split Tunneling : Corporate VPN, Internet direct (UX ✅, sécurité ⚠️)
