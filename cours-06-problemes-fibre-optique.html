Physical problems

Connecteurs sales / cassés — Cause #1

Fibre optique = Signaux lumineux (photons) transmis core verre/plastique. Propreté critique : Poussière/saleté bloque lumière.

// Connecteur sale (dirty connector)

Problème #1 fibre : Poussière, huile peau, résidus bloquent passage lumière → signal loss.

Symptômes connecteur sale :

→ No link (link down, LED OFF)
→ Intermittent link (flapping UP/DOWN)
→ High error rate (CRC errors, retransmissions)
→ Reduced throughput (signal weak, low SNR)

// Nettoyage connecteur

Procédure : Toujours nettoyer connecteurs AVANT troubleshooting avancé.

Outils nettoyage :

1. Fiber cleaning pen (click-type cleaner)
→ Insert connector ferrule, click 2-3×, rotate
→ Removes dust, oil, debris

2. Lint-free wipes + 99% isopropyl alcohol
→ Dampen wipe alcohol, wipe ferrule gently
→ Let dry 30 sec (alcohol evaporates)

3. Compressed air (canned air)
→ Blow dust out connector (BEFORE insertion)
→ Hold can upright (avoid liquid spray)

⚠️ NEVER :
→ Touch ferrule fingers (oils contaminate)
→ Use regular tissues (lint remains)
→ Blow with mouth (moisture contamination)

// Connecteur cassé (broken connector)

Dommages physiques : Ferrule cracked, fiber core broken, alignment sleeve damaged.

  • Symptômes : Complete signal loss (no link), no recovery cleaning
  • Causes : Drop connector, excessive insertion force, overtightening
  • Diagnostic : Visual inspection (microscope fiber scope), swap connector
  • Résolution : Replace connector (re-terminate fiber OU swap patch cable)
🔬
Fiber inspection microscope = essential tool Fiber microscope (portable, USB) examine ferrule end-face.
Good : Clean, smooth surface, no scratches.
Bad : Dust, cracks, pits, scratches visible.
Rule : ALWAYS inspect + clean BEFORE troubleshoot electrical/optical.
🎯
Question fréquente examen "Fiber link intermittent, LED flapping. FIRST troubleshooting step ?"
A) Replace transceiver
B) Check optical budget
C) Clean connectors
D) Run OTDR

Réponse : C) Clean connectors
Dirty connector = cause #1 fiber problems (90%+). Clean first, advanced tests after.
Fiber types

SMF vs MMF — Type mismatch

Incompatibilité type fibre = SMF transceiver + MMF cable (ou inverse) → No link OU performance terrible.

// SMF (Single-Mode Fiber)

Caractéristiques SMF :

Core diameter : 8-10 microns (très small)
Light source : Laser (1310 nm, 1550 nm)
Mode propagation : Single mode (straight path)
Distance : Long (10-80+ km datacenter, 100+ km telco)
Bandwidth : Very high (100 Gbps+)
Cost : Higher (laser transceivers expensive)
Color : Yellow jacket (OS2 standard)

// MMF (Multi-Mode Fiber)

Caractéristiques MMF :

Core diameter : 50 ou 62.5 microns (larger)
Light source : LED (850 nm, 1300 nm)
Mode propagation : Multiple modes (bouncing paths)
Distance : Short (300m-550m typical datacenter)
Bandwidth : Moderate (10-40 Gbps)
Cost : Lower (LED transceivers cheaper)
Color : Orange/Aqua jacket (OM1/OM2 orange, OM3/OM4 aqua)

// Problèmes incompatibilité

ScenarioTransceiverCableRésultat
✅ Correct SMF (1000BASE-LX) SMF (Yellow OS2) ✅ Works perfectly
✅ Correct MMF (1000BASE-SX) MMF (Orange OM1/OM2) ✅ Works perfectly
❌ Mismatch SMF laser MMF cable ⚠️ May work short distance (<100m), unreliable
❌ Mismatch MMF LED SMF cable ❌ No link (core too small, light disperses)

// Diagnostic type mismatch

Symptom : No link fiber connection

Step 1 : Check cable jacket color
→ Yellow = SMF
→ Orange/Aqua = MMF

Step 2 : Check transceiver type
show interfaces transceiver (Cisco)
→ 1000BASE-LX, 10GBASE-LR = SMF (Long Range laser)
→ 1000BASE-SX, 10GBASE-SR = MMF (Short Range LED)

Step 3 : Verify match
SMF transceiver + SMF cable ? ✅
MMF transceiver + MMF cable ? ✅
Mismatch ? ❌ → Replace cable OR transceiver
⚠️
SMF laser into MMF = peut fonctionner short distance SMF laser assez puissant → works MMF cable <100m (unreliable, not recommended).
MMF LED into SMF = NEVER works (core trop small, light loss massive).
Always match types correctly (safety + reliability).
Signal strength

Budget optique — Power budget

Budget optique (Optical Power Budget) = Perte signal maximale acceptable entre transmitter → receiver. Détermine distance max fiber link.

// Composants budget optique

TX Power (Transmit Power) :
→ Puissance lumière transmitter (dBm)
→ Example : -3 dBm

RX Sensitivity (Receiver Sensitivity) :
→ Minimum puissance receiver detect signal (dBm)
→ Example : -20 dBm

Power Budget :
→ Power Budget = TX Power - RX Sensitivity
→ Example : -3 dBm - (-20 dBm) = 17 dB budget

// Pertes signal (attenuation)

Atténuation = Perte signal le long fiber. Composants :

  • Fiber cable loss : 0.3-0.5 dB/km (SMF), 2-3 dB/km (MMF)
  • Connector loss : 0.3-0.5 dB per connector (SC, LC)
  • Splice loss : 0.1-0.3 dB per splice (fusion splice)
  • Bend loss : Variable (excessive bends add loss)

// Calcul budget optique

Example scenario :

Link : Building A ↔ Building B (5 km SMF)

TX Power : -3 dBm
RX Sensitivity : -20 dBm
Power Budget : 17 dB

Losses :
Fiber cable : 5 km × 0.4 dB/km = 2.0 dB
Connectors : 4 × 0.5 dB = 2.0 dB
Splices : 2 × 0.2 dB = 0.4 dB
Total Loss : 4.4 dB

Margin : 17 dB (budget) - 4.4 dB (loss) = 12.6 dB margin ✅

→ Link viable (margin positive)

// Problème budget insuffisant

Example problème :

Link : 10 km MMF (INCORRECT - MMF max ~500m)

TX Power : -3 dBm
RX Sensitivity : -18 dBm
Power Budget : 15 dB

Losses :
Fiber cable : 10 km × 2.5 dB/km = 25 dB
Connectors : 4 × 0.5 dB = 2 dB
Total Loss : 27 dB

Margin : 15 dB - 27 dB = -12 dB (NEGATIVE) ❌

→ Signal trop faible (no link OU intermittent errors)

// Solutions budget insuffisant

  • Use SMF instead MMF (longer distance support)
  • Add fiber amplifier (EDFA - Erbium Doped Fiber Amplifier)
  • Reduce connectors/splices (minimize insertion loss)
  • Higher power transceiver (upgrade TX power)
Testing tool

OTDR — Optical Time Domain Reflectometer

OTDR = Test equipment advanced localiser breaks, splices, connectors, measure loss fiber link.

// Principe OTDR

Workflow OTDR :

1. OTDR sends light pulse into fiber

2. Light reflects back (backscatter) from :
   → Fiber imperfections
   → Connectors
   → Splices
   → Breaks (fiber end)

3. OTDR measures time reflection returns
   → Calculate distance : Speed light × Time / 2

4. OTDR displays trace graph :
   → X-axis : Distance (meters/km)
   → Y-axis : Signal loss (dB)
   → Shows events (connectors, splices, breaks)

// OTDR trace interprétation

Typical OTDR trace :

Signal (dB)
  0 ─┐
      │                             (Gradual slope = fiber attenuation)
-10 ─┤
      │
-20 ─┤       ▼ (Small drop = connector 0.5 dB loss)
      │
-30 ─┤                       ▼ (Splice 0.2 dB)
      │
-40 ─┴─────────────────────────────────┐
      0        1 km        2 km       3 km  (End reflection)

Events visible :
→ Connector @ 500m : 0.5 dB loss
→ Splice @ 1.5 km : 0.2 dB loss
→ Fiber end @ 3 km : Total link 3 km

// Usages OTDR

  • Locate fiber break : Distance exact break (repair crews)
  • Measure total loss : Verify optical budget compliance
  • Identify bad splice/connector : Excessive loss specific point
  • Certification new fiber install : Document baseline performance
  • Troubleshoot intermittent issues : Detect microbends, stress points

// OTDR vs Light Meter

ToolFunctionInformationCost
Light Meter Measure total loss end-to-end Total dB loss (no distance info) $$ (hundreds)
OTDR Locate events, measure loss per segment Distance + loss each event $$$$ (thousands)
📡
OTDR = advanced troubleshooting Light meter : Simple total loss (quick test OK/NOK).
OTDR : Detailed analysis (pinpoint exact location problems).
Datacenter : Light meter sufficient most cases.
Telco/Long haul : OTDR essential (locate breaks km fiber runs).
⚠️ Pièges d'examen

Ce qui trompe les candidats

❌ Piège #1 : SMF = always better (longer distance)
NUANCE. SMF = longer distance MAIS higher cost.

Short distance datacenter (<300m) : MMF sufficient, cheaper (LED transceivers).
Long distance inter-building (>1 km) : SMF required (laser transceivers).

Choose based distance requirements + budget.
Examen : Match fiber type to scenario distance.
❌ Piège #2 : Optical budget = transceiver power only
INCOMPLET. Budget optique = TX Power - RX Sensitivity.

Pas juste TX power → Need BOTH TX (transmit) + RX (receive sensitivity).
Budget = maximum loss tolerable (TX to RX threshold).

Example : TX -3 dBm, RX -20 dBm → Budget 17 dB.
❌ Piège #3 : OTDR = only tool test fiber
FAUX. Multiple tools fiber testing.

Light Meter : Measure total loss (simple, cheap).
OTDR : Locate events, distance info (complex, expensive).
Visual Fault Locator (VFL) : Red laser visible breaks/bends (<5 km).
Fiber microscope : Inspect connector cleanliness.

Choose appropriate tool task.
Résumé

🔑 Ce que tu dois retenir absolument

→Connecteurs sales/cassés : Cause #1 fiber problems. Clean FIRST (fiber pen, alcohol wipes, compressed air). Inspect microscope. Never touch ferrule, use lint-free only
→SMF : Core 8-10µm, laser 1310/1550nm, long distance 10-100km, yellow jacket. MMF : Core 50/62.5µm, LED 850/1300nm, short distance 300-550m, orange/aqua jacket
→Type mismatch : SMF transceiver + MMF cable = may work <100m unreliable. MMF transceiver + SMF cable = no link (core too small). Always match types
→Budget optique : Power Budget = TX Power - RX Sensitivity. Total Loss = fiber + connectors + splices. Margin = Budget - Loss (must be positive). SMF 0.3-0.5 dB/km, MMF 2-3 dB/km
→OTDR : Optical Time Domain Reflectometer. Sends pulse, measures reflections, calculates distance. Trace shows events (connectors, splices, breaks) with distance + loss. Expensive, advanced troubleshooting
→Tools : Fiber microscope (inspect), Cleaning pen (clean), Light meter (total loss simple), OTDR (locate events detailed), VFL (visual red laser locate breaks). Choose appropriate tool scenario
