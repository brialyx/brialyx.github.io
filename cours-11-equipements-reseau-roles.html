Layer 1

Hub — répéteur multiport (obsolète)

Hub = équipement Layer 1 (Physique) qui répète signal électrique sur TOUS les ports.

// Fonctionnement

  • Reçoit trame sur port 1 → répète sur TOUS les autres ports
  • ❌ Pas d'intelligence (ne lit pas MAC, ne filtre rien)
  • ❌ Domaine de collision unique (tous ports partagent bande passante)
  • ❌ Half-duplex uniquement

// Problèmes Hub

  • Collisions : 2 équipements transmettent en même temps → collision
  • Bande passante partagée : Hub 100Mbps avec 10 PCs = 10Mbps par PC
  • Sécurité zéro : Tout le monde voit tout le trafic (facile à sniffer)
🚫
Hub = OBSOLÈTE Plus utilisé depuis ~2000. Remplacé par switches. Si tu vois Hub en production → remplacer immédiatement par switch. CompTIA teste concept pour culture générale uniquement.
Layer 2

Switch — commutation intelligente MAC

Switch = équipement Layer 2 (Data Link) qui forward trames basé sur adresses MAC.

// Fonctionnement

  1. Apprentissage MAC : Switch construit table MAC (MAC ↔ port)
  2. Forward intelligent : Trame vers MAC connue → envoi UNIQUEMENT sur port destination
  3. Flooding : MAC inconnue → broadcast sur tous ports (sauf source)
  4. Filtrage : Trame intra-port ignorée

// Avantages vs Hub

  • ✅ Domaines de collision séparés (1 par port)
  • ✅ Full-duplex : TX et RX simultanés
  • ✅ Bande passante dédiée : Chaque port = 100Mbps/1Gbps complet
  • ✅ Meilleure sécurité : Trafic pas broadcast (sauf ARP/broadcast)

Layer 3

Routeur — routage inter-réseaux

Routeur = équipement Layer 3 (Network) qui route paquets entre réseaux différents basé sur adresses IP.

// Rôle principal

  • Interconnecter réseaux (LANs, WANs, Internet)
  • Prendre décisions routage basées sur table de routage
  • Séparer domaines de broadcast (1 broadcast domain par interface)
  • NAT/PAT (traduire IPs privées ↔ publiques)

Sécurité

Firewall — filtrage et protection

Firewall = équipement Layer 3-7 qui filtre trafic selon règles de sécurité.

// Rôles

  • Bloquer trafic non autorisé (entrant/sortant)
  • Autoriser uniquement services légitimes
  • Protéger périmètre réseau (Inside ↔ Outside)
  • Créer zones sécurisées (DMZ)

Types : Stateless, Stateful, NGFW (Next-Generation Firewall).

→ Détails complets dans Cours 12

Sans-fil

WAP (Wireless Access Point)

WAP = équipement Layer 2 qui permet connexion Wi-Fi. Bridge entre réseau filaire (Ethernet) et sans-fil (802.11).

// Rôle

  • Émettre signal Wi-Fi (SSID, sécurité WPA2/WPA3)
  • Accepter connexions clients sans-fil
  • Forward trames entre Wi-Fi ↔ Ethernet
  • Gérer authentification clients

Types : AP autonome, AP léger (contrôlé par WLC).

→ Détails complets dans Cours 7-10 (Wi-Fi)

Intermédiaire

Proxy — intermédiaire applicatif

Proxy = équipement Layer 7 (Application) qui agit comme intermédiaire entre client et serveur.

// Types

Forward Proxy : Client → Proxy → Internet

  • Cache web (accélérer accès sites fréquents)
  • Filtrage contenu (bloquer sites interdits)
  • Anonymisation (masquer IP client)
  • Usage : Entreprises (contrôle employés)

Reverse Proxy : Internet → Proxy → Serveurs internes

  • Load balancing (répartir requêtes)
  • SSL termination (déchiffrer HTTPS)
  • Protection serveurs (cache, WAF)
  • Usage : Datacenters, sites web haute disponibilité

→ Détails complets dans Cours 14

Haute disponibilité

Load Balancer — répartition de charge

Load Balancer = équipement Layer 4-7 qui répartit trafic entre plusieurs serveurs.

// Rôles

  • Haute disponibilité : Si serveur tombe → redirect vers autres
  • Scalabilité : Ajouter serveurs = plus de capacité
  • Performances : Répartir charge uniformément

// Algorithmes

  • Round-robin : Tour à tour (serveur 1, 2, 3, 1, 2...)
  • Least-connection : Serveur avec moins de connexions actives
  • IP hash : Client IP → toujours même serveur (persistence)

Détection/Prévention

IDS vs IPS — intrusion detection/prevention

// IDS (Intrusion Detection System)

IDS = système de détection PASSIF. Écoute trafic en copie (mirroring), alerte admin si attaque détectée.

  • Mode : Out-of-band (passif)
  • Action : Alerte uniquement (logs, SIEM)
  • Impact : ❌ Ne bloque rien

// IPS (Intrusion Prevention System)

IPS = système de prévention ACTIF. Inline dans flux, bloque attaques en temps réel.

  • Mode : In-band (inline)
  • Action : Bloque + alerte
  • Impact : ✅ Arrête attaques

⚠️ Pièges d'examen

Ce qui trompe les candidats

❌ Piège #1 : Switch = Layer 3
FAUX. Switch standard = Layer 2 (MAC).

Switch Layer 3 existe MAIS c'est une catégorie spéciale (switch + routeur). Par défaut, switch = L2.
❌ Piège #2 : IDS bloque attaques
FAUX. IDS = alerte uniquement (passif).

IPS = bloque attaques (actif, inline)
❌ Piège #3 : Hub = Switch lent
FAUX. Hub ≠ Switch lent.

Hub = Layer 1 (répète tout, domaine collision unique, obsolète)
Switch = Layer 2 (décisions MAC, domaines collision séparés)
Résumé

🔑 Ce que tu dois retenir absolument

→Hub (L1) : Répète tout, obsolète. Switch (L2) : MAC, intelligent. Routeur (L3) : IP, inter-réseaux
→Switch L3 : Combine switching L2 + routage L3 (inter-VLAN rapide)
→Firewall (L3-7) : Filtrage sécurité, règles ACL, zones DMZ
→WAP (L2) : Access Point Wi-Fi, bridge Ethernet ↔ 802.11
→Proxy (L7) : Forward (client→Internet) vs Reverse (Internet→serveurs)
→Load Balancer (L4-7) : Répartition charge, haute disponibilité (round-robin, least-connection)
→IDS : Passif, alerte. IPS : Actif inline, bloque attaques
