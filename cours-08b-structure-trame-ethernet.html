Avant de commencer

C'est quoi une trame Ethernet ?

✉️
Analogie — une lettre postale Une trame Ethernet, c'est comme une lettre dans une enveloppe :
- Adresse expéditeur (MAC source) — qui envoie
- Adresse destinataire (MAC destination) — qui reçoit
- Le contenu de la lettre (Payload) — les données réelles
- Le cachet de la poste (FCS) — vérification que rien n'est abîmé

Une trame Ethernet est l'unité de données de la couche 2 (Liaison de données) du modèle OSI. C'est l'enveloppe qui transporte les paquets IP sur un réseau local (LAN).

Rappel de l'encapsulation :

  • Couche 7-5 (Application) → Données
  • Couche 4 (Transport) → Segment (TCP) ou Datagramme (UDP)
  • Couche 3 (Réseau) → Paquet (IP)
  • Couche 2 (Liaison) → Trame (Ethernet) ← ON EST ICI
  • Couche 1 (Physique) → Bits électriques/optiques
🎯
Pour l'examen Network+ CompTIA teste la structure exacte de la trame : les 7 champs, leurs tailles en octets, la taille min/max de la trame. C'est du par cœur pur — apprends le tableau et tu gagnes des points faciles.
Vocabulaire essentiel

Les mots clés de ce cours

Trame (Frame)
Unité de données de la couche 2. Contient les adresses MAC, les données, et un contrôle d'erreur.
Préambule (Preamble)
Séquence de bits (10101010...) qui synchronise l'horloge du récepteur. 7 octets.
SFD (Start Frame Delimiter)
1 octet (10101011) qui marque le début de la trame juste après le préambule.
Adresse MAC (Media Access Control)
Adresse physique unique de 6 octets (48 bits) attribuée à chaque carte réseau. Format : AA:BB:CC:DD:EE:FF
EtherType
Champ de 2 octets qui indique le protocole de la couche supérieure (0x0800 = IPv4, 0x86DD = IPv6, 0x0806 = ARP)
Payload (Charge utile)
Les données transportées. Contient généralement un paquet IP. Taille : 46 à 1500 octets.
FCS (Frame Check Sequence)
Somme de contrôle de 4 octets (CRC32) qui permet de détecter les erreurs de transmission.
Runt
Trame trop petite (< 64 octets). Considérée comme invalide et rejetée.
Giant / Jumbo
Trame plus grande que la taille standard max (1518 octets). Giant = erreur. Jumbo = voulu (jusqu'à 9000 octets).

// Détail de chaque champ

ChampTailleRôleExemple / Notes
Préambule 7 octets Synchronisation de l'horloge du récepteur Séquence : 10101010 répétée 7 fois. Pas inclus dans le calcul de taille de trame.
SFD
(Start Frame Delimiter)		 1 octet Marque le début de la trame Valeur : 10101011. Signal "les vraies données commencent maintenant".
MAC Destination 6 octets Adresse physique du destinataire Exemple : AA:BB:CC:DD:EE:FF. Peut être unicast, multicast, ou broadcast (FF:FF:FF:FF:FF:FF).
MAC Source 6 octets Adresse physique de l'expéditeur Permet au destinataire de savoir qui a envoyé la trame.
EtherType (ou Length) 2 octets Indique le protocole de couche 3 0x0800 = IPv4, 0x86DD = IPv6, 0x0806 = ARP. Si < 1536 = champ Length (Ethernet 802.3).
Payload
(Données)		 46-1500 octets Contenu utile : paquet IP, ARP, etc. Si < 46 octets → padding ajouté pour atteindre le minimum. 1500 octets = MTU standard.
FCS
(Frame Check Sequence)		 4 octets Détection d'erreurs (CRC32) Calculé sur tous les champs sauf Préambule + SFD + FCS lui-même. Si erreur détectée → trame rejetée.
🔑
Calcul de la taille de trame Minimum : 6 (MAC dest) + 6 (MAC src) + 2 (EtherType) + 46 (payload min) + 4 (FCS) = 64 octets
Maximum : 6 + 6 + 2 + 1500 (payload max) + 4 = 1518 octets

Note : Préambule (7B) + SFD (1B) = 8 octets ne sont PAS comptés dans la taille de trame.
Adressage couche 2

Les adresses MAC — identifiant physique unique

Une adresse MAC (Media Access Control) est un identifiant unique de 48 bits (6 octets) attribué à chaque carte réseau par le fabricant.

// Format d'une adresse MAC

Exemple : 00:1A:2B:3C:4D:5E

  • 6 octets séparés par ":" (ou "-" selon les systèmes)
  • Chaque octet = 2 chiffres hexadécimaux (00 à FF)
  • Les 3 premiers octets = OUI (Organizationally Unique Identifier) — identifie le fabricant
  • Les 3 derniers octets = ID unique attribué par le fabricant

Exemple concret :

00:1A:2B:3C:4D:5E

  • 00:1A:2B → OUI de Cisco Systems
  • 3C:4D:5E → Numéro de série unique de cette carte réseau Cisco

// Types d'adresses MAC

TypeAdresseSignification
Unicast Bit 0 du 1er octet = 0 Destinée à UNE seule machine. Exemple : 00:1A:2B:3C:4D:5E
Multicast Bit 0 du 1er octet = 1 Destinée à un groupe de machines. Exemple : 01:00:5E:XX:XX:XX (multicast IPv4)
Broadcast FF:FF:FF:FF:FF:FF Destinée à TOUTES les machines du réseau local
🎯
Pour l'examen Retiens : FF:FF:FF:FF:FF:FF = broadcast Ethernet (tous les bits à 1). Utilisé pour ARP, DHCP Discovery, Wake-on-LAN…
Détection d'erreurs

FCS — Frame Check Sequence (contrôle d'intégrité)

Le FCS est un champ de 4 octets à la fin de la trame qui contient une somme de contrôle calculée avec l'algorithme CRC-32 (Cyclic Redundancy Check).

// Comment ça fonctionne

Côté émetteur :

  1. L'émetteur calcule un CRC32 sur tous les champs de la trame (MAC dest, MAC src, EtherType, Payload)
  2. Il place le résultat (4 octets) dans le champ FCS
  3. Il envoie la trame complète

Côté récepteur :

  1. Le récepteur recalcule le CRC32 sur les mêmes champs
  2. Il compare son calcul avec le FCS reçu
  3. Si ça correspond → trame valide ✅
  4. Si ça ne correspond pas → trame corrompue → rejetée ❌
⚠️
FCS détecte, ne corrige pas Le FCS permet de détecter les erreurs de transmission (bits flippés, corruption), mais il ne peut PAS les corriger. Si une erreur est détectée, la trame est simplement jetée. C'est à la couche supérieure (TCP) de redemander les données perdues.

// Causes d'erreurs FCS

  • Câble défectueux ou trop long (atténuation excessive)
  • Interférences électromagnétiques (EMI/RFI)
  • Connecteur sale ou oxydé
  • Duplex mismatch (un côté half, l'autre full → collisions)
  • Carte réseau défectueuse
Contraintes de taille

Taille minimum et maximum d'une trame

// Taille minimum : 64 octets

Pourquoi 64 octets minimum ? Pour permettre la détection de collisions en CSMA/CD (mode half-duplex). Une trame doit être assez longue pour qu'une collision soit détectée avant la fin de la transmission.

Calcul : 6 (MAC dest) + 6 (MAC src) + 2 (EtherType) + 46 (payload min) + 4 (FCS) = 64 octets

Si les données sont < 46 octets ? On ajoute du padding (remplissage de zéros) pour atteindre 46 octets minimum.

Exemple : Envoyer un paquet ARP de 28 octets → ajouter 18 octets de padding → payload total = 46 octets → trame valide.

⚠️
Runt frame Une trame < 64 octets s'appelle un runt. Elle est considérée comme invalide et rejetée par le switch. Cause typique : collision, câble défectueux, carte réseau défaillante.

// Taille maximum : 1518 octets (standard)

Calcul : 6 + 6 + 2 + 1500 (payload max) + 4 = 1518 octets

Le MTU (Maximum Transmission Unit) standard Ethernet = 1500 octets de payload. C'est la taille max du paquet IP qu'on peut transporter dans une trame Ethernet standard.

💡
Jumbo Frames Certains datacenters utilisent des Jumbo Frames (jusqu'à 9000 octets de payload). Avantages : moins d'overhead, meilleure performance pour les gros transferts. Inconvénient : tous les équipements du chemin doivent supporter les jumbo frames, sinon → fragmentation ou perte de paquets.
