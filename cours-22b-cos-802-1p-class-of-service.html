Définition

CoS (Class of Service) — Priorité Layer 2

🎫
Analogie — files d'attente aéroport Aéroport security : plusieurs files priorité (First Class, Business, Economy). CoS = badge priorité sur ton billet → switch traite trafic selon priorité (VoIP First Class, web Economy).

CoS (Class of Service) = mécanisme QoS Layer 2 (Data Link). Marque trames Ethernet pour indiquer priorité trafic.

// Où se trouve CoS ?

CoS = 3 bits PCP (Priority Code Point) dans tag 802.1Q (VLAN tag).

Trame Ethernet 802.1Q :

[Dest MAC] [Source MAC] [802.1Q Tag] [Type] [Payload] [FCS]

802.1Q Tag (4 bytes) :
[TPID 2 bytes] [TCI 2 bytes]

TCI (Tag Control Information) :
┌─────────┬─────┬──────────────┐
│ PCP │ DEI │ VLAN ID │
│ 3 bits │ 1 │ 12 bits │
└─────────┴─────┴──────────────┘
↑
CoS = PCP (Priority Code Point)

Remarque critique : CoS nécessite 802.1Q tag. Trafic non-VLAN (untagged) = pas de CoS possible.

⚠️
CoS = Layer 2 UNIQUEMENT CoS perdu au routage (Layer 3). Switch → Switch = CoS préservé. Switch → Routeur = CoS disparu (besoin remarquer DSCP Layer 3).
Valeurs priorité

8 valeurs CoS (0-7) — IEEE 802.1p

3 bits = 2³ = 8 valeurs possibles (0 à 7). Plus élevé = plus prioritaire.

// Tableau valeurs standard

CoSPrioritéAcronymeTrafic typique
0 Best Effort BE Trafic standard (web, email)
1 Background BK Trafic bas priorité (backups)
2 Excellent Effort EE Rarement utilisé
3 Critical Applications CA Signalisation VoIP (SIP, H.323)
4 Video VI Vidéo conférence (streaming temps réel)
5 Voice VO VoIP media (RTP audio) 🔊
6 Internetwork Control IC Protocoles réseau (OSPF, BGP, HSRP)
7 Network Control NC Contrôle réseau critique (STP, LACP)

// Cas d'usage VoIP (LE PLUS IMPORTANT)

Appel téléphonique IP :

1. Signalisation (SIP/H.323) : CoS 3
→ Établir/terminer appel
→ Latence acceptable (quelques secondes OK)

2. Media audio (RTP) : CoS 5
→ Voix temps réel
→ Latence CRITIQUE (<150ms)
→ Priorité MAXIMALE applications business

Réponse : CoS 5 (Voice)

Réponse : CoS 3 (Critical Applications)
Configuration

Marquage CoS — Qui marque et comment ?

// Endpoints (téléphones IP, PCs)

Trust endpoint : Téléphone IP marque lui-même CoS 5 (trafic VoIP sortant).

Remarque : PCs non-trust peuvent abuser → marquer TOUT en CoS 7 (contourner QoS).

// Switch Access (port endpoint)

Trust mode : Préserve CoS marqué par endpoint (téléphones IP Cisco trust).

Untrust mode : Ignore CoS endpoint, remarque selon policy (PCs).

Config Cisco switch (exemple) :

interface GigabitEthernet0/1
 description Telephone IP
 switchport mode access
 switchport voice vlan 100
 mls qos trust cos
 !

→ Port trust CoS téléphone (VLAN 100 voix)
→ Switch préserve CoS 5 trafic RTP

// Queuing (Files d'attente)

Switch utilise CoS pour prioriser files sortie (egress queues).

Exemple 4 queues :

  • Queue 4 (stricte priorité) : CoS 5-7 (VoIP, contrôle)
  • Queue 3 : CoS 3-4 (signalisation, vidéo)
  • Queue 2 : CoS 2
  • Queue 1 (best effort) : CoS 0-1 (web, email)

Strict priority : Queue 4 vidée AVANT queue 3 → VoIP jamais attente (latence minimale).

Layer 2 vs Layer 3

CoS vs DSCP — Différence critique

// CoS (802.1p)

  • Layer 2 (Data Link) — dans trame Ethernet
  • 3 bits (8 valeurs 0-7)
  • Nécessite 802.1Q tag (VLAN)
  • Portée : Domaine Layer 2 (switch → switch)
  • Perdu au routage (Layer 3)

// DSCP (Differentiated Services Code Point)

  • Layer 3 (Network) — dans header IP
  • 6 bits (64 valeurs 0-63)
  • Pas besoin VLAN
  • Portée : End-to-end (traverse routeurs)
  • Préservé au routage

// Comparaison

CritèreCoS (802.1p)DSCP
Couche OSILayer 2 (Data Link)Layer 3 (Network)
Bits3 bits (8 valeurs)6 bits (64 valeurs)
Localisation802.1Q tag (TCI)IP header (ToS/DSCP)
PortéeLAN (switches)End-to-end (routeurs)
Routage❌ Perdu✅ Préservé
Granularité⚠️ 8 classes✅ 64 classes

// Mapping CoS ↔ DSCP

Routeur/Switch L3 peut mapper CoS → DSCP (et vice versa).

Mapping standard VoIP :

CoS 5 (Voice) → DSCP 46 (EF - Expedited Forwarding)
CoS 3 (CA) → DSCP 24 (CS3 - Class Selector 3)
CoS 0 (BE) → DSCP 0 (Best Effort)

→ Trafic Layer 2 (CoS) converti Layer 3 (DSCP) au routage
🔄
Best practice réseau entreprise LAN : Utilise CoS (Layer 2, switches)
WAN/Internet : Utilise DSCP (Layer 3, routeurs)
Border : Switch L3/routeur mappe CoS ↔ DSCP
⚠️ Pièges d'examen

Ce qui trompe les candidats

❌ Piège #1 : CoS fonctionne sans 802.1Q
FAUX. CoS = PCP dans 802.1Q tag.

Trafic untagged (pas VLAN tag) = PAS de CoS possible.
Native VLAN untagged → pas CoS (problème VoIP si native VLAN)
❌ Piège #2 : VoIP = CoS 7
FAUX. VoIP audio = CoS 5 (Voice).

CoS 7 = Network Control (STP, protocoles réseau critique)
CoS 3 = Signalisation VoIP (SIP)
CoS 5 = Media VoIP (RTP audio) ✅
❌ Piège #3 : CoS traverse routeurs
FAUX. CoS = Layer 2 (trame Ethernet).

Routeur décapsule trame → header IP uniquement.
CoS perdu au routage (besoin mapper vers DSCP Layer 3)
Résumé

🔑 Ce que tu dois retenir absolument

→CoS : 3 bits PCP dans 802.1Q tag (Layer 2), 8 valeurs (0=Best Effort, 7=Network Control)
→VoIP : Media audio = CoS 5 (Voice), Signalisation = CoS 3 (Critical Apps)
→Nécessite 802.1Q : Trafic untagged = pas CoS possible. Perdu au routage (Layer 3)
→vs DSCP : CoS = Layer 2 (switches, 8 classes), DSCP = Layer 3 (routeurs, 64 classes, end-to-end)
→Queuing : Switch utilise CoS pour prioriser egress queues. Strict priority pour CoS 5-7 (VoIP/contrôle)
