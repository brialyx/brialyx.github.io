Attack lifecycle

Cyber Kill Chain — Cycle de vie attaque

🎯
Analogie — cambriolage maison Attaquant ne débarque pas directement. D'abord : observe quartier (reconnaissance), teste portes/fenêtres (scanning), trouve fenêtre ouverte (exploitation), entre maison (post-exploitation). Cyber kill chain = même logique méthodique.

Cyber Kill Chain = Framework Lockheed Martin décrivant étapes attaque cyber. Comprendre kill chain = anticiper/bloquer attaques AVANT dommages.

// 7 phases kill chain (version originale)

  1. Reconnaissance : Collecter infos cible (réseaux, employees, technologies)
  2. Weaponization : Créer payload malveillant (exploit + backdoor)
  3. Delivery : Livrer payload (email phishing, USB, exploit kit)
  4. Exploitation : Exécuter payload, exploiter vulnérabilité
  5. Installation : Installer backdoor/malware persistant
  6. Command & Control (C2) : Établir communication attaquant ↔ système compromis
  7. Actions on Objectives : Exfiltration data, destruction, ransomware

// Kill chain réseau (simplifié 4 phases)

Network+ focus 4 phases essentielles :

1. RECONNAISSANCE
→ Collecter infos cible (OSINT, DNS, WHOIS)
→ Passive (pas contact direct) ou Active (scans)

2. SCANNING
→ Identifier services actifs (ports ouverts)
→ Détecter vulnérabilités (OS, versions software)

3. EXPLOITATION
→ Exploiter vulnérabilité détectée
→ Gain accès initial système/réseau

4. POST-EXPLOITATION
→ Maintenir accès (backdoors, persistence)
→ Lateral movement (propagation réseau)
→ Exfiltration data, objectifs finaux
🛡️
Défense = casser kill chain Attaque réussie nécessite TOUTES phases. Bloquer 1 phase = attaque échoue. Défense proactive : bloquer reconnaissance/scanning (early phases) = empêche exploitation.
Phase 1

Reconnaissance — Collecte d'informations

Reconnaissance = Collecter maximum informations cible AVANT attaque. Objectif : comprendre infrastructure, identifier cibles vulnérables.

// Passive Reconnaissance

Définition : Collecter infos SANS contact direct cible (pas détectable).

Techniques passives :

OSINT (Open Source Intelligence)

  • Moteurs recherche (Google dorking : site:company.com filetype:pdf)
  • Réseaux sociaux (LinkedIn employees, organigramme)
  • Job postings (technologies utilisées mentionnées offres emploi)
  • Nouvelles/presse (acquisitions, projets publics)

WHOIS

  • Base données registrars domaines
  • Infos : Nom propriétaire, email contact, nameservers, plages IP
  • Commande : whois company.com

DNS Interrogation

  • nslookup company.com : IPs serveurs
  • dig company.com ANY : Records DNS (A, MX, TXT)
  • MX records → Serveurs mail (cibles phishing)
  • TXT records → Infos SPF, DKIM (tech email)

Shodan / Censys

  • Moteurs recherche devices Internet-exposed
  • Cherche : IPs, ports ouverts, services, versions software
  • Ex: Shodan: org:"Company Inc" port:22 → SSH servers exposés

// Active Reconnaissance

Définition : Contact DIRECT cible (détectable IDS/IPS).

Techniques actives :

Ping sweeps

  • Identifier hosts actifs subnet
  • nmap -sn 192.168.1.0/24 (ping scan, pas ports)

DNS Zone Transfer

  • Tenter télécharger TOUTE zone DNS (mal configuré)
  • dig @nameserver company.com AXFR
  • Si réussit → Liste complète subdomains/IPs (jackpot attaquant)

Traceroute

  • Mapper chemin réseau vers cible
  • traceroute company.com
  • Révèle routers intermédiaires, topologie réseau
🎯
Question fréquente examen "Technique collecte infos cible SANS contact direct ?"

Réponse : Passive reconnaissance (OSINT, WHOIS, recherche publique)

"Technique révèle TOUS subdomains DNS si mal configuré ?"
Réponse : DNS Zone Transfer (AXFR)
Phase 2

Scanning — Identification vulnérabilités

Scanning = Identifier services actifs, ports ouverts, OS, vulnérabilités. Prépare exploitation.

// Port Scanning

Objectif : Détecter ports ouverts (services disponibles).

Types scans Nmap :

TCP Connect Scan : nmap -sT target

  • 3-way handshake complet (SYN → SYN-ACK → ACK)
  • ✅ Fiable (completion handshake)
  • ⚠️ Bruyant (logs connexions)

SYN Stealth Scan : nmap -sS target

  • Half-open scan (SYN → SYN-ACK → RST, pas ACK final)
  • ✅ Furtif (pas completion handshake, moins logs)
  • Scan préféré attaquants

UDP Scan : nmap -sU target

  • Scanne ports UDP (DNS 53, SNMP 161, NTP 123)
  • ⚠️ Lent (UDP = pas réponse si fermé)

FIN/NULL/Xmas Scans

  • Envoie paquets TCP anormaux (FIN seul, aucun flag, tous flags)
  • Bypass certains firewalls basiques
  • ✅ Très furtif

// OS Fingerprinting

Objectif : Identifier OS/version cible.

Passive : Analyser TTL, window size paquets (pas scan actif).

Active : nmap -O target (envoie probes spécifiques, analyse réponses).

// Service Version Detection

Objectif : Identifier service exact + version (ex: Apache 2.4.41).

nmap -sV target : Interroge services ports ouverts, détecte versions.

Exemple scan complet :

$ nmap -sS -sV -O -p- 192.168.1.100

PORT     STATE  SERVICE     VERSION
22/tcp   open   ssh         OpenSSH 7.4 (Ubuntu)
80/tcp   open   http        Apache 2.4.41
3306/tcp open   mysql       MySQL 5.7.30

OS: Linux 4.15 (Ubuntu 18.04)

→ Attaquant sait maintenant :
  - SSH ouvert (brute-force possible)
  - Apache 2.4.41 (chercher CVE vulnérabilités)
  - MySQL exposé (tentative login default creds)

// Vulnerability Scanning

Outils : Nessus, OpenVAS, Qualys.

Fonction : Scan automatique vulnérabilités connues (CVE database).

Output : Liste vulns + sévérité CVSS + exploits disponibles.

🚨
Scanning = activité détectable IDS/IPS détectent scans ports (patterns suspects : 1000 connexions 10 secondes). Attaquants sophistiqués : slow scans (1 port/heure), randomize IPs sources, fragmentent paquets.
Phase 3

Exploitation — Compromission initiale

Exploitation = Exploiter vulnérabilité détectée → gain accès système/réseau.

// Types exploits

Remote Code Execution (RCE)

  • Exécuter code arbitraire serveur distant
  • Exemple : CVE-2021-44228 (Log4Shell Java)
  • Impact : Contrôle total serveur

SQL Injection

  • Injecter SQL malveillant input web app
  • Exemple : admin' OR '1'='1 (bypass login)
  • Impact : Dump database, authentification bypass

Buffer Overflow

  • Écrire au-delà buffer mémoire → crash ou code execution
  • Exemple : Exploit vulnérabilité C/C++ (pas bounds checking)

Privilege Escalation

  • Élever privilèges (user → root/admin)
  • Exemple : Exploit kernel Linux (local privilege escalation)

Password Attacks

  • Brute-force : Tester toutes combinaisons (lent)
  • Dictionary attack : Wordlist passwords communs
  • Credential stuffing : Réutiliser leaks passwords (breach databases)
  • Pass-the-hash : Utiliser hash password (pas besoin plaintext)

// Frameworks exploitation

Metasploit : Framework exploitation le plus connu (database exploits + payloads).

Exploit-DB : Repository exploits publics (CVE-based).

Workflow exploitation typique :

1. Scan identifie : Apache 2.4.41 (port 80)
2. Recherche CVE : CVE-2021-41773 (path traversal RCE)
3. Exploit disponible Metasploit
4. Lancer exploit → Gain shell reverse (accès serveur)
5. Vérifier accès : whoami → www-data (user web)
6. Prochaine étape : Post-exploitation (privilege escalation)
Phase 4

Post-Exploitation — Maintien accès & objectifs

Post-exploitation = Actions APRÈS compromission initiale. Objectifs : maintenir accès, étendre contrôle, atteindre objectifs finaux.

// 1. Persistence (Maintien accès)

Objectif : Survivre reboots, updates, détections.

Techniques persistence :

  • Backdoors : Installer shell permanent (netcat listener, SSH key)
  • Scheduled tasks : Cron jobs (Linux), Task Scheduler (Windows) exécutant malware boot
  • Rootkits : Modifier kernel OS (très furtif, hard detect)
  • Compromised accounts : Créer user admin caché

// 2. Privilege Escalation

Objectif : User standard → Admin/Root.

Techniques :

  • Exploiter vuln locale (kernel exploits)
  • Misconfiguration (sudo permissions trop larges)
  • SUID binaries (Linux files exécutent root)

// 3. Lateral Movement

Objectif : Propager réseau (host initial → autres systems).

Techniques :

  • Pass-the-hash : Réutiliser hashes NTLM capturés (Windows)
  • Pivoting : Host compromis = proxy accès réseau interne
  • Credential harvesting : Dumper passwords mémoire (Mimikatz)
  • SMB exploitation : EternalBlue (WannaCry), spreads via SMB

// 4. Data Exfiltration

Objectif : Voler données sensibles.

Méthodes :

  • FTP/SCP : Upload data serveur attaquant
  • DNS tunneling : Encoder data requêtes DNS (bypass firewall)
  • HTTPS exfil : POST data via HTTPS (looks legitimate traffic)
  • Cloud storage : Upload Dropbox/Google Drive (blend normal usage)

// 5. Covering Tracks

Objectif : Effacer preuves intrusion.

Techniques :

  • Clear logs (/var/log/auth.log, Windows Event Logs)
  • Timestamps manipulation (fichiers malware = dates légitimes)
  • Disable monitoring (arrêter syslog, AV)
🔴
Dwell time moyen : 200+ jours Stat industry : Attaquants restent réseaux compromis moyenne 6+ mois AVANT détection. Post-exploitation = phase longue, discrète. Detection = challenge majeur défenseurs.
⚠️ Pièges d'examen

Ce qui trompe les candidats

❌ Piège #1 : Reconnaissance = toujours passive
FAUX. Reconnaissance peut être passive OU active.

Passive : OSINT, WHOIS (pas contact direct)
Active : Ping sweeps, DNS zone transfer, traceroute (contact direct, détectable)

Passive = furtif, Active = détectable IDS
❌ Piège #2 : Post-exploitation = après exfiltration data
INCOMPLET. Post-exploitation = TOUT après compromission initiale.

Inclut : Persistence, privilege escalation, lateral movement, exfiltration, covering tracks.

Exfiltration = 1 action post-exploitation (pas définition complète)
❌ Piège #3 : SYN scan = 3-way handshake complet
FAUX. SYN scan = half-open (SYN → SYN-ACK → RST).

TCP Connect scan = 3-way complet (SYN → SYN-ACK → ACK)
SYN scan = half-open (pas ACK final, plus furtif)

SYN scan = préféré attaquants (stealth)
Résumé

🔑 Ce que tu dois retenir absolument

→Kill Chain : 4 phases réseau : Reconnaissance (collecter infos), Scanning (ports/vulns), Exploitation (gain accès), Post-exploitation (persistence/exfil). Bloquer 1 phase = attaque échoue
→Reconnaissance : Passive (OSINT, WHOIS, DNS, pas contact) vs Active (ping sweeps, zone transfer, traceroute, détectable). Shodan/Censys = moteurs recherche devices exposés
→Scanning : Port scan (nmap -sS stealth, -sT connect, -sU UDP), OS fingerprinting (-O), service version (-sV). Vulnerability scanners (Nessus, OpenVAS)
→Exploitation : RCE (remote code exec), SQL injection, buffer overflow, privilege escalation, password attacks (brute-force, dictionary, pass-the-hash). Metasploit framework
→Post-exploitation : Persistence (backdoors, rootkits), privilege escalation, lateral movement (pivoting, pass-the-hash), data exfiltration (DNS tunneling, HTTPS), covering tracks
