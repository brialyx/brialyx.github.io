Fondamental

Pourquoi documenter le réseau ?

🗺️
Analogie — plan ville Réseau sans documentation = ville sans carte ni plan. Nouveau technicien = touriste perdu. Documentation = GPS réseau → troubleshooting rapide, changements planifiés, continuité opérationnelle.

Documentation réseau = ensemble documents décrivant topologie, équipements, configurations, câblage. Indispensable opérations quotidiennes et gestion changements.

// Problèmes sans documentation

  • ❌ Troubleshooting lent : "Où est connecté ce serveur ?"
  • ❌ Risque changements : Modifier config sans savoir impact global
  • ❌ Onboarding difficile : Nouveaux ingénieurs perdus
  • ❌ Dépendance personnes : "Seul Bob sait comment fonctionne réseau"
  • ❌ Compliance impossible : Audits (PCI-DSS, SOC2) nécessitent documentation

// Bénéfices documentation

  • ✅ Troubleshooting rapide : Diagrammes → identifier problème rapidement
  • ✅ Planification changements : Comprendre dépendances avant modifier
  • ✅ Disaster recovery : Rebuild réseau depuis documentation
  • ✅ Onboarding : Nouveaux techniciens autonomes
  • ✅ Compliance : Preuves audits
📝
Documentation = investissement temps payant 30 minutes documenter changement maintenant = économiser 4 heures troubleshooting dans 6 mois. Documentation pas luxe → nécessité opérationnelle.
Topologies

Diagrammes logiques vs physiques

// Diagramme logique (Logical Network Diagram)

Focus : Comment données circulent (Layer 3). VLANs, subnets, routage, adressage IP.

Contenu :

  • Subnets IP : 10.0.1.0/24 (VLAN 10 Data), 10.0.2.0/24 (VLAN 20 Voice)
  • VLANs : VLAN 10, 20, 30 + noms
  • Routage : Routes statiques, protocoles (OSPF, BGP)
  • Adresses IP équipements : Router 192.168.1.1, Firewall 192.168.1.254
  • Services réseau : DHCP, DNS, NTP
  • Flux trafic : Internet → Firewall → Core → Distribution → Access

Exemple visuel :

[Internet]
↓
[Router 203.0.113.1/30]
↓
[Firewall DMZ]
↓
[Core Switch Layer 3]
├─ VLAN 10 (10.0.1.0/24) — Data
├─ VLAN 20 (10.0.2.0/24) — Voice
└─ VLAN 30 (10.0.3.0/24) — Serveurs

→ Pas localisation physique salles
→ Focus : IP, VLANs, routage

Usage : Planifier routage, dépannage Layer 3, comprendre flux trafic.

// Diagramme physique (Physical Network Diagram)

Focus : Où sont équipements physiquement. Salles, racks, câblage, connexions ports.

Contenu :

  • Localisation salles : MDF Étage 1, IDF Étage 3
  • Positions racks : Rack A, Rack B
  • Unités rack : Switch Core 1 (Rack A, U20-U22)
  • Connexions physiques : Port Gi0/1 Switch A → Port Gi0/24 Switch B
  • Types câbles : Fibre OM3, Cat6, longueurs
  • Patch panels : PP1 Port 12 → Switch Port 12

Exemple visuel :

MDF — Salle serveurs Étage 1
[Rack A — 42U]
U38-40: Core Switch 1
U35-37: Core Switch 2
U30: Firewall

IDF — Placard Étage 3
[Rack B — 24U]
U20-21: Access Switch 1 (48 ports)
U18: Patch Panel (48 ports)

Connexions :
Core SW1 Port Te1/1 ←(Fibre OM3 150m)→ Access SW1 Port Gi0/48

→ Pas d'IPs ni VLANs
→ Focus : Localisation physique, câblage

Usage : Installation équipements, dépannage câblage, maintenance physique.

  • Numéro série : FOC2345ABCD (warranty, support)

Wiring

Schémas de câblage

Wiring diagram = documentation connexions câbles point-à-point. Essentiel dépannage physique.

// Contenu schéma câblage

  • Point A (source) : Switch Core-01 Port Gi0/1
  • Point B (destination) : Patch Panel PP1 Port 12
  • Type câble : Cat6 UTP
  • Longueur : 25m
  • ID câble : CAB-0123
  • Chemin physique : Cable tray plafond, riser vertical

// Exemple documentation connexions

Connexion 1 :
Source: SW-CORE-01 Gi0/1
Dest: PP-MDF-01 Port 12
Câble: CAB-0045 (Cat6, 15m, Bleu)
Chemin: Cable tray Nord, conduit vertical

Connexion 2 :
Source: PP-MDF-01 Port 12
Dest: Bureau 305 Prise murale
Câble: CAB-0046 (Cat6, 60m, permanent)
Chemin: Riser vertical, faux-plafond étage 3

// Étiquetage câbles

Indispensable : Identifier physiquement chaque câble.

  • Étiquettes 2 extrémités : CAB-0045 (même ID chaque bout)
  • Label machines : Imprimantes étiquettes réseau
  • Codes couleur (optionnel) : Bleu = Data, Jaune = Voice, Rouge = Backbone
🏷️
Convention étiquetage Cohérence CRITIQUE. Exemples :
- CAB-XXXX (numéro séquentiel)
- LOC-SRC-DST (MDF-SW01-PP12)
Choisir convention, documenter, appliquer partout.
Référence performance

Baseline réseau

Baseline = mesure performances réseau normales (conditions opérationnelles standards). Référence détecter anomalies.

// Métriques baseline typiques

  • Utilisation bande passante : Lien WAN 40% utilisé heures pointe (baseline)
  • Latence : Ping DC1 → DC2 moyenne 15ms
  • Perte paquets : 0.01% perte normale
  • CPU équipements : Core switch CPU 25% moyen
  • Nombre connexions : Firewall 5000 sessions simultanées typiques
  • Erreurs interfaces : <10 CRC errors/jour acceptable

// Capturer baseline

Période représentative : Mesurer 1-4 semaines (couvrir cycles business).

Heures variées : Jour/nuit, semaine/weekend (patterns différents).

Outils :

  • SNMP monitoring : PRTG, Zabbix, Nagios
  • NetFlow/sFlow : Analyser flux trafic
  • Scripts ping/traceroute : Mesurer latence régulièrement

// Usage baseline

Détection anomalies : Utilisation lien 95% (vs baseline 40%) → investigation.

Capacity planning : Baseline montre croissance → planifier upgrade.

Troubleshooting : Comparer performance actuelle vs baseline → identifier dégradation.

Exemple baseline utilisé troubleshooting :

Utilisateurs : "Internet lent ce matin"

Check monitoring :
- Lien WAN utilisation : 98% (baseline normal = 40%)
- Latence Internet : 250ms (baseline = 20ms)

→ Anomalie détectée (déviation baseline)
→ Investigation trafic (NetFlow) → Backup job anormal
→ Action : Throttle backup, problème résolu
📊
Baseline = données, pas gut feeling "Réseau semble lent" = subjectif. Baseline = données objectives. Pas baseline = impossible prouver dégradation ou normalité.

⚠️ Pièges d'examen

Ce qui trompe les candidats

❌ Piège #1 : Diagramme physique montre VLANs
FAUX. VLANs = concept logique.

Diagramme physique : Localisation, racks, câbles
Diagramme logique : VLANs, IPs, subnets
❌ Piège #2 : Baseline = limite maximale performance
NON. Baseline = performance normale (moyenne).

Pas limite max. Baseline utilisation 40% ≠ lien capacité 40%.
Baseline = référence détecter anomalies (hausse/baisse anormale)
❌ Piège #3 : Documentation unique document
FAUX. Documentation = ensemble documents :

- Diagrammes (logique + physique)
- Inventaire équipements
- Schémas câblage
- Configs backup
- Baseline performance
- Procédures (runbooks)
Résumé

🔑 Ce que tu dois retenir absolument

→Diagramme logique : IPs, VLANs, subnets, routage (flux données). Diagramme physique : Salles, racks, câbles, ports (localisation)
→Inventaire : Liste complète équipements (hostname, modèle, SN, IP, localisation, garantie). Mise à jour chaque changement
→Schémas câblage : Connexions point-à-point (source, dest, type câble, longueur, ID). Étiquetage 2 extrémités CRITIQUE
→Baseline : Performances normales (bande passante, latence, CPU). Référence détecter anomalies. Capturer 1-4 semaines période représentative
→Outils : Draw.io/Visio (diagrammes), NetBox (inventaire/IPAM), PRTG/Zabbix (monitoring baseline)
