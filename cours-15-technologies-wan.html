Circuit dédié legacy

Leased Line — circuit point-à-point dédié

Leased Line = connexion physique dédiée point-à-point entre 2 sites. Bande passante garantie 100% du temps.

// Types

  • T1 (USA) : 1.544 Mbps (24 canaux DS0 de 64 Kbps)
  • E1 (Europe) : 2.048 Mbps (32 canaux de 64 Kbps)
  • T3 : 44.736 Mbps (28 circuits T1)
  • E3 : 34.368 Mbps

// Caractéristiques

  • ✅ Bande passante garantie (SLA strict)
  • ✅ Latence faible et prévisible
  • ✅ Sécurité (ligne privée physique)
  • ❌ Coût extrêmement élevé ($/km)
  • ❌ Débit faible pour standards modernes
  • ❌ Installation longue (semaines/mois)
📉
Leased Lines = OBSOLÈTE T1/E1 très peu utilisés aujourd'hui (débit faible 1.5Mbps, coût élevé). Remplacés par Metro Ethernet, MPLS, SD-WAN. CompTIA teste concept pour culture générale.
Fibre haute capacité

SONET / SDH — transport optique

SONET (Synchronous Optical Network - USA) et SDH (Synchronous Digital Hierarchy - International) = standards transport données sur fibre optique.

// Hiérarchie SONET

// Usage

  • Backbones ISP (Internet Service Providers)
  • Réseaux télécoms longue distance
  • Connexions datacenters haute capacité

Remarque : SONET/SDH utilisé par ISPs, pas directement par entreprises typiques (trop complexe/cher).

Ethernet métropolitain

Metro Ethernet — Ethernet sur MAN

Metro Ethernet = service Ethernet étendu sur zone métropolitaine (MAN). Connecte sites d'une même ville/région via infrastructure ISP.

// Avantages

  • ✅ Interface familière : Ethernet standard (pas de conversion protocole)
  • ✅ Débits élevés : 10 Mbps à 100 Gbps
  • ✅ Coût raisonnable : Moins cher que leased lines
  • ✅ Scalable : Augmenter débit facilement
  • ✅ SLA : Bande passante garantie, latence faible

// Types services Metro Ethernet

E-Line : Point-à-point (2 sites)

E-LAN : Multipoint (plusieurs sites, any-to-any)

E-Tree : Multipoint hiérarchique (hub → spokes, spokes ne communiquent pas entre eux)

// Usage

  • Relier bureaux d'une entreprise dans même ville
  • Connexion datacenter à bureaux locaux
  • Alternative moins chère que MPLS pour zones métro
✅
Metro Ethernet = excellent choix multi-sites local Entreprise avec 5 bureaux dans même ville → Metro Ethernet E-LAN = connexion any-to-any, débits élevés (1-10 Gbps), coût raisonnable. Standard moderne.
Label switching

MPLS — Multi-Protocol Label Switching

MPLS = technologie routage basée sur labels (pas IP). Utilisée pour créer WANs privés haute performance entre sites distribués.

// Fonctionnement

  1. Trafic entre dans réseau MPLS → routeur edge (PE - Provider Edge) ajoute label
  2. Routeurs core (P - Provider) switchent basé sur label (PAS lookup IP → ultra rapide)
  3. Routeur edge destination retire label, délivre paquet

// Avantages MPLS

  • ✅ QoS (Quality of Service) : Priorisation trafic (voix, vidéo prioritaires)
  • ✅ Performances : Switching labels plus rapide que routage IP
  • ✅ Any-to-any : Full mesh logique entre tous sites
  • ✅ Sécurité : Réseau privé isolé (pas Internet public)
  • ✅ SLA garantis : Latence, jitter, disponibilité

// Inconvénients MPLS

  • ❌ Coût TRÈS élevé : Mensuel par site + bande passante
  • ❌ Provisioning lent : 30-90 jours ajouter nouveau site
  • ❌ Rigidité : Contrats longs, difficile changer
  • ❌ Vendor lock-in : Dépendant d'un seul ISP

// Cas d'usage

  • Entreprises multi-sites (10-100+ bureaux)
  • Applications critiques (VoIP, vidéoconférence, SAP/ERP)
  • Quand QoS et SLA stricts nécessaires
🎯
MPLS = historiquement solution WAN entreprise #1 Avant SD-WAN, MPLS était LE standard pour WANs entreprise (performance, QoS, any-to-any). Aujourd'hui challengé par SD-WAN (moins cher, plus agile).
Overlay software moderne

SD-WAN — Software-Defined WAN

SD-WAN = technologie overlay créant WAN virtuel au-dessus de multiples liens (Internet, MPLS, 4G/5G). Contrôlé par logiciel centralisé.

// Architecture

  • Orchestrateur centralisé : Console cloud gérant toutes politiques
  • Appliances sites : Équipements physiques/virtuels par bureau
  • Liens multiples : Combine Internet broadband + MPLS + 4G/5G
  • Tunnels chiffrés : Overlay sécurisé site-to-site

// Fonctions clés

1. Agrégation liens : Utilise TOUS les liens simultanément (Internet + MPLS + LTE)

2. Application-aware routing : Route intelligent par application (Office 365 → Internet, VoIP → MPLS)

3. Zero-touch provisioning : Nouveau site online en minutes (plug & play)

4. Centralized policy : Politiques déployées depuis cloud vers tous sites

5. Path optimization : Bascule automatiquement si lien dégrade (failover)

// Avantages SD-WAN

  • ✅ Coût 50-70% moins cher que MPLS (utilise Internet)
  • ✅ Agilité : Nouveau site en heures (pas 90 jours)
  • ✅ Bande passante élevée : Internet broadband >> MPLS
  • ✅ Résilience : Multiples liens (failover auto)
  • ✅ Cloud-ready : Optimisé pour SaaS (Office 365, Salesforce)

⚠️ Pièges d'examen

Ce qui trompe les candidats

❌ Piège #1 : SD-WAN remplace MPLS complètement
PAS TOUJOURS. SD-WAN peut remplacer MPLS MAIS :

Applications ultra-critiques (trading financier, santé temps réel) peuvent nécessiter SLA MPLS garantis.
Solution hybrid SD-WAN + MPLS = courante (critique → MPLS, reste → Internet)
❌ Piège #2 : MPLS = VPN
DIFFÉRENT.
MPLS : Réseau privé Layer 3 avec labels, géré par ISP
VPN : Tunnel chiffré sur Internet

MPLS plus performant/fiable MAIS beaucoup plus cher
❌ Piège #3 : Metro Ethernet = Internet
FAUX.
Metro Ethernet : Connexion privée point-à-point/multipoint via ISP (pas Internet public)
Internet : Réseau public

Metro Ethernet = sécurité + SLA, pas Internet
Résumé

🔑 Ce que tu dois retenir absolument

→Leased Line (T1/E1) : Circuit dédié obsolète (1.5Mbps, très cher)
→SONET/SDH : Fibre optique ISP (51Mbps-10Gbps), backbones télécoms
→Metro Ethernet : Ethernet MAN (10Mbps-100Gbps), multi-sites zone métro, coût raisonnable
→MPLS : Label switching, QoS garanti, any-to-any, très cher, provisioning lent
→SD-WAN : Overlay software, agrégation liens, 50-70% moins cher MPLS, agile, cloud-ready (standard moderne)
