Avant de commencer

C'est quoi un VLAN — et pourquoi c'est essentiel

🏢
Analogie — immeuble avec étages séparés Sans VLAN : tout le monde dans un seul grand open space. Tout le monde entend tout.
Avec VLAN : l'immeuble est divisé en étages. Le département RH (VLAN 10) ne peut pas voir/entendre le département IT (VLAN 20), même s'ils sont sur le même switch physique.

Un VLAN (Virtual Local Area Network) est un réseau local virtuel créé en segmentant logiquement un switch physique en plusieurs réseaux isolés.

Principe clé : Un switch physique peut héberger plusieurs VLANs. Chaque VLAN est un domaine de broadcast séparé.

// Pourquoi utiliser des VLANs ?

AvantageExplication
SécuritéIsoler les départements. Le VLAN RH ne peut pas communiquer directement avec le VLAN invités.
PerformanceRéduire le trafic broadcast. Un broadcast dans VLAN 10 reste dans VLAN 10.
OrganisationGrouper logiquement : VLAN par département, par fonction, par niveau de sécurité.
FlexibilitéChanger un PC de VLAN = changer la config du port. Pas besoin de le déplacer physiquement.
Économie1 switch physique = plusieurs réseaux logiques. Pas besoin d'un switch par département.
🎯

Vocabulaire essentiel

Les mots clés de ce cours

VLAN ID
Numéro unique qui identifie un VLAN (1 à 4094). Exemple : VLAN 10 = RH, VLAN 20 = IT, VLAN 30 = Invités.
Access Port
Port configuré pour transporter le trafic d'UN SEUL VLAN. Utilisé pour connecter des appareils finaux (PC, imprimante, téléphone).
Trunk Port
Port configuré pour transporter le trafic de PLUSIEURS VLANs simultanément. Utilisé entre switches ou entre switch et routeur.
802.1Q (Dot1Q)
Standard IEEE de tagging VLAN. Ajoute un tag de 4 octets dans la trame Ethernet pour identifier le VLAN.
VLAN Tag
Champ de 4 octets inséré dans la trame Ethernet qui contient le VLAN ID.
Native VLAN
VLAN utilisé pour les trames non taggées sur un trunk. Par défaut = VLAN 1.
Voice VLAN (VLAN voix)
VLAN dédié au trafic des téléphones IP pour prioriser la voix sur les données.
Management VLAN
VLAN utilisé pour l'administration à distance du switch (SSH, Telnet, HTTP). Recommandé de ne PAS utiliser VLAN 1.
Types de ports

Access Port — pour les appareils finaux

Un Access Port est un port configuré pour appartenir à UN SEUL VLAN. C'est le type de port utilisé pour connecter des appareils finaux (PC, imprimantes, serveurs).

// Caractéristiques

  • Porte le trafic d'un seul VLAN
  • Les trames envoyées/reçues sur ce port ne sont PAS taggées (pas de VLAN tag visible pour l'appareil final)
  • L'appareil final (PC) n'a pas besoin de savoir qu'il est dans un VLAN
  • Utilisé pour : PC, imprimantes, serveurs, caméras IP, points d'accès Wi-Fi (en mode simple)

// Configuration exemple (Cisco)

Switch(config)# interface FastEthernet0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10
! Le port Fa0/1 est maintenant dans le VLAN 10

Résultat : Tout appareil branché sur le port Fa0/1 sera automatiquement dans le VLAN 10, sans aucune configuration côté appareil.

✅
Règle simple Access port = 1 VLAN, pour appareils finaux, pas de tag visible
Interconnexion switches

Trunk Port — pour transporter plusieurs VLANs

Un Trunk Port est un port configuré pour transporter le trafic de PLUSIEURS VLANs simultanément. C'est le type de port utilisé entre switches ou entre switch et routeur.

// Caractéristiques

  • Porte le trafic de multiples VLANs sur un seul câble
  • Les trames sont taggées avec le VLAN ID (standard 802.1Q)
  • Permet à deux switches de partager tous leurs VLANs sur une seule connexion
  • Utilisé pour : liaison switch-to-switch, liaison switch-to-routeur (inter-VLAN routing)

// Configuration exemple (Cisco)

Switch(config)# interface GigabitEthernet0/1
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk allowed vlan 10,20,30
! Le trunk autorise les VLANs 10, 20, 30

Résultat : Le port Gi0/1 transporte maintenant le trafic des VLANs 10, 20, et 30 simultanément.

🎯
Règle simple Trunk port = plusieurs VLANs, entre équipements réseau, trames taggées
Standard de tagging

802.1Q — le tagging VLAN standard

IEEE 802.1Q est le standard qui définit comment ajouter un tag VLAN dans une trame Ethernet.

// Structure du tag 802.1Q

Le tag est inséré entre les champs "MAC Source" et "EtherType" de la trame Ethernet. Il fait 4 octets :

ChampTailleDescription
TPID
(Tag Protocol Identifier)		2 octetsValeur fixe : 0x8100. Indique "c'est une trame 802.1Q taggée".
PCP
(Priority Code Point)		3 bitsPriorité QoS (0-7). Utilisé pour prioriser le trafic (voix > données).
DEI
(Drop Eligible Indicator)		1 bitIndique si la trame peut être droppée en cas de congestion.
VLAN ID12 bitsIdentifiant du VLAN (1-4094). 0 et 4095 sont réservés.

Taille de trame avec tag : Une trame Ethernet standard fait 64-1518 octets. Avec un tag 802.1Q, elle fait 68-1522 octets (4 octets de plus).

// Processus de tagging/untagging

Tagging (à l'entrée d'un trunk) :

  1. Trame arrive sur un access port (VLAN 10)
  2. Le switch identifie le VLAN du port : VLAN 10
  3. En sortant sur le trunk, il insère un tag 802.1Q avec VLAN ID = 10
  4. La trame taggée voyage sur le trunk

Untagging (à la sortie d'un trunk) :

  1. Trame taggée arrive sur le trunk (VLAN ID = 10)
  2. Le switch lit le tag : "cette trame est pour VLAN 10"
  3. Il retire le tag 802.1Q
  4. Il envoie la trame non-taggée sur l'access port du VLAN 10
⚠️
Important Les appareils finaux (PC, imprimantes) ne voient JAMAIS les tags VLAN. Le switch ajoute le tag en entrée de trunk et le retire en sortie de trunk. Les tags n'existent que sur les trunks.
Cas particulier

Native VLAN — pour les trames non taggées

Le Native VLAN est un VLAN spécial sur un trunk qui gère les trames non taggées.

// Comment ça fonctionne

Sur un trunk :

  • Toutes les trames des VLANs sont normalement taggées
  • SAUF les trames du Native VLAN qui sont envoyées sans tag

Par défaut, le Native VLAN = VLAN 1

// Pourquoi un Native VLAN ?

  1. Compatibilité : Permet de communiquer avec des équipements qui ne supportent pas 802.1Q
  2. Trafic de contrôle : Certains protocoles (CDP, VTP, STP) utilisent le Native VLAN
  3. Simplification : Évite de tagger inutilement un VLAN si c'est le seul
⚠️
PIÈGE DE SÉCURITÉ — Native VLAN Mismatch Les deux extrémités d'un trunk DOIVENT avoir le même Native VLAN configuré !

Si Switch A a Native VLAN 1 et Switch B a Native VLAN 99 :
- Switch A envoie trame VLAN 1 sans tag
- Switch B reçoit trame sans tag → la met dans VLAN 99
- Résultat : les appareils VLAN 1 communiquent avec VLAN 99 → faille de sécurité !

// Bonne pratique de sécurité

🔒
Recommandation sécurité Ne jamais utiliser VLAN 1 comme Native VLAN. Créer un VLAN inutilisé (ex: VLAN 999) et le configurer comme Native VLAN sur tous les trunks. Comme aucun appareil n'est dans VLAN 999, aucune faille possible.

// Configuration Native VLAN

Switch(config)# interface GigabitEthernet0/1
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk native vlan 999
! Native VLAN changé à 999 (au lieu de 1 par défaut)
VLANs spécialisés

Voice VLAN — pour les téléphones IP

Un Voice VLAN (VLAN voix) est un VLAN dédié au trafic des téléphones IP. Il permet de séparer et prioriser la voix par rapport aux données.

// Pourquoi un VLAN séparé pour la voix ?

  • QoS (Quality of Service) : La voix est sensible à la latence et à la gigue. Un VLAN dédié permet de prioriser le trafic voix.
  • Sécurité : Isoler les téléphones du réseau de données.
  • Gestion : Facilite l'administration (tous les téléphones dans un seul VLAN).

// Configuration typique

Un port avec Voice VLAN supporte 2 VLANs simultanément :

  • Data VLAN (access) : pour le PC branché sur le téléphone
  • Voice VLAN : pour le téléphone IP lui-même
Switch(config)# interface FastEthernet0/5
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10
! VLAN 10 = données (PC)
Switch(config-if)# switchport voice vlan 20
! VLAN 20 = voix (téléphone IP)

Résultat :

  • Le téléphone IP envoie son trafic dans VLAN 20 (taggé)
  • Le PC branché sur le téléphone envoie son trafic dans VLAN 10 (non taggé)
  • Les deux appareils partagent le même câble, mais sont dans des VLANs différents
Administration du switch

Management VLAN — pour gérer le switch

Le Management VLAN est le VLAN utilisé pour l'administration à distance du switch (SSH, Telnet, HTTP, SNMP).

// Pourquoi un VLAN de management dédié ?

  • Sécurité : Isoler le trafic d'administration du trafic utilisateur
  • Contrôle d'accès : Seuls les admins réseau ont accès au VLAN management
  • Disponibilité : Même si le réseau utilisateur est en panne, l'admin peut toujours se connecter

// Configuration

Switch(config)# interface vlan 99
! Créer l'interface virtuelle VLAN 99
Switch(config-if)# ip address 192.168.99.10 255.255.255.0
Switch(config-if)# no shutdown
Switch(config-if)# exit
Switch(config)# ip default-gateway 192.168.99.1
! Le switch est maintenant accessible via 192.168.99.10
🔒
Bonne pratique de sécurité Ne JAMAIS utiliser VLAN 1 comme Management VLAN. VLAN 1 est le VLAN par défaut et présente des risques de sécurité (attaques VLAN hopping). Utiliser un VLAN dédié (ex: VLAN 99).

Résumé

→VLAN = réseau local virtuel, segmente un switch en domaines de broadcast séparés
→Access port : 1 VLAN, appareils finaux, pas de tag visible

→Tag 802.1Q = 4 octets insérés dans la trame (TPID + PCP + DEI + VLAN ID)
